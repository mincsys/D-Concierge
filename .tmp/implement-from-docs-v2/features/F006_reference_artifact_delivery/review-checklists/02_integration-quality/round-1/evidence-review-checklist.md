# Evidence Review Checklist

## 証跡と方針の整合

- [x] 証跡の保存場所、ファイル名、記録項目が対応するテスト方針と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: 単体証跡は `docs/04_テスト/02_単体テスト/evidence/backend-unit-coverage.txt`、結合証跡は `docs/04_テスト/03_結合テスト/evidence/backend-integration-evidence.txt` に保存され、方針が定める key=value 形式で対象、実行日時、コマンド、分岐カバレッジ、分岐数、結果を記録している。
- [x] 証跡がどのテストケース、コマンド、範囲指定、実施日時に対応するか追跡できるか。
  - 検証結果: 指摘なし
  - 確認根拠: 両証跡に `executed_at` と `command` があり、結合証跡には `design_coverage` と `tests=136` が記録されている。生成役報告にもF006対象テスト件数と全体テスト件数がある。
- [x] 証跡に対象コミット、対象差分、実行環境、主要バージョン、ブラウザ、画面サイズなど、再現に必要な前提が残っているか。
  - 検証結果: 対象外
  - 確認根拠: 単体・結合テスト方針のcoverage証跡項目は対象、実行日時、実行コマンド、分岐カバレッジ、総分岐数、通過分岐数、判定結果を求めている。今回の自動バックエンド単体・結合証跡ではブラウザや画面サイズは不要であり、対象差分は依頼元の差分情報で渡されている。
  - 理由: 方針が必須項目としていないため。
- [x] 再テスト時に古い証跡と新しい証跡の扱いが方針と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: 証跡ファイルは方針どおり既定ファイル名で最新結果へ上書きされ、内容もF006実装後の `executed_at=2026-06-23T06:13:51+09:00` と生成役報告の実行結果に対応している。
- [x] 証跡に秘密情報、個人情報、絶対パス、不要な詳細ログが含まれていないか。
  - 検証結果: 指摘なし
  - 確認根拠: 証跡はkey=valueのcoverage summaryに限定され、秘密情報、個人情報、DB URL、詳細ログ、スタックトレースは含まれていない。

## カバレッジ証跡

- [x] 方針で指定された指標だけが記録されているか。
  - 検証結果: 指摘なし
  - 確認根拠: 単体・結合証跡はいずれも分岐カバレッジ、通過分岐数、総分岐数を記録し、行カバレッジ、ステートメントカバレッジ、関数カバレッジは記録していない。
- [x] 対象、実行日時、実行コマンド、カバレッジ値、総数、通過数、判定結果がそろっているか。
  - 検証結果: 指摘なし
  - 確認根拠: 単体証跡は `target`、`executed_at`、`command`、`branch_coverage=95.97%`、`covered_branches=357`、`total_branches=372`、`result=pass` を記録している。結合証跡も同項目に加え `tests=136`、`failures=0` を記録している。

- [x] 測定から外した範囲がテスト方針と矛盾していないか。

  - 確認根拠: 証跡と生成役報告では `--cov=src/backend` でbackend全体を対象にしており、F006のために不自然な追加除外をした記録は確認されなかった。

  - 検証結果: 指摘なし
  - 確認根拠: 依頼元の生成役報告はF006実装後に単体・結合・静的テストを実行した結果として証跡保存先を示し、証跡の `executed_at=2026-06-23T06:13:51+09:00` と対応している。
- [x] 実行コマンドが方針書、設定、package manager、依存管理の現行定義と一致しているか。
  - 検証結果: 指摘なし

## 修正方針の判断

- [x] 証跡が不足している場合は、証跡の追加、再実行、テスト仕様更新のどれが必要かを書く。
  - 検証結果: 対象外
  - 確認根拠: coverage証跡自体の不足は確認されなかった。作成した指摘は実装・テスト観点不足であり、修正後に同じ証跡を再取得する対象である。
  - 理由: 証跡不足ではなく実装・テスト不足のため。
- [x] 証跡が方針と矛盾する場合は、証跡を直すべきか、方針を直すべきかを根拠から判断する。
  - 検証結果: 対象外
  - 確認根拠: 証跡形式と方針の矛盾は確認されなかった。
  - 理由: 証跡修正・方針修正対象ではないため。
- [x] 証跡だけでは判定不能な場合は、必要な再実行条件と対象成果物を具体的に書く。
  - 検証結果: 対象外
  - 確認根拠: 生成役報告と証跡により、既存テスト実行結果とcoverageは判定可能である。
  - 理由: 判断不能ではないため。
