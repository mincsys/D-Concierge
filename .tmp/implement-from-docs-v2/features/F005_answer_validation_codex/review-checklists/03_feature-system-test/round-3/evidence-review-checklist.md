# Evidence Review Checklist

## 証跡と方針の整合

- [x] 証跡の保存場所、ファイル名、記録項目が対応するテスト方針と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: `state.md` の `機能別総合テスト証跡` は `F005-system-test-summary.txt`、`F005-system-test-api-db-codex-boundary.txt`、ST-CHAT-001/003/004 のスクリーンショット、ST-CHAT-025 の3つのテキスト証跡の8件のみになっており、削除済み `evidence/playwright-cli/` は除外済みであることを確認した。総合テスト方針の「詳細実行ログは保存しない」と矛盾しない。
- [x] 証跡がどのテストケース、コマンド、範囲指定、実施日時に対応するか追跡できるか。
  - 検証結果: 指摘なし
  - 確認根拠: `チャット実行テスト.md` の証跡欄から ST-CHAT-001/003/004 のスクリーンショット、ST-CHAT-025 の CLI/version/script/docker ps 証跡、API/DB/Codex境界証跡を追跡できる。各テキスト証跡には `executed_at` と実行内容が記録されていることを確認した。
- [x] 証跡に対象コミット、対象差分、実行環境、主要バージョン、ブラウザ、画面サイズなど、再現に必要な前提が残っているか。
  - 検証結果: 指摘なし
  - 確認根拠: 管理役提示の差分情報で docs 正本非変更、staged pycache/pyc混入なし、対象 issue の修正内容が示されている。証跡には Codex CLI `0.130.0`、Chrome利用、ST-CHAT-025実施時刻、Dockerコマンド、API/DB/Codex境界の実行結果が記録されていることを確認した。
- [x] 再テスト時に古い証跡と新しい証跡の扱いが方針と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: ST-CHAT-025 の3証跡は `executed_at=2026-06-23T05:08:53+09:00` で、旧 `exit_code=127` ではなく backend script 到達後の Codex JSONL が記録されている。state の証跡一覧から旧Playwright詳細ログパスも除外されていることを確認した。
- [x] 証跡に秘密情報、個人情報、絶対パス、不要な詳細ログが含まれていないか。
  - 検証結果: 指摘なし
  - 確認根拠: ST-CHAT-025証跡はコマンド、JSONLのthread ID、refresh token失効メッセージ、exit_codeのみで、トークン値や個人情報は含まれていない。state の証跡一覧にも `evidence/playwright-cli/` は残っていない。

## 総合テスト証跡

- [x] スクリーンショットやテキスト証跡がテストケースIDと対応しているか。
  - 検証結果: 指摘なし
  - 確認根拠: `ST-CHAT-001_initial_display.png`、`ST-CHAT-003_suggestion_edit.png`、`ST-CHAT-004_blank_validation.png`、`ST-CHAT-025_*` の各証跡が `チャット実行テスト.md` の該当ケース証跡欄および state の証跡一覧に記録されていることを確認した。
- [x] スクリーンショットやログが、ロード中、エラー画面、古い状態ではなく、判定対象の状態を示しているか。
  - 検証結果: 指摘なし
  - 確認根拠: スクリーンショットは開始画面、入力候補編集、空白入力検証に対応している。ST-CHAT-025テキスト証跡は旧参照先欠落ではなく、backend script到達後のJSONL返却を示している。
- [x] 必要最小限の証跡になっているか。
  - 検証結果: 指摘なし
  - 確認根拠: state の `機能別総合テスト証跡` は8件のみで、削除済み `evidence/playwright-cli/` が除外されている。総合テスト方針が保存しない Playwright 詳細ログを現行証跡として扱っていないことを確認した。
- [x] 画面確認、HTTP確認、DB確認、ファイル確認の記録が、ケースの判定に必要な粒度で残っているか。
  - 検証結果: 指摘なし
  - 確認根拠: 画面確認は3つのスクリーンショット、API/DB/Codex境界は `F005-system-test-api-db-codex-boundary.txt`、Docker実行はST-CHAT-025の3証跡として残っている。DB/API詳細は総合テスト仕様の部分確認備考と対応している。
- [x] 未実施、一部確認、保留、不合格、再テストが曖昧に合格扱いされていないか。
  - 検証結果: 指摘なし
  - 確認根拠: `F005-system-test-summary.txt` と `チャット実行テスト.md` は ST-CHAT-025 を `環境・承認待ち` とし、不合格0、部分確認16、後続機能待ち5、対象外79を記録している。正常応答未確認は正式総合テスト持ち越しで、合格扱いされていないことを確認した。
- [x] 証跡だけを見て、どのケースが合格、不合格、部分確認、未実施、保留なのか誤読なく分かるか。
  - 検証結果: 指摘なし
  - 確認根拠: summary に分類別件数と `environment_pending=ST-CHAT-025 requires refreshed Codex authentication...` があり、各テスト仕様の実施結果欄にも部分確認、後続機能待ち、環境・承認待ち、対象外が記録されている。

## 実行結果の信頼性

- [x] 証跡の実行日時が対象成果物の変更後になっているか。
  - 検証結果: 指摘なし
  - 確認根拠: 管理役提示の差分情報では ST-CHAT-025 証跡が round-1 修正後に更新され、証跡内 `executed_at=2026-06-23T05:08:53+09:00` は修正後再実施として state に記録されている。
- [x] 実行コマンドが方針書、設定、package manager、依存管理の現行定義と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: ST-CHAT-025は `docker run --rm codex-python-runner:latest codex --version` と `infra/codex_docker/scripts/run_codex_docker.sh` を用いた証跡として記録され、summaryでも backend script 到達、JSONL返却、refresh token失効が明示されていることを確認した。
- [x] 証跡と実際のテストコード、テスト仕様、設定ファイルに矛盾がないか。
  - 検証結果: 指摘なし
  - 確認根拠: state の証跡一覧、summary、`チャット実行テスト.md` の証跡欄はいずれも8件の必要証跡に整理されている。削除済み `evidence/playwright-cli/` を証跡として扱う記録は現行の state 証跡一覧から除外されている。
- [x] 同じ対象に対する古い証跡が残り、最新結果と誤認される状態になっていないか。
  - 検証結果: 指摘なし
  - 確認根拠: 旧Playwright詳細ログ実体は前回時点で削除済みで、今回 state の証跡一覧からも `evidence/playwright-cli/` が除外されている。旧 `exit_code=127` のST-CHAT-025結果は最新証跡としては扱われていない。

## 修正方針の判断

- [x] 証跡が不足している場合は、証跡の追加、再実行、テスト仕様更新のどれが必要かを書く。
  - 検証結果: 指摘なし
  - 確認根拠: 証跡実体の不足は確認されず、ST-CHAT-025正常応答未確認はCodex認証更新後の正式総合テスト持ち越しとして既に記録されている。追加証跡や再実行を求める新規指摘はない。
- [x] 証跡が方針と矛盾する場合は、証跡を直すべきか、方針を直すべきかを根拠から判断する。
  - 検証結果: 指摘なし
  - 確認根拠: 方針は詳細ログを保存しないと定義し、state の証跡一覧も必要最小限の8件に整理済みであるため、方針または証跡を追加修正すべき矛盾は確認されなかった。
- [x] 証跡だけでは判定不能な場合は、必要な再実行条件と対象成果物を具体的に書く。
  - 検証結果: 指摘なし
  - 確認根拠: ST-CHAT-025の正常応答未確認は `refresh token` 失効が原因として証跡に記録され、Codex認証更新後に正式総合テストで再確認する条件が summary、state、テスト仕様に記録されている。判断不能として追加すべき不足根拠はない。
