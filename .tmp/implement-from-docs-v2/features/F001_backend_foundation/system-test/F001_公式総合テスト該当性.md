# F001 公式総合テスト該当性

## 1. 直接該当する公式ケース

F001単独で公式手順どおりに完了できるケースはない。公式総合テストはWeb画面から業務を完了する前提であり、認証、チャット、履歴、削除、アカウント管理の業務実装が必要である。

## 2. 部分確認した公式ケース

| 仕様書 | ケースID | F001で確認した範囲 | 残る確認 |
| --- | --- | --- | --- |
| 認証テスト.md | ST-AUTH-003 | Cookieなしの保護対象backend APIが `error` / `message` 形式、`x-trace-id` header付きで401を返し、内部情報を漏らさないこと | 未ログイン時の画面遷移、認証状態確認API、ログイン画面誘導 |
| チャット実行テスト.md | ST-CHAT-001 | 認証済み相当の `/api/app-config` が歓迎メッセージ、補足案内、入力候補だけを返し、内部設定を漏らさないこと | 開始画面表示、フロントエンド連携 |
| チャット実行テスト.md | ST-CHAT-002 | UI設定欠落時の `/api/app-config` が空の公開payloadで200を返すこと | 開始画面でウェルカムメッセージと入力候補を表示しないこと |
| 履歴再表示テスト.md | ST-HISTORY-001 | 履歴一覧を支えるDB索引 `user_id, chat_state, updated_at DESC` がmigrationで作成されること | 履歴一覧APIと画面で最終更新日時降順に表示されること |
| チャット削除テスト.md | ST-DELETE-013 | トレースログの保存、保持期間、同日上限の基盤が動作すること | チャット削除処理後も対象トレースログが保持されること |
