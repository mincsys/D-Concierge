# Evidence Review Checklist

## 証跡と方針の整合

- [x] 証跡の保存場所、ファイル名、記録項目が対応するテスト方針と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: 機能別総合テスト保存先の `evidence/F001-backend-smoke.txt` が提示され、F001 backend smoke の実行証跡として配置されている。
- [x] 証跡がどのテストケース、コマンド、範囲指定、実施日時に対応するか追跡できるか。
  - 検証結果: 指摘なし
  - 確認根拠: 生成役報告に smoke 実行コマンドと `28 passed in 1.24s` の結果が明記され、F001の部分確認5件と後続機能待ち96件の分類に対応している。
- [x] 証跡に対象コミット、対象差分、実行環境、主要バージョン、ブラウザ、画面サイズなど、再現に必要な前提が残っているか。
  - 検証結果: 指摘なし
  - 確認根拠: F001 smokeはbackend結合テストのpytestコマンドであり、ブラウザと画面サイズは不要。Playwright未実行理由はF001単独で業務画面が完結しないためと整理されている。
- [x] 再テスト時に古い証跡と新しい証跡の扱いが方針と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: 生成役報告ではF001用の作業用system-test配下に結果とevidenceを保存し、正本docsは変更していない。
- [x] 証跡に秘密情報、個人情報、絶対パス、不要な詳細ログが含まれていないか。
  - 検証結果: 指摘なし
  - 確認根拠: 報告された証跡内容はコマンドと集約結果であり、秘密情報や個人情報を含むとは確認していない。

## カバレッジ証跡

- [x] 方針で指定された指標だけが記録されているか。
  - 検証結果: 対象外
  - 確認根拠: 機能別総合テストのF001 smoke証跡であり、coverage証跡レビューではない。
  - 理由: coverageは前段の結合品質レビューで確認済みのため。
- [x] 対象、実行日時、実行コマンド、カバレッジ値、総数、通過数、判定結果がそろっているか。
  - 検証結果: 対象外
  - 確認根拠: F001機能別総合テストの証跡対象はsmoke実行結果で、分岐coverage証跡ではない。
  - 理由: 機能別総合テスト証跡レビューのため。
- [x] 目標未達を合格扱いしていないか。
  - 検証結果: 対象外
  - 確認根拠: coverage目標は結合品質レビューround-3で合格済み。
  - 理由: 今回の証跡は総合テストsmoke結果であるため。
- [x] 測定から外した範囲がテスト方針と矛盾していないか。
  - 検証結果: 対象外
  - 確認根拠: coverage測定除外設定のレビューではない。
  - 理由: 機能別総合テスト証跡の確認に限定されるため。

## 総合テスト証跡

- [x] スクリーンショットやテキスト証跡がテストケースIDと対応しているか。
  - 検証結果: 指摘なし
  - 確認根拠: F001では画面が完結しないためスクリーンショットは作成せず、backend smokeのテキスト証跡をF001部分確認に対応させている。
- [x] スクリーンショットやログが、ロード中、エラー画面、古い状態ではなく、判定対象の状態を示しているか。
  - 検証結果: 指摘なし
  - 確認根拠: 画面証跡は対象外で、backend smoke証跡は `28 passed` の完了状態を示すと報告されている。
- [x] 必要最小限の証跡になっているか。
  - 検証結果: 指摘なし
  - 確認根拠: F001単独で検証可能なbackend smokeに絞ったテキスト証跡で、未完結UIのPlaywright証跡を無理に作っていない。
- [x] 画面確認、HTTP確認、DB確認、ファイル確認の記録が、ケースの判定に必要な粒度で残っているか。
  - 検証結果: 指摘なし
  - 確認根拠: app-config API、trace/error boundary、REST error boundary、database migrations を含む smokeコマンドで、F001のHTTP/DB/ログ境界の部分確認を示している。
- [x] 未実施、一部確認、保留、不合格、再テストが曖昧に合格扱いされていないか。
  - 検証結果: 指摘なし
  - 確認根拠: 分類は合格0、不合格0、部分確認5、後続機能待ち96、環境・承認待ち0、対象外0で、未完結ケースを合格扱いしていない。
- [x] 証跡だけを見て、どのケースが合格、不合格、部分確認、未実施、保留なのか誤読なく分かるか。
  - 検証結果: 指摘なし
  - 確認根拠: `F001_機能別総合テスト結果.md` と `F001_公式総合テスト該当性.md` に分類別件数と持ち越し理由を記録できる粒度で整理されたとの生成役報告がある。

## 実行結果の信頼性

- [x] 証跡の実行日時が対象成果物の変更後になっているか。
  - 検証結果: 指摘なし
  - 確認根拠: 生成役報告ではF001機能別総合テスト実施後に smoke evidence を保存している。検証役はコマンド実行禁止のため再実行はしていない。
- [x] 実行コマンドが方針書、設定、package manager、依存管理の現行定義と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: smokeコマンドは `env UV_CACHE_DIR=/tmp/uv-cache uv run pytest ... -q` で、AGENTS.mdのuv/UV_CACHE_DIR指定と整合する。
- [x] 証跡と実際のテストコード、テスト仕様、設定ファイルに矛盾がないか。
  - 検証結果: 指摘なし
  - 確認根拠: smoke対象の4結合テストファイルはF001のapp-config、trace/error、REST error、database migration範囲に対応し、F001機能概要と矛盾しない。
- [x] 同じ対象に対する古い証跡が残り、最新結果と誤認される状態になっていないか。
  - 検証結果: 指摘なし
  - 確認根拠: F001用のsystem-test/evidence配下に `F001-backend-smoke.txt` が提示されており、同名の旧証跡が残るとの報告はない。

## 修正方針の判断

- [x] 証跡が不足している場合は、証跡の追加、再実行、テスト仕様更新のどれが必要かを書く。
  - 検証結果: 指摘なし
  - 確認根拠: F001単独で実施可能なbackend smoke証跡はあり、追加証跡不足は確認していない。
- [x] 証跡が方針と矛盾する場合は、証跡を直すべきか、方針を直すべきかを根拠から判断する。
  - 検証結果: 指摘なし
  - 確認根拠: F001単独で画面確認を後続機能待ちにし、backend smokeを部分確認にする方針と証跡に矛盾はない。
- [x] 証跡だけでは判定不能な場合は、必要な再実行条件と対象成果物を具体的に書く。
  - 検証結果: 指摘なし
  - 確認根拠: 生成役報告、分類件数、smoke evidence、docs正本非変更報告により機能別総合テストの完了可否を判定できる。検証役はコマンド実行禁止のため再実行はしていない。
