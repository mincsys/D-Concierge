## 総合テスト証跡

- [x] スクリーンショットやテキスト証跡がテストケースIDと対応しているか。
  - 検証結果: 指摘なし
  - 確認根拠: スクリーンショットは F002 では未取得と `認証テスト.md`、`アカウント管理テスト.md` の実施結果サマリに記録されている。テキスト証跡 `F002-auth-account-api-db.txt` は `ST-AUTH-*`、`ST-ACCOUNT-*` のケース ID ごとに結果を記録しており対応を追える。
- [x] スクリーンショットやログが、ロード中、エラー画面、古い状態ではなく、判定対象の状態を示しているか。
  - 検証結果: 指摘なし
  - 確認根拠: 画面スクリーンショットは未取得で、判定に使われた証跡は HTTP ステータス、Cookie、DB 件数、user_state、chat_state、field_errors のテキスト記録である。`result=pass` とケース別状態値があり、ロード中や古い画面状態の証跡ではないことを確認した。
- [x] 必要最小限の証跡になっているか。
  - 検証結果: 指摘なし
  - 確認根拠: `F002-auth-account-api-db.txt` は F002 API/DB 判定に必要なケース別要約に限定され、詳細実行ログ、動画、Playwright trace、不要な DB ダンプは含まれていない。`F002-system-test-summary.txt` も分類と持ち越し要約に絞られている。
- [x] 画面確認、HTTP確認、DB確認、ファイル確認の記録が、ケースの判定に必要な粒度で残っているか。
  - 検証結果: 指摘なし
  - 確認根拠: 画面確認は正式総合テストへ持ち越しと明示され、F002 で合格判定されたケースは HTTP ステータス、Cookie 発行/失効、users/login_sessions 件数、user_state、chat_state、field_errors、password_hash 変更有無などの DB/HTTP 粒度が記録されている。F007 の物理削除やファイル確認は後続機能待ちに分類されている。
- [x] 未実施、一部確認、保留、不合格、再テストが曖昧に合格扱いされていないか。
  - 検証結果: 指摘なし
  - 確認根拠: `F002-system-test-summary.txt` は `classification_pass=16`、`classification_partial=16`、`classification_waiting_later_feature=69`、`classification_failed=0`、`classification_environment_pending=0` を分けて記録している。部分確認と後続機能待ちは各仕様ファイルでも合格とは別分類で記録されている。
- [x] 証跡だけを見て、どのケースが合格、不合格、部分確認、未実施、保留なのか誤読なく分かるか。
  - 検証結果: 指摘なし
  - 確認根拠: `F002-system-test-summary.txt` に carry_over_auth、carry_over_account、carry_over_non_f002 があり、`認証テスト.md` と `アカウント管理テスト.md` のケース表にも実施結果と備考がケース単位で記録されている。不合格と環境・承認待ちは 0 と明示されている。

  - 確認根拠: `F002-auth-account-api-db.txt` と `F002-system-test-summary.txt` はいずれも `executed_at=2026-06-21T22:40:00+09:00` を記録し、state には機能別総合テスト実行結果として同証跡が記録されている。今回のレビューでは再実行は禁止のため、提出証跡と state の整合を確認した。

- [x] 証跡と実際のテストコード、テスト仕様、設定ファイルに矛盾がないか。
  - 検証結果: 指摘なし

- [x] 同じ対象に対する古い証跡が残り、最新結果と誤認される状態になっていないか。

## 修正方針の判断

- [x] 証跡が不足している場合は、証跡の追加、再実行、テスト仕様更新のどれが必要かを書く。
  - 検証結果: 指摘なし
  - 確認根拠: F002 機能別総合テストで合格判定されたケースは証跡から追跡でき、未画面確認や後続機能依存は正式総合テスト・後続機能へ持ち越し済みである。追加証跡や再実行を要する不足は見つからなかった。
- [x] 証跡が方針と矛盾する場合は、証跡を直すべきか、方針を直すべきかを根拠から判断する。
  - 検証結果: 指摘なし
  - 確認根拠: 証跡は `.tmp` 側の機能別レビュー用コピーに保存され、docs 正本は変更されていない。`git diff -- docs/04_テスト/04_総合テスト` が空であり、方針と証跡の矛盾は確認されなかった。
- [x] 証跡だけでは判定不能な場合は、必要な再実行条件と対象成果物を具体的に書く。
  - 検証結果: 指摘なし
  - 確認根拠: 判定不能のケースは確認されず、画面操作や後続機能は判定不能ではなく `部分確認` または `後続機能待ち` として明示分類されている。F002 機能別総合テストの完了可否判断に必要な証跡は揃っている。
