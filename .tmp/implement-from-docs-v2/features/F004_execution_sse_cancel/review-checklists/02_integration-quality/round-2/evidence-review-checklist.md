# Evidence Review Checklist

## 証跡と方針の整合

- [x] 証跡の保存場所、ファイル名、記録項目が対応するテスト方針と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: 単体 coverage は `docs/04_テスト/02_単体テスト/evidence/backend-unit-coverage.txt`、結合 evidence は `docs/04_テスト/03_結合テスト/evidence/backend-integration-evidence.txt` に保存されている。
- [x] 証跡がどのテストケース、コマンド、範囲指定、実施日時に対応するか追跡できるか。
  - 検証結果: 指摘なし
  - 確認根拠: それぞれ `executed_at`、`command`、`target`、`tests`/`failures` または coverage 数値が記録され、対象範囲を追跡できる。
- [x] 証跡に対象コミット、対象差分、実行環境、主要バージョン、ブラウザ、画面サイズなど、再現に必要な前提が残っているか。
  - 検証結果: 指摘なし
  - 確認根拠: backend の単体・結合 coverage 証跡として必要な対象、コマンド、日時、結果が残っている。ブラウザ・画面サイズは今回の backend 結合レビューでは対象外。
- [x] 再テスト時に古い証跡と新しい証跡の扱いが方針と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: evidence は round-2 修正後の `2026-06-22T06:17:31+09:00` の値へ更新され、古い F004 round-1 値と混在していない。
- [x] 証跡に秘密情報、個人情報、絶対パス、不要な詳細ログが含まれていないか。
  - 検証結果: 指摘なし
  - 確認根拠: evidence は key=value 形式のコマンド、coverage、件数、result のみで、秘密情報や個人情報は含まれていない。

## カバレッジ証跡

- [x] 方針で指定された指標だけが記録されているか。
  - 検証結果: 指摘なし
  - 確認根拠: 単体・結合とも branch coverage、covered/total branches、result を中心に記録され、不要な詳細ログは保存されていない。
- [x] 対象、実行日時、実行コマンド、カバレッジ値、総数、通過数、判定結果がそろっているか。
  - 検証結果: 指摘なし
  - 確認根拠: 単体 evidence は target/executed_at/command/branch_coverage/covered_branches/total_branches/result、結合 evidence は加えて design_coverage/tests/failures を含む。
- [x] 目標未達を合格扱いしていないか。
  - 検証結果: 指摘なし
  - 確認根拠: 単体 95.88% は単体方針の 95% 以上、結合 80.92% は結合方針の 80% 以上を満たしている。
- [x] 測定から外した範囲がテスト方針と矛盾していないか。
  - 検証結果: 指摘なし
  - 確認根拠: command は `--cov=src/backend --cov-branch` で backend 全体を測定しており、F004 だけを都合よく切り出していない。

## 総合テスト証跡

- [x] スクリーンショットやテキスト証跡がテストケースIDと対応しているか。
  - 検証結果: 対象外
  - 確認根拠: 現在フェーズは結合・品質レビューであり、総合テストスクリーンショットやケース ID 証跡は対象外。
- [x] スクリーンショットやログが、ロード中、エラー画面、古い状態ではなく、判定対象の状態を示しているか。
  - 検証結果: 対象外
  - 確認根拠: backend 単体・結合 coverage/evidence の確認であり、画面証跡は扱っていない。
- [x] 必要最小限の証跡になっているか。
  - 検証結果: 対象外
  - 確認根拠: この小項目は総合テスト証跡向けで、今回の key=value coverage 証跡とは対象が異なる。
- [x] 画面確認、HTTP確認、DB確認、ファイル確認の記録が、ケースの判定に必要な粒度で残っているか。
  - 検証結果: 対象外
  - 確認根拠: F004 の機能別総合テスト証跡レビューではなく、結合テスト evidence の確認である。
- [x] 未実施、一部確認、保留、不合格、再テストが曖昧に合格扱いされていないか。
  - 検証結果: 対象外
  - 確認根拠: 総合テスト分類の確認は今回対象外で、結合 evidence は tests=83/failures=0/result=pass を記録している。
- [x] 証跡だけを見て、どのケースが合格、不合格、部分確認、未実施、保留なのか誤読なく分かるか。
  - 検証結果: 対象外
  - 確認根拠: 総合テストケース別分類の証跡ではないため対象外。

## 実行結果の信頼性

- [x] 証跡の実行日時が対象成果物の変更後になっているか。
  - 検証結果: 指摘なし
  - 確認根拠: state の round-2 修正報告後の evidence として `2026-06-22T06:17:31+09:00` が記録され、修正対象の coverage 値も更新されている。
- [x] 実行コマンドが方針書、設定、package manager、依存管理の現行定義と一致しているか。
  - 検証結果: 指摘なし
  - 確認根拠: `env UV_CACHE_DIR=/tmp/uv-cache uv run pytest ... --cov=src/backend --cov-branch` を使用し、AGENTS.md の uv/cache 指示とテスト方針の coverage 測定に沿っている。
- [x] 証跡と実際のテストコード、テスト仕様、設定ファイルに矛盾がないか。
  - 検証結果: 指摘なし
  - 確認根拠: design_coverage は F004 の SSE live event delivery、runner completed cancel、startup recovery by unfinished state、startup recovery trace log を含み、追加テスト本文と対応している。
- [x] 同じ対象に対する古い証跡が残り、最新結果と誤認される状態になっていないか。
  - 検証結果: 指摘なし
  - 確認根拠: evidence ファイルは単一の最新 key=value セットに更新されており、古いテスト件数や coverage が併記されていない。

## 修正方針の判断

- [x] 証跡が不足している場合は、証跡の追加、再実行、テスト仕様更新のどれが必要かを書く。
  - 検証結果: 対象外
  - 確認根拠: 証跡不足は確認されず、追加再実行や証跡修正の指摘は不要。
- [x] 証跡が方針と矛盾する場合は、証跡を直すべきか、方針を直すべきかを根拠から判断する。
  - 検証結果: 対象外
  - 確認根拠: 証跡と方針の矛盾は確認されなかった。
- [x] 証跡だけでは判定不能な場合は、必要な再実行条件と対象成果物を具体的に書く。
  - 検証結果: 対象外
  - 確認根拠: 生成役報告、state、evidence、実装・テスト本文の照合で判定可能であり、追加実行は不要。
