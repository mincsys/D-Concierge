# 結合テスト方針レビュー観点

## 文書範囲

- 成果物が `docs/04_テスト/03_結合テスト/結合テスト方針.md` の 1 ファイルに限定されているか。
- 個別テストケース一覧、テスト結果一覧、静的テスト方針、単体テスト方針、総合テスト方針が混在していないか。
- 日本のウォーターフォール開発向けの正式文書として読めるか。

## 質問合意

- `【質問】` に相当する未確定項目が本文に残っていないか。
- 質問が 1 つずつ行われ、合意内容が整理されているか。
- 選択肢に `など`、`等`、`その他いろいろ` のような曖昧な表現が使われていないか。
- 選択肢が `1. 選択肢名`、`2. 選択肢名`、`3. 選択肢名` のような番号付きリストになっているか。
- ユーザが番号だけで回答できることを質問文で明記しているか。
- 選択肢番号が質問ごとに `1` から始まっているか。
- 推奨選択肢が `1. 選択肢名（推奨）` のように 1 つだけ明示されているか。
- 推奨理由が、選択肢本文または質問説明の中で簡潔に説明されているか。
- 複数の選択肢が同時に推奨扱いになっていないか。
- 本文作成直前に、全質問項目が `合意済み`、`未合意`、`リポジトリ事実として確定` のいずれかで確認されているか。
- 本文作成直前に、質問表の全質問項目と合意結果がユーザへ表示されているか。
- 合意結果一覧に `質問項目`、`合意結果`、`状態`、`根拠` が含まれているか。
- `根拠` に、合意結果を本文へ反映してよいと判断した理由が書かれているか。
- `合意済み` の根拠に、ユーザがどの回答で合意したかが書かれているか。
- `リポジトリ事実として確定` の根拠に、どのファイルや設定から一意に判断したかが書かれているか。
- `未合意` の根拠に、何が未回答または未確定かが書かれているか。
- 合意結果一覧の表示後、ユーザから明確な作成指示を受けてから本文作成へ進んでいるか。
- `確認しました`、`OKです`、`問題ありません` だけで本文作成へ進んでいないか。
- 明確な作成指示がない場合に、合意結果一覧の確認後も本文作成を開始せず待機しているか。
- 未合意項目が本文へ持ち込まれていないか。

## 処理設計書との対応

- `docs/03_内部設計/04_処理設計/` がある場合、処理設計書を読み取っているか。
- 各処理設計書の `事前条件`、`事後条件`、`不変条件` が結合テスト観点へ反映されているか。
- 処理設計書がない場合、ユーザ合意済みの基準資料に基づいて観点を定義しているか。

## 観点網羅

- 層間連携、IF契約、DB永続化、トランザクション、認証認可、エラー伝播、ログ、リトライ、タイムアウトが含まれているか。
- API、画面、バッチ、外部IF、データ結合のうち、対象システムに必要な観点が含まれているか。
- 実外部サービス、本番相当データ、性能負荷、ブラウザ実機差分、運用手順確認が、結合テスト方針の確認対象として混在していないか。

## 網羅基準

- 機能一覧、API一覧、画面一覧、処理一覧、外部IF一覧、CRUD種別、権限種別のうち、主基準が明確か。
- コードカバレッジが主基準ではなく補助指標として扱われているか。
- 網羅基準未達時の対応と例外扱いが明確か。

## データ・環境・コマンド

- seed、fixture、factory、DB初期化、外部応答fake、時刻固定、認証状態の扱いが明確か。
- ローカル環境、共通開発環境、CI、DB、外部サービス代替、コンテナ利用のうち、自動実行可能な環境が定義されているか。
- テスト実行コマンドが実行可能な形で書かれているか。
- 失敗時調査、再実行、データ初期化手順が明確か。

## テストコード方針

- 結合テストはテストコードを書いて実行する方針になっているか。
- すべての結合テストコードに `観点` と `確認` の docstring 形式を必須記載する方針になっているか。
- 手動確認を前提にした観点が混在していないか。
- 自動テストコード、実行コマンド、テストデータ初期化、CIまたはローカル自動実行が定義されているか。

## 成果物・証跡方針

- 文書成果物が `結合テスト方針.md` のみに固定されているか。
- テストコード、実行ログ、JUnit XML、カバレッジ、DB初期化ログ、CI結果の扱いが明確か。
- 証跡の保存場所、命名、保存期間、再テスト時の更新方針が明確か。

## 表現

- 文書が断定形で書かれているか。
- 差分説明、検討メモ、会話ログ、旧仕様の痕跡が残っていないか。
- `【template-guidance】` ブロック、`【質問】`、未置換プレースホルダが最終成果物に残っていないか。
- 既存 docs と用語、文体、リンク表記がそろっているか。
