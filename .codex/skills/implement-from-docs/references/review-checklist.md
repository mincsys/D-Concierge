# Review Checklist

完了前に次を確認する。

## docs 読解

- `AGENTS.md` を確認した。
- `docs/01_要件定義` から対象範囲と対象外範囲を確認した。
- `docs/02_外部設計` から IF、画面、データ、メッセージ、ログを確認した。
- `docs/03_内部設計` から責務分割、依存方向、クラス契約、処理契約、物理データを確認した。
- `docs/04_テスト` から各テストの対象、対象外、カバレッジ、証跡を確認した。
- `docs/05_開発標準` から実装手順、起動方法、規約を確認した。
- `.issue/` の作業対象影響を確認した。

## TDD と静的テスト

- Red、Green、Refactor の順で進めた。
- `.tmp/implement-from-docs/contract-matrix/unit/` と `.tmp/implement-from-docs/contract-matrix/integration/` 配下に今回対象の契約マトリクスを作成した。
- 単体テスト用と結合テスト用の契約マトリクスの全行が `単体テスト済み`、`結合テスト済み`、`対象外理由あり`、`未実装として.issue化` のいずれかになっている。
- 単体テストまたは結合テストのどちらかを今回対象外にした場合は、対象外理由を契約マトリクスまたは最終報告に明記した。
- Red はクラス設計の `事前条件`、`事後条件`、`不変条件` を含んでいる。
- Refactor ごとに対象範囲の静的テストを実行した。
- 静的テストの warning/error を残していない。

## 結合契約テスト

- 結合テストは処理設計書の `事前条件`、`事後条件`、`不変条件` を含んでいる。
- 外部 IF、DB 状態、認証認可、エラー変換、外部連携モック、ログ、副作用抑止を確認した。
- 失敗時の部分保存や後続副作用が残らないことを確認した。

## カバレッジと証跡

- カバレッジ算出前に、単体・結合テスト方針の対象外範囲を読んだ。
- 除外対象が測定対象へ混入しないように設計した。
- 単体カバレッジ目標を満たした。
- 結合カバレッジ目標を満たした。
- カバレッジ達成を契約網羅の代替にしていない。
- 未達例外がある場合は、対象、理由、代替確認を証跡に残した。
- 方針書どおりに最新の evidence を更新した。

## mock 分離

- `mock/` が存在する場合、デザイン実装部分を完全コピー対象にした。
- mock 固有のデータ、状態、遷移制御、fixture、業務ロジック、構成を持ち込んでいない。
- 正式実装のディレクトリと責務分割は docs に準拠している。
- 総合テスト段階でスクリーンショットと画面仕様書の画面イメージを目視確認する準備ができている。

## issue と完了報告

- 新しい仕様・設計・実装乖離は `.issue/` に記録した。
- 解消済み issue は削除または更新した。
- ユーザが確定した対象範囲に未処理契約が残る場合、最終報告で完了と書かない。
- 実装済み範囲、今回対象外、未解消 issue、未反映の任意成果物、実行できなかったテストを分けて報告した。
- 実行できなかったテストや保留事項を最終報告に含めた。
- sudo が必要な未実施操作は、ユーザ実行コマンドとして提示した。
