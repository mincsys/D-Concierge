# 総合テスト方針

## 1. 文書の目的
【template-guidance】<br>
- 本書が総合テストの方針、対象、実施ルール、成果物・証跡、完了条件を定義する文書であることを書く。
- 個別のテストケースとケースごとの実施結果は `テスト仕様・結果/〇〇テスト.md` に記載することが分かるようにする。

【/template-guidance】<br>

本書は、対象システムの総合テストにおける対象範囲、総合テスト分割単位、テスト観点、Playwright 利用方針、手動テスト実施方針、実施環境、成果物・証跡および完了条件を定義することを目的とする。

## 2. 前提
【template-guidance】<br>
- 参照する要件定義書、外部設計書、内部設計書、静的テスト方針、単体テスト方針、結合テスト方針、開発標準、ユーザ合意事項を書く。
- 総合テストは手動テストを基本とし、Playwright が利用できる環境では Playwright で検査できる範囲をすべて Playwright で実施することを書く。
- 実装手順、詳細な環境構築手順、テストデータ方針は書かない。

【/template-guidance】<br>

- 本書は、要件定義書、外部設計書、内部設計書、静的テスト方針、単体テスト方針、結合テスト方針および開発標準を前提とする。
- 本書の対象範囲は、ユーザと合意した総合テスト対象に限定する。
- 総合テストは手動テストを基本とする。
- Playwright が利用できる環境では、Playwright で検査できる範囲をすべて Playwright で実施する。
- 個別のテストケースおよびケースごとの実施結果は、`テスト仕様・結果/〇〇テスト.md` に記載する。

## 3. 総合テスト対象範囲
【template-guidance】<br>
- 総合テストで確認するシステム、サブシステム、機能、画面、API、バッチ、外部IFを書く。
- 静的テスト、単体テスト、結合テストとの境界が分かるようにする。
- 対象外範囲の独立章は作らない。必要な境界説明はこの章または該当章の補足に収める。

【/template-guidance】<br>

| 対象区分 | 対象範囲 | 確認目的 |
| --- | --- | --- |
| ユーザと合意した対象区分 | ユーザと合意した対象範囲 | ユーザと合意した確認目的 |

## 4. 総合テスト分割単位
【template-guidance】<br>
- `テスト仕様・結果/〇〇テスト.md` を何単位で作成するかを書く。
- 画面単位、業務シナリオ単位、機能単位、ユーザロール単位、外部IF単位のうち、ユーザ合意に基づく単位を書く。
- 作成するテスト仕様・結果ファイルの一覧を具体名で書く。

【/template-guidance】<br>

| 分割単位 | 作成するテスト仕様・結果ファイル | 対象 |
| --- | --- | --- |
| ユーザと合意した分割単位 | `テスト仕様・結果/〇〇テスト.md` | ユーザと合意した対象 |

## 5. 総合テスト観点
【template-guidance】<br>
- 総合テストで必ず確認する共通観点を書く。
- 詳細化が必要な場合は `references/test-viewpoints.md` を参照する。
- テストケースの具体手順と結果は、この章ではなく `テスト仕様・結果/〇〇テスト.md` に書く。

【/template-guidance】<br>

総合テストでは、少なくとも以下の観点を確認する。

| 観点 | 確認内容 |
| --- | --- |
| 業務シナリオ | 利用者の業務目的に沿った一連の操作が設計どおりに完了すること |
| 画面遷移 | 画面間の遷移、戻る操作、再表示が設計どおりであること |
| 入力検証 | 必須、形式、範囲、相関、エラー表示が設計どおりであること |
| 権限 | ロール、所有者、認証状態ごとの利用可否が設計どおりであること |
| 登録・更新・削除 | 業務データの登録、更新、削除、参照結果が設計どおりであること |
| 外部IF | 外部サービス、外部ファイル、通知、連携結果が設計どおりであること |
| 異常系 | 入力誤り、通信失敗、権限不足、競合、処理失敗時の表示と後続操作が設計どおりであること |
| メッセージ表示 | 成功、警告、エラー、確認メッセージが設計どおりに表示されること |
| ブラウザ表示 | 対象ブラウザで主要画面の表示、操作、レイアウトが利用可能であること |

## 6. Playwright 利用方針
【template-guidance】<br>
- Playwright が利用できる環境では、Playwright で検査できる範囲をすべて Playwright で実施する方針を書く。
- Playwright で検査する範囲、検査しない範囲、検査できない範囲を分けて書く。
- Playwright の詳細なテストコード実装手順は書かない。

【/template-guidance】<br>

| 区分 | 方針 |
| --- | --- |
| Playwright で検査する範囲 | Playwright で検査できる範囲はすべて Playwright で実施する |
| Playwright で検査しない範囲 | ユーザと合意した範囲 |
| Playwright で検査できない範囲 | ユーザと合意した範囲 |
| 証跡 | ユーザと合意したスクリーンショット、動画、レポートを保存する |

## 7. 手動テスト実施方針
【template-guidance】<br>
- Playwright で検査できない確認を、人がどのように操作、目視確認、証跡取得、結果記入するかを書く。
- 手動実施時の結果記録は `テスト仕様・結果/〇〇テスト.md` に記載する方針を書く。

【/template-guidance】<br>

| 項目 | 方針 |
| --- | --- |
| 操作 | ユーザと合意した操作方針 |
| 目視確認 | ユーザと合意した確認方針 |
| 証跡取得 | ユーザと合意した取得方針 |
| 結果記入 | `テスト仕様・結果/〇〇テスト.md` に実施結果を記入する |
| 不具合記録 | ユーザと合意した記録方針 |

- 詳細な環境構築手順やテストデータ方針は書かない。

【/template-guidance】<br>

| 項目 | 方針 |

| 外部サービス接続先 | ユーザと合意した接続先 |

## 9. 成果物・証跡方針
【template-guidance】<br>
- 文書成果物は `総合テスト方針.md` と、分割単位に従って作成する `テスト仕様・結果/〇〇テスト.md` とする。
- スクリーンショット、動画、Playwright レポート、不具合票、再テスト結果、保存場所、命名、保存期間、更新方針を書く。
- 文書成果物以外の証跡の扱いは必ずユーザに質問し、合意した内容だけを書く。

【/template-guidance】<br>

| 項目 | 方針 |
| --- | --- |
| 文書成果物 | `総合テスト方針.md`、`テスト仕様・結果/〇〇テスト.md` |
| スクリーンショット | ユーザと合意した扱い |
| 動画 | ユーザと合意した扱い |
| Playwright レポート | ユーザと合意した扱い |
| 不具合票 | ユーザと合意した扱い |
| 再テスト結果 | ユーザと合意した扱い |
| 証跡の保存場所 | ユーザと合意した保存場所 |
| 証跡の命名 | ユーザと合意した命名 |
| 証跡の保存期間 | ユーザと合意した保存期間 |
| 証跡の更新方針 | ユーザと合意した更新方針 |

## 10. 完了条件
【template-guidance】<br>
- 全ケース実施、合格、不具合残件、再テスト、未実施ケース、承認条件を書く。
- 何を満たしたら総合テスト完了とするかを断定形で書く。

【/template-guidance】<br>

- ユーザと合意した総合テスト対象範囲が確認されている。
- ユーザと合意した `テスト仕様・結果/〇〇テスト.md` が作成されている。
- ユーザと合意した全テストケースが実施されている。
- ユーザと合意した合格条件を満たしている。
- ユーザと合意した証跡が保存されている。
- 未実施ケース、不具合残件、再テスト未完了項目がある場合は、扱いと承認条件が明確になっている。
