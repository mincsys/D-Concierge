# 総合テスト観点整理

## 共通観点

- 業務シナリオ: 利用者の業務目的に沿った一連の操作が設計どおりに完了すること。
- 画面遷移: 画面間の遷移、戻る操作、再表示、直接URLアクセス時の扱いが設計どおりであること。
- 入力検証: 必須、形式、範囲、相関、エラー表示、入力保持が設計どおりであること。
- 権限: ロール、所有者、認証状態ごとの表示、操作、アクセス可否が設計どおりであること。
- 登録・更新・削除: 業務データの登録、更新、削除、参照結果が設計どおりであること。
- 外部IF: 外部サービス、外部ファイル、通知、連携結果が設計どおりであること。
- 異常系: 入力誤り、通信失敗、権限不足、競合、処理失敗時の表示と後続操作が設計どおりであること。
- メッセージ表示: 成功、警告、エラー、確認メッセージが設計どおりに表示されること。
- ブラウザ表示: 対象ブラウザで主要画面の表示、操作、レイアウトが利用可能であること。

## Playwright で検査しやすい観点

- 画面遷移、フォーム入力、ボタン操作、一覧表示、詳細表示、登録・更新・削除後の表示を確認する。
- 入力検証、エラーメッセージ、成功メッセージ、認証状態、権限別表示を確認する。
- 主要業務シナリオの正常系と代表的な異常系を確認する。
- スクリーンショット、動画、レポートを証跡として取得できる確認を優先する。

## 手動確認が残りやすい観点

- 人の判断が必要な見た目の自然さ、文言の妥当性、業務上の違和感を確認する。
- 実機固有の操作感、外部サービス側の画面、手元の通知受信、紙媒体や口頭確認を伴う業務を確認する。
- Playwright で安定的に再現できない環境依存の確認は、手動実施方針と証跡方針に基づいて扱う。

## 分割単位別の観点

- 画面単位: 初期表示、入力、操作、遷移、権限、メッセージ、主要データ反映を確認する。
- 業務シナリオ単位: 開始条件、操作順序、業務データ変化、完了状態、例外時の復帰を確認する。
- 機能単位: 機能の入口、主要操作、結果表示、関連画面や関連データへの反映を確認する。
- ユーザロール単位: ロールごとの表示、操作、アクセス制御、承認や閲覧範囲を確認する。
- 外部IF単位: 送信、受信、失敗、再実行、利用者への表示、証跡の残り方を確認する。
