# 総合テスト設計レビュー観点

## 文書範囲

- 成果物が `docs/04_テスト/04_総合テスト/総合テスト方針.md` と、合意済みの `docs/04_テスト/04_総合テスト/テスト仕様・結果/〇〇テスト.md` に限定されているか。
- 静的テスト方針、単体テスト方針、結合テスト方針が混在していないか。
- 日本のウォーターフォール開発向けの正式文書として読めるか。

## 質問合意

- `【質問】` に相当する未確定項目が本文に残っていないか。
- 質問が 1 つずつ行われ、合意内容が整理されているか。
- 選択肢に `など`、`等`、`その他いろいろ` のような曖昧な表現が使われていないか。
- 選択肢が `1. 選択肢名`、`2. 選択肢名`、`3. 選択肢名` のような番号付きリストになっているか。
- ユーザが番号だけで回答できることを質問文で明記しているか。
- 選択肢番号が質問ごとに `1` から始まっているか。
- 推奨選択肢が `1. 選択肢名（推奨）` のように 1 つだけ明示されているか。
- 推奨理由が、選択肢本文または質問説明の中で簡潔に説明されているか。
- 複数の選択肢が同時に推奨扱いになっていないか。
- 本文作成直前に、全質問項目が `合意済み`、`未合意`、`リポジトリ事実として確定` のいずれかで確認されているか。
- 本文作成直前に、質問表の全質問項目と合意結果がユーザへ表示されているか。
- 合意結果一覧に `質問項目`、`合意結果`、`状態`、`根拠` が含まれているか。
- `根拠` に、合意結果を本文へ反映してよいと判断した理由が書かれているか。
- 合意結果一覧の表示後、ユーザから明確な作成指示を受けてから本文作成へ進んでいるか。
- `確認しました`、`OKです`、`問題ありません` だけで本文作成へ進んでいないか。
- 未合意項目が本文へ持ち込まれていないか。

## 方針書章構成

- `総合テスト方針.md` が 1章から10章までの指定章構成になっているか。
- `総合テスト対象外範囲`、`テストデータ方針`、`テスト仕様・結果ファイルの記述方針` が独立章として作られていないか。
- 総合テスト分割単位と作成するテスト仕様・結果ファイルが対応しているか。

## テスト仕様・結果テンプレート

- `テスト仕様・結果/〇〇テスト.md` が汎用ひな型として使われ、成果物では具体名へ複製されているか。
- 成果物に未置換の `〇〇` が残っていないか。
- テストケース一覧に `ケースID`、`観点`、`前提条件`、`操作手順`、`期待結果`、`実施結果`、`証跡`、`備考` が含まれているか。
- ケースごとの実施結果、証跡、備考がテストケース一覧内で矛盾なく記録されているか。

## 観点網羅

- 業務シナリオ、画面遷移、入力検証、権限、登録・更新・削除、外部IF、異常系、メッセージ表示、ブラウザ表示が含まれているか。
- 対象システムに不要な観点は、無理に詳細化されていないか。
- テストケースの具体手順と結果が `テスト仕様・結果/〇〇テスト.md` に整理されているか。

## Playwright・手動実施

- Playwright が利用できる環境では、Playwright で検査できる範囲をすべて Playwright で実施する方針になっているか。
- Playwright で検査する範囲、検査しない範囲、検査できない範囲が明確か。
- Playwright で検査できない確認について、人による操作、目視確認、証跡取得、結果記入の方針が明確か。

## 環境・成果物・完了条件

- ローカル、検証環境、共通開発環境、Playwright 実行環境、利用ブラウザ、外部サービス接続先の扱いが明確か。
- `総合テスト方針.md`、`テスト仕様・結果/〇〇テスト.md`、スクリーンショット、動画、Playwright レポート、不具合票、再テスト結果の扱いが明確か。
- 証跡の保存場所、命名、保存期間、再テスト時の更新方針が明確か。
- 全ケース実施、合格、不具合残件、再テスト、未実施ケース、承認条件が明確か。

## 表現

- 文書が断定形で書かれているか。
- 差分説明、検討メモ、会話ログ、旧仕様の痕跡が残っていないか。
- `【template-guidance】` ブロック、`【質問】`、未置換プレースホルダが最終成果物に残っていないか。
- 既存 docs と用語、文体、リンク表記がそろっているか。
