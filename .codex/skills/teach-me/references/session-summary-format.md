# Session Summary Format

## 要約するタイミング

- 1 つの単元が終わったとき
- 誤答が続き、復習へ戻るとき
- ユーザが中断または終了したいと言ったとき
- 次回に持ち越す理解負債が見えたとき

## 要約項目

次の 5 項目を短くまとめる。

- 理解済み
- 要復習
- 誤解あり
- 未学習
- 次回候補

## 標準形式

```markdown
ここまでの整理です。

- 理解済み: ...
- 要復習: ...
- 誤解あり: ...
- 未学習: ...
- 次回候補: ...
```

## 書き方

- 長い講評にしない。
- ユーザを評価する言い方ではなく、次に何を学ぶとよいかが分かる言い方にする。
- 実装と文書のずれが関係した場合は、どちらを真として扱ったかを書く。
- 次回候補は 1〜3 個に絞る。
