# Final Report

最終報告は、管理役が検証役の正式総合テスト前横断レビュー、正式総合テスト結果分析レビュー、必要な修正ループ結果に基づいて作成する。

## 報告内容

- 完了した機能
- 機能結合完了済みだが、総合テスト後に完了扱いできない機能
- 完了不可または打ち切りになった機能
- TBC へ移動した issue
- 実行した静的テスト、単体テスト、結合テスト、機能別総合テスト、正式総合テスト
- 保留した機能別総合テスト、正式総合テストと理由
- 保存または更新した証跡
- `review-artifacts` checklist の処理サマリ（未処理、指摘あり、対象外、判断不能、根拠なし `- [x]` の有無）
- `.tmp` は最終成果物ではなく、削除はユーザ管理であること
- ユーザ実行が必要な sudo 操作や環境操作

## 書き方

- 検証役が完了不可と判定した対象を完了と表現しない。
- 完了した機能は、正式総合テスト前横断レビュー、正式総合テスト、総合テスト結果分析レビュー、必要な修正ループ後に完了と表現できるものだけに限定する。
- 機能結合完了済みでも、正式総合テストで問題が残った機能は TBC または未解決として報告する。
- High issue または通常 issue が TBC に残る場合は明示し、未解決または未完成として報告する。
- テスト未実行、部分確認、保留、TBC 管理妥当性を合格と混同しない。
- TBC 移動は管理上の整理であり、合格、解消、完成ではない。
- 未処理項目、根拠なし `- [x]`、未反映の判断不能が残る場合は、レビュー完了またはアプリ完成と表現しない。
- 指摘ありの checklist 項目が issue、TBC、修正済みのいずれにも反映されていない場合は、完了と表現しない。
- 正式総合テスト不合格、TBC 残、High issue 残、または通常 issue の TBC 残存のいずれかがある場合、アプリ完成と表現しない。
- 管理役は、検証役の判定、生成役の報告、state、タスクリスト、issue/TBC 状態を照合した結果だけを最終報告に反映し、技術的な合否判断を補完しない。
- 詳細な指摘本文をチャットへ貼らず、issue パスを示す。
- ステージング済みであることを必要に応じて報告する。

## 完了前確認

正式総合テスト前の横断レビュー、正式総合テスト、総合テスト結果分析レビュー、必要な修正ループの後に [manager-completion-checklist.md](manager-completion-checklist.md) を `.tmp/implement-from-docs-v2/manager-completion-checklist.md` へコピーして完了前確認を行う。

- docs、実装、テスト、証跡の整合
- TBC issue の妥当性
- ステージング境界
- review-artifacts checklist の保存先、未処理項目、指摘あり、対象外、判断不能、根拠なし `- [x]` の有無
- テスト方針書・設計書観点に対するテストコード検証結果
- 実装コード品質の横断的な問題
- 最終報告で完了と表現できる範囲
- TBC 残存時に未解決または未完成として報告されていること
