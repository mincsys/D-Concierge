# Manager Completion Checklist

管理役は、最終報告前に次を確認する。このチェックリストは成果物レビュー観点ではなく、v2 の進行手順、state、tasklist、issue/TBC、ステージング、正式総合テスト、検証役判定を照合するために使う。

## docs と対象範囲

- `AGENTS.md` を確認した。
- `docs/01_要件定義` から対象範囲と対象外範囲を確認した。
- `docs/02_外部設計` から IF、画面、データ、メッセージ、ログを確認した。
- `docs/03_内部設計` から責務分割、依存方向、クラス契約、処理契約、物理データを確認した。
- `docs/04_テスト` から各テストの対象、対象外、カバレッジ、証跡を確認した。
- `docs/05_開発標準` から実装手順、起動方法、規約を確認した。

## 役割分担

- 管理役が結合テストと総合テストを通しやすい検証可能な業務機能単位に分解した。
- 管理役が機能分解時に総合テスト方針と `テスト仕様・結果` を読み、各機能の対応総合テスト仕様、主対象ケース ID、後続依存ケース ID を整理した。
- 生成役と検証役を 1 回だけ起動し、使い回した。
- 生成役と検証役の初回起動、再利用再開、新規再起動時に `SKILL.md` の軽読を指示した。
- 生成役と検証役に管理用タスクリストを編集させていない。
- `subagent-protocol/` 配下のフェーズ別テンプレートで依頼した。
- 生成役と検証役の責務境界、禁止事項、報告形式が依頼に含まれている。
- `subagent-protocol/` 配下のテンプレート本文が `管理役`、`生成役`、`検証役` に依存していない。
- 依頼を受けたサブエージェントが、実施事項、禁止事項、報告内容を自分の作業として読める。
- `state.md`、タスクリスト、issue 削除、TBC 移動などの禁止事項が、内部役割名なしで理解できる。
- 管理役が生成作業を代行していない。
- 管理役が検証作業を代行していない。
- 管理役が技術的な合否判断を代行していない。
- 管理役が全ての生成役・検証役依頼について完了報告を待ってから次へ進んだ。
- 空の完了メッセージまたは実質的な完了報告なしの応答を、成功、失敗、TBC、レビュー完了として扱っていない。
- 空完了メッセージ発生時は、state にリミット到達による一時中断、対象役割、直前フェーズ、再開方針、引き継ぎ要約を記録した。
- リミット解除後は前回の同一サブエージェントの再利用を優先し、再利用できない場合だけ新規サブエージェントへ標準引き継ぎを渡した。
- 管理役が検証役の判定、生成役の報告、state、タスクリスト、issue/TBC 状態を照合し、手順上の完了条件を満たしていない対象を完了扱いしていない。
- 検証役の各 `review-artifacts` checklist に未処理項目と根拠なし `- [x]` が残っていない。
- 検証役の各 `review-artifacts` checklist の指摘あり、対象外、判断不能の件数が state と完了可否へ反映されている。
- 判断不能または未反映の指摘が残る対象を完了扱いしていない。
- 検証役の結合完了判定または全体完了判定に基づいて管理役がチェックを付けた。

## 機能別フロー

- 生成役が本実装前に、テスト方針書と設計書から単体・結合テストコードを作成した。
- 検証役がテストコードの網羅性、docstring、ディレクトリ構成、テスト品質を確認した。
- 検証役が再レビューでもゼロベースで同一フェーズ全体を確認した。
- 検証役がコマンドを実行していない。
- 生成役が実装、単体テスト、結合テスト、カバレッジ確認を行った。
- 検証役が結合テスト完了時にカバレッジ、テスト品質、実装コード品質を確認した。
- 生成役が正式総合テストと同じ流れ、同じ記録形式、同じ evidence 保存方針で機能別総合テストを行った。
- 生成役が対象ケースの `前提条件`、`操作手順`、`期待結果` どおりに機能別総合テストを行った。
- 生成役が API 確認、DB 確認、結合テスト、ログ確認などを総合テストの代替合格根拠にしていない。
- 仕様どおり実施できなかったケースは、未実施の手順番号、阻害要因、確認済み期待結果、未確認期待結果、補助証跡、再実施条件が記録されている。
- 機能別総合テストの仕様コピー、結果、evidence が `.tmp/implement-from-docs-v2/features/<機能>/system-test/` に保存されている。
- 機能別総合テストで `docs/04_テスト/04_総合テスト/` を変更していない。
- 検証役が機能別総合テスト結果、証跡、仕様どおりの実施可否、保留妥当性、`docs` 非変更を確認した。
- 検証役が仕様どおり実施できなかった項目を `修正可能 / 後続機能待ち / 外部認証待ち / 自動操作不可 / 環境制約 / 対象外` に分類した。
- 実装不足、設定不足、テストデータ不足、手順誤り、証跡不足は保留にせず issue 化されている。
- 検証役が妥当と判定した `後続機能待ち`、`外部認証待ち`、`自動操作不可`、`環境制約`、`対象外` だけが正式総合テストへの持ち越しとして state に記録されている。
- 主業務パスの中核ケースが未確認の場合、state に `中核未確認` が記録され、最終完了扱いしていない。
- 全機能が機能結合完了まで進んだ。

## 正式総合テスト

- 全機能の機能結合完了後に、各機能 state の `正式総合テストへの持ち越し` を全体 state へ集約した。
- 正式総合テスト前に検証役が横断レビューを実施した。
- 横断レビューで issue が作成された場合、正式総合テストへ進む前に生成役と検証役の修正ループを実施した。
- 全機能の機能結合完了後に、生成役が正式総合テストと証跡保存を行った。
- 生成役が既存の総合テスト方針と `テスト仕様・結果` に従って実行、結果記録、証跡保存を行った。
- 正式総合テスト結果と正式 evidence が `docs/04_テスト/04_総合テスト/` に保存されている。
- `.tmp` の機能別総合テスト結果を正式結果として転記していない。
- 検証役が正式総合テスト完了時に総合テスト結果分析、未ステージ内容、最終整合、実装コード品質を確認した。
- 正式総合テストの不合格・保留が残る場合、検証役が原因分析し、修正可能 issue、仕様不整合 issue、テスト不能項目、TBC 候補へ分類した。
- 正式総合テストの修正可能 issue がある場合、生成役が issue 修正、関連テスト、該当総合テスト再実行、証跡更新を行った。
- 正式総合テスト issue の再レビューとステージングを最大 3 回まで実施した。
- テスト不能項目が残る場合、理由、確認済み証跡、再実施条件を state と最終報告に記録した。
- 機能結合完了済みでも、正式総合テストで問題が残った機能を最終完了扱いしていない。

## issue とステージング

- 指摘は `.issue/implement-from-docs/` に保存した。
- 検証役が `解消済み` と判定し `削除可 issue` に列挙した issue だけを管理役が削除した。
- `未解消`、`判断不能`、`TBC候補`、`仕様書側修正` に分類された issue を削除していない。
- 打ち切り時の残 issue を `.issue/implement-from-docs/TBC/` へ移動した。
- 仕様書側修正 issue を修正後も TBC へ移動した。
- 仕様不整合 issue を修正後も TBC へ移動した。
- TBC 移動を合格、解消、完成として扱っていない。
- TBC 残存は重大度に関係なく未解決として state、タスクリスト、最終報告に反映した。
- 検証役の初回レビュー・各再レビュー完了後に v2 作業差分を毎回ステージングした。
- 既存のユーザ変更を勝手にステージングしていない。

## 最終確認

- 全機能の機能結合完了、正式総合テスト前の横断レビュー、正式総合テスト、総合テスト結果分析レビュー、必要な総合テスト issue 修正ループが完了している。
- 各機能の対応総合テスト仕様、主対象ケース ID、後続依存ケース ID、仕様どおり実施できなかったケース、中核未確認、正式総合テストへの持ち越しが state に反映されている。
- 全レビュー checklist に未処理項目と根拠なし `- [x]` が残っていない。
- 判断不能または未反映の指摘が残る対象を、完了またはアプリ完成と表現していない。
- 検証役が完了不可と判定した対象を完了と表現していない。
- 実行できなかったテストや保留事項を最終報告に含めた。
- 総合テスト不合格・保留が issue 分析済みであり、修正可能 issue の 3 回ループが完了している。
- 正式総合テスト不合格、TBC 残、High issue 残、または通常 issue の TBC 残存のいずれかがある場合、アプリ完成と表現していない。
- `.tmp` は最終成果物ではないが、削除はユーザが行うため、この skill の完了前確認に `.tmp` 削除確認を含めていない。
- sudo が必要な未実施操作は、ユーザ実行コマンドとして提示した。
