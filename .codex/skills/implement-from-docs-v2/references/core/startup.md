# Startup

管理役、生成役、検証役は、実作業の前にそれぞれ docs 読込フェーズを行う。

## 共通サブエージェント起動ルール

- サブエージェントの初回起動、再利用再開、新規再起動時は、依頼冒頭で `.codex/skills/implement-from-docs-v2/SKILL.md` を軽く読むよう指示する。
- `SKILL.md` は Overview、Workflow、Rules、Done Criteria、Resources を中心に読み、自分の役割、他エージェントの役割、依頼元との責務境界、TBC、総合テスト、ステージング境界、完了条件を理解させる。
- `SKILL.md` の軽読後に、役割別 bootstrap テンプレートと、依頼に必要な reference だけを読む。全 reference を毎回読ませない。
- 再利用再開または新規再起動では、[handoff.md](../../subagent-protocol/resume/handoff.md) に従い、前回の状態と未完了作業を渡す。

## 管理役

- `reading-order.md` の必読順序に従い、対象範囲、関連 docs、既存実装、既存テスト、既存 issue を読む。
- 実装対象が曖昧な場合だけ、ユーザへ 1 問ずつ確認する。
- 対象を機能単位へ分解する前に、テスト方針と開発標準も確認する。

## 生成役

- サブエージェントは `worker` として 1 回だけ起動し、以後の機能タスクで使い回す。
- 起動時にモデル、reasoning、service tier を指定しない。
- 初期指示では `SKILL.md` の軽読後に [generator.md](../../subagent-protocol/bootstrap/generator.md) を使い、docs 固定構成、TDD、単体・結合テスト先行、機能別総合テスト、正式総合テスト、mock 分離、issue 保存先を理解させる。
- 初期 docs 読込フェーズでは成果物を作成または修正しない。
- 生成役はテスト方針書と設計書から単体・結合テストコードを先行作成し、機能別には実装、結合テスト、`.tmp` 上の機能別総合テストまでを行う。全機能の機能結合完了後に正式総合テスト実行と証跡保存を行うが、管理用タスクリストを編集しない。

## 検証役

- サブエージェントは 1 回だけ起動し、以後の機能タスクで使い回す。
- 起動時にモデル、reasoning、service tier を指定しない。
- 初期指示では `SKILL.md` の軽読後に [verifier.md](../../subagent-protocol/bootstrap/verifier.md) を使い、`review-artifacts` checklist、v2 の指摘保存先、段階レビュー、実装コード品質レビューを理解させる。
- 初期 docs 読込フェーズでは検証を開始しない。
- 検証役は各レビュー時に `review-artifacts` checklist を作業用ディレクトリへコピーして確認し、レビュー結果と完了可否を返す。
- 検証役は管理用タスクリストを編集しない。

## 使い回しルール

- 生成役と検証役は、タスクごとに作り直さない。
- 追加依頼は `send_input` で同じサブエージェントへ送る。
- 生成役と検証役には、前の機能の文脈を保持しつつ、今回機能の状態ファイルと関連 docs を参照させる。
- サブエージェントの完了メッセージが空、または実質的な完了報告を含まない場合は、リミット到達による一時中断として扱い、完了、失敗、TBC、レビュー完了にはしない。
- 空完了メッセージを検知した場合、依頼元は state に中断理由、対象役割、直前フェーズ、再開時の引き継ぎ要約を記録し、ユーザからリミット解除の指示があるまで作業を止める。
- リミット解除後は、前回の同一サブエージェントへ `send_input` で再開依頼を送る。前回サブエージェントが利用不能、文脈喪失、または再利用できない場合だけ、同じ役割のサブエージェントを新規起動する。
- 新規起動する場合は、通常の bootstrap に加えて [handoff.md](../../subagent-protocol/resume/handoff.md) の内容を渡し、前回までの状態、未完了作業、禁止事項、完了報告形式を明確にする。
- サブエージェントが長時間不要になった場合だけ close する。
