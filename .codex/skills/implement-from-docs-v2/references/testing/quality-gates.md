# Quality Gates

結合テスト完了時レビューでは、検証役が `review-artifacts` の `implementation`、`test`、`evidence` checklist を使い、カバレッジ基準、テスト品質、テスト方針書・設計書の観点網羅を確認する。

## カバレッジ算出前に読むもの

- `docs/04_テスト/02_単体テスト/単体テスト方針.md`
- `docs/04_テスト/03_結合テスト/結合テスト方針.md`
- 静的テスト方針
- コーディング規約
- coverage や evidence の保存先を定義する文書

## 生成役が実行する品質ゲート

1. 検証役が承認したテストコードがあることを確認する。
2. 対象範囲の静的テストをすべて合格させる。
3. 単体テストをカバレッジ付きで実行する。
4. 結合テストをカバレッジ付きで実行する。
5. カバレッジ目標が未達の場合は、方針書の観点に従ってテストを追加する。
6. 追加後、同じコマンドで再算出する。
7. 証跡方針に従って最新の実行ログまたは coverage summary を保存する。

## 検証役が確認すること

- 検証役はコマンドを実行しない。
- 生成役の報告、実行ログ、coverage summary、証跡、差分を確認する。
- カバレッジ目標を満たしている。
- 除外設定が方針書の対象外範囲と一致している。
- 目標未達をごまかす除外が追加されていない。
- テストコードがテスト方針書・設計書の観点を実際に確認している。
- 単体、結合テストの役割が混ざっていない。
- 静的テストの warning/error を残していない。
- evidence 更新が品質ゲートの一部として行われている。

## 未達時

- 不足観点を issue にする。
- 実行結果、ログ、coverage summary、証跡が不足している場合も issue にする。
- 生成役へテスト追加または実装修正を依頼する。
- 最大 3 回のループで解消できない場合は、issue を TBC へ移動し、state に記録する。
