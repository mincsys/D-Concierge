# Generator Official System Test

## あなたが担当すること

あなたはこの依頼で、全機能の機能結合完了後に、既存の総合テスト方針と `テスト仕様・結果` に従い、正式総合テストを実行、記録、証跡保存します。

## 今回の目的

全機能の機能結合完了後の正式総合テストを、既存 docs に基づいて実施します。既存の総合テスト方針と `テスト仕様・結果` を入力として扱います。`.tmp` の機能別総合テスト結果は参考情報であり、正式結果として転記しません。

## 実施すること

- `docs/04_テスト/04_総合テスト/総合テスト方針.md` を読む。
- 指定された `docs/04_テスト/04_総合テスト/テスト仕様・結果/*.md` を読む。
- 対象機能一覧と結合完了済み機能を確認し、総合テスト仕様の対象範囲と照合する。
- 機能別総合テスト集約と正式総合テスト重点確認項目を読み、正式総合テストで重点確認する。
- Playwright CLI で確認できる範囲は CLI 実行、画面確認、スクリーンショット取得を行う。
- Sandbox 環境で Playwright CLI、画面確認、スクリーンショット取得、証跡保存に承認が必要な場合は、承認付きで実行する。承認が必要であることだけを理由に未実施や保留にしない。
- 手動確認が必要な項目は、実施可否、結果、保留理由を記録する。
- 総合テスト結果を既存の `テスト仕様・結果` に記録する。
- 方針に従って証跡を保存する。
- `.tmp` の機能別総合テスト結果を正式結果として転記しない。正式総合テストで改めて確認した結果だけを `docs/04_テスト/04_総合テスト/` に記録する。
- 不合格または保留となった項目は、単なる再実行、証跡追加、または保留再記録だけで完了扱いにしない。確認した画面、ログ、DB、trace、frontend console、実行不能理由、再実施条件を報告し、検証役が原因分析できる材料を残す。
- 失敗や保留が実装、設定、テストデータ、テスト手順、仕様不整合の可能性を示す場合は、推測で合格化せず、確認済み事実と未確認事項を分けて報告する。

## 実施しないこと

- 総合テスト仕様の新規作成。
- 不要な Playwright テストコードの作成。
- 単体・結合テストコードの作成。
- `.tmp` の機能別総合テスト結果を正式結果としてコピーすること。
- issue 削除、TBC 移動。
- 全体 state、機能別 `state.md`、タスクリストの更新。
- 不合格・保留 issue の作成または削除。
