あなたはこの依頼で、対象機能の結合レビュー後に、正式総合テストと同じ流れ、同じ記録形式、同じ evidence 保存方針で機能別総合テストを実行、記録、証跡保存します。

## 実施すること

- `docs/04_テスト/04_総合テスト/` 全体を `.tmp/implement-from-docs-v2/features/<機能>/system-test/` へコピーする。
- コピー後の `.tmp` 側の総合テスト方針、`テスト仕様・結果`、evidence 保存先を使う。
- 対応総合テスト仕様、主対象ケース ID、後続依存ケース ID を読み、対象ケースの `前提条件`、`操作手順`、`期待結果` どおりに実施する。
- 正式総合テストと同じ流れ、同じ記録形式、同じ evidence 保存方針で機能別総合テストを実行する。
- `.tmp` 側の `テスト仕様・結果` に結果、証跡、保留理由を記録する。
- `.tmp` 側の `evidence/` に証跡を保存する。
- Sandbox 環境で Playwright CLI、画面確認、スクリーンショット取得、証跡保存に承認が必要な場合は、承認付きで実行する。承認が必要であることだけを理由に未実施や `環境制約` にしない。
- API確認、DB確認、結合テスト、ログ確認などを実施した場合も、総合テスト仕様の操作手順と期待結果を満たしていない限り、自分の判断で合格扱いしない。これらは `補助証跡` として記録する。
- 仕様どおりに実施できない場合は、未実施の手順番号、阻害要因、確認済み期待結果、未確認期待結果、補助証跡、再実施条件を記録する。
- 各テスト結果を `合格 / 不合格 / 部分確認 / 後続機能待ち / 外部認証待ち / 自動操作不可 / 環境制約 / 対象外` に分類する。
- `部分確認`、`後続機能待ち`、`外部認証待ち`、`自動操作不可`、`環境制約`、`対象外` のうち、正式総合テストで再確認すべき項目を整理する。

## 実施しないこと

- `docs/04_テスト/04_総合テスト/` の変更。
- 総合テスト仕様の新規作成。
- 不要な Playwright テストコードの作成。
- 単体・結合テストコードの作成。
- issue 削除、TBC 移動。
- `state.md`、タスクリストの更新。
- レビュー結果の自己承認。
- 仕様書の操作手順を生成役判断で別手順へ置き換えること。
- 補助証跡だけで総合テストケースを合格扱いすること。

## 結果分類

- `合格`: 機能別時点で仕様書の前提条件、操作手順、期待結果どおりに確認でき、期待結果を満たす。
- `不合格`: 実施したが期待結果を満たさない。
- `部分確認`: 一部の期待結果だけ確認でき、未実施の手順番号、阻害要因、確認済み期待結果、未確認期待結果、補助証跡、再実施条件を記録した状態。部分確認は合格相当ではない。
- `後続機能待ち`: 未完了の後続機能が必要で、現時点では実施できない。
- `外部認証待ち`: 外部サービスや外部認証が必要で、現時点の実行環境では実施できない。
- `自動操作不可`: 総合テスト仕様の手順が自動操作できず、手動操作または別手段の合意が必要で実施できない。
- `環境制約`: Sandbox 承認以外の権限、ネットワーク、ブラウザ、デバイス、外部リソースなどの制約で実施できない。Sandbox 承認が必要なだけの場合は、承認付き実行を試してから分類する。
- `対象外`: 対象機能時点では関係しない。
