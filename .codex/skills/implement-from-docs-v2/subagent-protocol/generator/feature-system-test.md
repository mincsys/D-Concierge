あなたはこの依頼で、対象機能の結合レビュー後に、正式総合テストと同じ流れ、同じ記録形式、同じ evidence 保存方針で機能別総合テストを実行、記録、証跡保存します。

正式総合テスト前に、対象機能の利用者視点の確認を `.tmp` 上で行います。`docs/04_テスト/04_総合テスト/` は正本として読み、変更しません。

## 実施すること

- `docs/04_テスト/04_総合テスト/` 全体を `.tmp/implement-from-docs-v2/features/<機能>/system-test/` へコピーする。
- コピー後の `.tmp` 側の総合テスト方針、`テスト仕様・結果`、evidence 保存先を使う。
- 正式総合テストと同じ流れ、同じ記録形式、同じ evidence 保存方針で機能別総合テストを実行する。
- `.tmp` 側の `テスト仕様・結果` に結果、証跡、保留理由を記録する。
- `.tmp` 側の `evidence/` に証跡を保存する。
- Sandbox 環境で Playwright CLI、画面確認、スクリーンショット取得、証跡保存に承認が必要な場合は、承認付きで実行する。承認が必要であることだけを理由に未実施や `環境・承認待ち` にしない。
- 各テスト結果を `合格 / 不合格 / 部分確認 / 後続機能待ち / 環境・承認待ち / 対象外` に分類する。
- `部分確認`、`後続機能待ち`、`環境・承認待ち`、`対象外` のうち、正式総合テストで再確認すべき項目を整理する。

## 実施しないこと

- `docs/04_テスト/04_総合テスト/` の変更。
- 総合テスト仕様の新規作成。
- 不要な Playwright テストコードの作成。
- 単体・結合テストコードの作成。
- issue 削除、TBC 移動。
- `state.md`、タスクリストの更新。
- レビュー結果の自己承認。

## 結果分類

- `合格`: 機能別時点で正式手順どおり確認でき、期待結果を満たす。
- `不合格`: 実施したが期待結果を満たさない。
- `部分確認`: 一部条件だけ確認でき、残条件は正式総合テストで確認する。
- `後続機能待ち`: 未完了の後続機能が必要で、現時点では実施できない。
- `環境・承認待ち`: 外部送信、手動判断、Sandbox 承認以外の権限、環境制約などの合意または準備が必要で実施できない。Sandbox 承認が必要なだけの場合は、承認付き実行を試してから分類する。
- `対象外`: 対象機能時点では関係しない。
