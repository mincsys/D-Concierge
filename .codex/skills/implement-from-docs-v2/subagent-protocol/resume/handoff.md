# Resume Handoff

## あなたが担当すること

あなたはこの依頼で、前回から継続して同じ役割の作業を再開します。前回の担当者を再利用している場合も、新規に起動された場合も、以下の引き継ぎを正として作業します。

## 今回の目的

空の完了メッセージ、リミット到達、利用不能、文脈喪失などで中断した作業を、前回の状態から再開します。依頼元が渡す state、tasklist、issue、差分情報、前回報告、未完了作業を確認し、正式な完了報告を返すまで作業します。

- 再開種別: 再利用再開 / 新規再起動
- 対象範囲:
- 対象機能:
- 現在フェーズ:
- 参照 skill: `.codex/skills/implement-from-docs-v2/SKILL.md`
- 参照 state:
- 参照 tasklist:
- 関連 docs:
- 参照 reference:
- 前回の作業報告:
- 前回の最終応答:
- 中断理由:
- 完了済み作業:
- 未完了作業:
- 対象 issue / TBC:
- 関連 diff / status:
- 保存済み evidence:
- 最新ユーザ制約:
- 禁止事項:
- 完了条件:
- 報告形式:

## 実施すること

- `.codex/skills/implement-from-docs-v2/SKILL.md` を軽く読み、Overview、Workflow、Rules、Done Criteria、Resources から、自分の担当、他エージェントの担当、依頼元との責務境界、完了条件を理解する。
- 依頼元が渡した state、tasklist、前回報告、issue/TBC、差分情報、evidence、未完了作業を確認する。
- 再開種別が `新規再起動` の場合、前回サブエージェントの文脈を持たない前提で、引き継ぎ内容から現在地を復元する。
- 成果物作成を担当する場合は、未完了の実装、テスト、総合テスト、証跡保存、仕様書側修正 issue 対応だけを進める。
- 検証を担当する場合は、依頼元が渡した観点、差分、報告、証跡、分類、完了条件に基づいて確認し、判定を返す。
- 判断できない仕様、証跡不足、実施不能な作業があれば、補完せず依頼元へ報告する。

## 実施しないこと

- 追加サブエージェント起動。
- 管理用タスクリストの更新。
- `state.md` の更新。
- 自分の役割外の生成、検証、完了可否判定。
- issue 削除、TBC 移動。
- 前回作業や他者変更の巻き戻し。
- 空の完了メッセージを完了扱いすること。

## 生成作業を担当する場合の追加確認

- 未完了のテストコード、実装、単体テスト、結合テスト、機能別総合テスト、正式総合テスト、evidence 保存のどこから再開するかを明示する。
- 実行したコマンド、結果、ログ、coverage summary、保存 evidence、変更ファイル、未完了事項を報告する。
- 機能別総合テストでは `docs/04_テスト/04_総合テスト/` を変更せず、`.tmp` 側へ記録する。

## 検証作業を担当する場合の追加確認

- レビュー対象、review-artifacts checklist、差分、証跡、生成作業報告を確認する。
- `docs` 非変更確認が必要なフェーズでは、依頼元が渡した status/diff 情報で確認する。
- 分類、TBC 管理、完了可否、アプリ完成可否を混同しない。
- 不合格、証跡不足、実施可能なのに未実施、未分類の未実施があれば、issue 化または追加依頼対象として報告する。

## 完了条件

- `SKILL.md` の軽読により、役割境界と完了条件を理解している。
- 引き継ぎ内容から現在地、完了済み作業、未完了作業、禁止事項を把握している。
- 依頼された再開作業または再開後レビューを実施し、正式な完了報告を返している。
- 未完了、保留、証跡不足、判断不能がある場合は、完了扱いせず明示している。

- evidence:

- 判断できない仕様:
- 完了条件の充足:
- 次に依頼元が行うべきこと:
