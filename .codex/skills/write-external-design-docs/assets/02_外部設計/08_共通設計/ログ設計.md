# ログ設計

## 1. 文書の目的
【template-guidance】<br>
必須: ログの出力方針と運用上の利用目的を書く。<br>
任意: 監査や障害解析用途を補足してよい。<br>
書かない: 実装コード断片。<br>
【/template-guidance】<br>

本書は、〇〇システムにおけるログの出力方針、出力項目、保持方針を定義することを目的とする。

## 2. 前提
【template-guidance】<br>
必須: 保存先、保持期間、秘密情報の非出力方針を書く。<br>
任意: ローテーションや相関IDの前提を書いてよい。<br>
書かない: 一時的なデバッグ出力。<br>
【/template-guidance】<br>

- ログは定められた保存先へ出力する。
- 機微情報はログへ出力しない。

## 3. ログ分類
【template-guidance】<br>
必須: ログ区分、レベル、主な内容を書く。<br>
任意: 出力契機や確認主体を列追加してよい。<br>
書かない: 実装ロガー設定値。<br>
【/template-guidance】<br>

| ログ区分 | レベル | 主な内容 |
| --- | --- | --- |
| 認証ログ | INFO/WARN | ログイン成功、失敗 |
| 業務ログ | INFO | 主要業務処理の開始、終了 |
| 例外ログ | ERROR | 異常発生内容 |

## 4. 共通出力項目
【template-guidance】<br>
必須: すべてのログで持たせたい主要項目を書く。<br>
任意: 相関IDや対象IDを追記してよい。<br>
書かない: 業務データ本文の全文。<br>
【/template-guidance】<br>

| 項目 | 内容 |
| --- | --- |
| timestamp | 発生日時 |
| level | ログレベル |
| operation | 処理種別 |
| message | 概要 |

## 5. 保持方針
【template-guidance】<br>
必須: 保存先、保持期間、ローテーション方針を書く。<br>
任意: バックアップや監査保全方針を追記してよい。<br>
書かない: OSコマンド手順。<br>
