# エラーメッセージ設計

## 1. 文書の目的
【template-guidance】<br>
必須: 利用者向けメッセージの統一と画面間整合を目的として書く。<br>
任意: 運用向け分類やログレベルとの対応を書いてよい。<br>
書かない: 内部例外メッセージの生文。<br>
【/template-guidance】<br>

本書は、〇〇システムで利用するメッセージID、表示文言、発生条件、表示画面を定義することを目的とする。

## 2. 前提
【template-guidance】<br>
必須: 文言は利用者向けに平易にすること、内部詳細を見せないことを書く。<br>
任意: 多言語や表示位置の前提を書いてよい。<br>
書かない: 画面ごとのレイアウト説明。<br>
【/template-guidance】<br>

- 利用者向けメッセージは理解しやすい文言とする。
- 内部例外の詳細はそのまま表示しない。

## 3. メッセージ一覧
【template-guidance】<br>
必須: メッセージID、区分、表示画面、表示文言、発生条件を書く。<br>
任意: ログレベルや備考を追加してよい。<br>
書かない: 文言検討履歴。<br>

## 4. 表示方針
【template-guidance】<br>
必須: 項目付近表示、画面上部表示などの方針を書く。<br>
任意: 注意とエラーの見せ分けを書いてよい。<br>
書かない: 色やスタイル実装の詳細。<br>
