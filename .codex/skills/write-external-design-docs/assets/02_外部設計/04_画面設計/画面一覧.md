# 画面一覧

## 1. 文書の目的
【template-guidance】<br>
必須: 画面全体を一覧化し、遷移と個別画面設計の起点にする目的を書く。<br>
任意: 画面ID体系や対象利用者を補足してよい。<br>
書かない: 画面部品の詳細説明。<br>
【/template-guidance】<br>

本書は、〇〇システムで提供する画面を一覧化し、用途と遷移関係を明確にすることを目的とする。

## 2. 前提
【template-guidance】<br>
必須: 画面一覧の整理単位や命名規則を書く。<br>
任意: SPA、サーバレンダリングなど表示方式を書いてよい。<br>
書かない: 実装ライブラリ名。<br>
【/template-guidance】<br>

- 各画面は個別画面設計と対応付ける。

## 3. 画面一覧
【template-guidance】<br>

任意: 備考列を追加してよい。<br>
書かない: API詳細。<br>

## 4. 画面遷移図

## 5. 共通方針
【template-guidance】<br>
必須: 認証前提、エラー表示、共通ナビゲーションなどを書く。<br>
任意: 表示端末前提やアクセシビリティ方針を追記してよい。<br>
書かない: CSSやUI実装詳細。<br>
【/template-guidance】<br>

- 画面ごとの詳細は個別画面設計で定義する。
