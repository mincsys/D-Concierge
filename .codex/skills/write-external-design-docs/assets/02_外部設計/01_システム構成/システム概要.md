【template-guidance】<br>
必須: 代表的な業務シナリオ1つを選び、処理の流れを段階で示す。<br>

書かない: 全例外をここへ詰め込むこと。<br>
【/template-guidance】<br>

1. 利用者が必要情報を入力または参照要求する。
2. 〇〇システムが入力内容を受け付け、必要な業務処理を実行する。
3. 必要に応じて外部サービスAと連携する。
4. 処理結果をデータ保存先へ反映し、利用者へ応答する。
