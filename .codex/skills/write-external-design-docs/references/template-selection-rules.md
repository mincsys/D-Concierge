# 外部設計テンプレートの選定ルール

## 1. 基本方針

- テンプレートの実体は `assets/02_外部設計` を正とする。
- ユーザがフォルダ構成を指定している場合は、その構成へ必要テンプレートだけを当て込む。
- 文書の採用可否は `必須` `任意` `汎用ひな型` の区分で判断する。

## 2. 区分ごとの扱い

| 区分 | 扱い |
| --- | --- |
| 必須 | 原則として採用する。対象システムでその設計観点自体が不要な場合のみ外す |
| 任意 | 要件、業務特性、ユーザ指定のいずれかで必要なときだけ採用する |
| 汎用ひな型 | assets 上に保持し、最終成果物へは具体名へ複製して採用する |

## 3. 汎用ひな型を増やす判断

- `〇〇フロー.md` は、業務一覧だけでは手順や例外を表しきれない業務ごとに増やす。
- `〇〇画面.md` は、画面一覧に載る各画面へ対応する具体名で増やす。
- `〇〇IF.md` は、外部IF設計で同一の IF 相手との連携を 1 文書にまとめて定義するときに増やす。
- 汎用ひな型は最終成果物に残さず、具体名へ置き換えた後にのみ採用完了とみなす。

## 4. 任意文書の採用基準

| 文書 | 採用する条件 |
| --- | --- |
| `08_共通設計/用語集.md` | 用語の意味が文書間でずれやすい、独自用語が多い、画像処理や外部連携の共通定義が必要 |
| `09_その他/補助資料.md` | ユーザ指定の素案外だが、補助資料がないと誤解が出る |

## 5. 仕上げ時の注意

- assets のテンプレート説明と同じ内容を、references 側へ重複して書かない。
- 任意文書を採用しない場合は、成果物側へ空の章や空のファイルを残さない。
- 外部インターフェース設計では、IF一覧文書を別に作らず、IF 相手ごとの IF 文書を具体名で起こす。
- `【template-guidance】` ブロックと未置換プレースホルダは成果物から除去する。
