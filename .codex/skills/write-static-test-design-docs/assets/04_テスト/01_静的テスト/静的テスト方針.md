# 静的テスト方針

## 1. 文書の目的
【template-guidance】<br>
- 本書が、ツール解析に限定した静的テストの方針、基準、実施ルールを定義する文書であることを書く。
- 手動レビュー、ウォークスルー、目視確認を扱う文書ではないことが分かるようにする。

【/template-guidance】<br>

本書は、対象システムの静的テストにおける対象範囲、対象外範囲、使用ツール、検出観点、設定方針、判定基準、実施手順および成果物・管理対象を定義することを目的とする。

## 2. 前提
【template-guidance】<br>
- 参照する要件定義書、設計書、開発標準、ユーザ合意事項を書く。
- どのシステム、リポジトリ、ディレクトリを対象にするかはユーザ合意に基づいて書く。
- 静的テストは `lint`、`format check`、`type check` に限定することを書く。
- 静的テスト方針書と設定ファイル以外の文書を原則作成しないことを書く。

【/template-guidance】<br>

- 本書は、要件定義書、設計書および開発標準を前提とする。
- 本書で扱う静的テストは、ツールによる `lint`、`format check`、`type check` に限定する。
- security scan、dependency scan、secret scan、license check は本書の対象外とする。
- 静的テスト設定ファイルは、リポジトリで管理する。
- 静的テスト方針書と設定ファイル以外の文書は、ユーザから明示依頼がある場合のみ作成する。

## 3. 静的テスト対象範囲
【template-guidance】<br>
- 静的テスト対象にするシステム、リポジトリ、ディレクトリ、ファイル種別を書く。
- 推奨は、全てのソースコードとテストコードを対象にすること。
- 設定ファイル、依存定義、CI 定義を対象に含めるかは、ユーザ合意に基づいて書く。

【/template-guidance】<br>

| 区分 | 対象範囲 |
| --- | --- |
| ソースコード | ユーザと合意した対象 |
| テストコード | ユーザと合意した対象 |
| 設定ファイル | ユーザと合意した対象 |
| 依存定義 | ユーザと合意した対象 |
| CI 定義 | ユーザと合意した対象 |

## 4. 静的テスト対象外範囲
【template-guidance】<br>
- 静的テスト対象範囲のうち、例外的に対象外にする内容を書く。
- 生成物、外部ライブラリ、ビルド成果物、配布物、手動レビュー、ウォークスルー、目視確認の扱いを書く。
- 対象外理由と代替確認方法を書く。

【/template-guidance】<br>

| 対象外範囲 | 対象外理由 | 代替確認方法 |
| --- | --- | --- |
| ユーザと合意した対象外範囲 | ユーザと合意した理由 | ユーザと合意した代替確認方法 |

## 5. 静的テスト種別
【template-guidance】<br>
- 本書で採用する静的テスト種別を `lint`、`format check`、`type check` の 3 種別に限定して書く。
- security scan、dependency scan、secret scan、license check は採用しないことを書く。

【/template-guidance】<br>

| 種別 | 採用方針 | 目的 |
| --- | --- | --- |
| lint | 採用する | コーディング規約違反、危険な構文、未使用要素を検出する |
| format check | 採用する | フォーマット設定との差分を検出する |
| type check | 採用する | 型不整合、未定義参照、型境界の破れを検出する |

## 6. 使用ツール方針
【template-guidance】<br>
- lint、format check、type check ごとに、使用ツール、対象、設定ファイル、実行コマンドを書く。
- ツール名と設定ファイルは、既存設定またはユーザ合意に基づいて書く。
- ローカル実行と CI 実行では、原則として同じ設定ファイルを参照する方針を書く。

【/template-guidance】<br>

| 種別 | 使用ツール | 対象 | 設定ファイル | 標準実行コマンド |
| --- | --- | --- | --- | --- |
| lint | ユーザと合意したツール | ユーザと合意した対象 | ユーザと合意した設定ファイル | ユーザと合意したコマンド |
| format check | ユーザと合意したツール | ユーザと合意した対象 | ユーザと合意した設定ファイル | ユーザと合意したコマンド |
| type check | ユーザと合意したツール | ユーザと合意した対象 | ユーザと合意した設定ファイル | ユーザと合意したコマンド |

## 7. ツール別検出観点
【template-guidance】<br>
- 各ツールで何を検出対象にするかを書く。
- 使用ツール決定後に公式ドキュメントを調査し、ユーザ合意した観点だけを書く。
- 検出対象を列挙する場合は、曖昧な包括表現を使わず具体名で書く。

【/template-guidance】<br>

| 種別 | 検出観点 |
| --- | --- |
| lint | ユーザと合意した検出観点 |
| format check | ユーザと合意した検出観点 |
| type check | ユーザと合意した検出観点 |

## 8. ツール設定方針
【template-guidance】<br>
- 使用ツール決定後に公式ドキュメントを調査し、設定可能項目を整理した結果を書く。
- 必須設定、推奨設定、任意設定、採用しない設定に分類する。
- 設定ファイルはリポジトリ管理する固定方針として書く。

【/template-guidance】<br>

| 種別 | 設定分類 | 設定項目 | 採用方針 |
| --- | --- | --- | --- |
| lint | ユーザと合意した分類 | ユーザと合意した設定項目 | ユーザと合意した採用方針 |
| format check | ユーザと合意した分類 | ユーザと合意した設定項目 | ユーザと合意した採用方針 |
| type check | ユーザと合意した分類 | ユーザと合意した設定項目 | ユーザと合意した採用方針 |

## 9. 判定基準
【template-guidance】<br>
- error と warning の扱い、失敗条件、既存指摘の暫定許容をどう扱うかを書く。
- 推奨は、error が 1 件でもあれば不合格、warning も不合格に含めること。

【/template-guidance】<br>

| 項目 | 判定基準 |
| --- | --- |
| error | 1 件でも検出された場合は不合格とする |
| warning | 1 件でも検出された場合は不合格とする |
| 自動修正可能な指摘 | ユーザと合意した扱い |
| 既存指摘 | ユーザと合意した扱い |

- 本文には、除外・抑止を許可する条件だけを書く。

【/template-guidance】<br>

## 12. 静的テスト実施手順
【template-guidance】<br>
- 静的テストを実施するタイミング、標準実行コマンド、失敗時対応、再実行手順を書く。
- 推奨は、ファイルを作成、編集したときに実施し、失敗時は修正して再テストを実施すること。

【/template-guidance】<br>

- 静的テストは、ファイルを作成、編集したときに実施する。
- 静的テストで不合格となった場合は、指摘箇所を修正して再テストを実施する。
- 標準実行コマンドは、ユーザと合意したコマンドを使用する。

## 13. 成果物・管理対象
【template-guidance】<br>
- 静的テスト方針書と設定ファイルを管理対象として書く。
- 実行ログ、解析レポート、設定変更理由、抑止理由の別ドキュメントは、ユーザから明示依頼がある場合のみ作成する方針を書く。

【/template-guidance】<br>

| 区分 | 扱い |
| --- | --- |
| 静的テスト方針書 | 正式成果物として管理する |
| 静的テスト設定ファイル | リポジトリで管理する |
| 実行ログ | ユーザから明示依頼がある場合のみ保存対象とする |
| 解析レポート | ユーザから明示依頼がある場合のみ保存対象とする |
| 設定変更理由 | ユーザから明示依頼がある場合のみ別ドキュメント化する |
| 抑止理由 | ユーザから明示依頼がある場合のみ別ドキュメント化する |
