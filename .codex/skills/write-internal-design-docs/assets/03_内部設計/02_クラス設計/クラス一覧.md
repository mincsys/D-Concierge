# クラス一覧

## 1. 文書の目的
【template-guidance】<br>
- ここには、本書がクラス設計の入口文書であることを書く。
- 詳細文書はソースコードディレクトリ構成と同じ階層へ展開することを必要に応じて補足してよい。
- 各クラスの詳細設計本文はここへ書かない。

【/template-guidance】<br>

本書は、〇〇システムで設計対象とする主要クラスを一覧化し、どのクラスに詳細設計を起こすかを整理することを目的とする。

## 2. 前提
【template-guidance】<br>
- ここには、フレームワーク標準クラスとライブラリ提供クラスを除外することを書く。
- 自動生成クラスや単純データ保持クラスの扱いは必要に応じて補足してよい。
- クラスの実装コードは書かない。

【/template-guidance】<br>

- 対象は、アプリケーション固有に定義するクラスのみとする。
- フレームワーク標準クラス、ライブラリ提供クラス、自動生成クラスは対象外とする。

## 3. クラス一覧
【template-guidance】<br>
- ここでは、フレームワーク標準クラスやライブラリ提供クラスを除いた全クラスの一覧表を作る。
- 列は `区分` `クラス名` `責務` `詳細ファイル除外理由` を基本とする。
- `区分` には、クラスが属するディレクトリパスを書く。
- `クラス名` は、対応する `〇〇クラス設計.md` への Markdown リンクにする。
- 詳細設計を書くクラスは `クラス名` を Markdown リンクにし、`詳細ファイル除外理由` 列には `-` を書く。
- 自動生成クラス、項目保持だけの単純クラス、フレームワーク規約どおりの薄いクラスは詳細ファイルを作らず、`クラス名` はリンクにせず、`詳細ファイル除外理由` 列に理由を簡潔に書く。
- 表の並びは区分の親子関係が追える順にし、親ディレクトリの直後にその配下のクラスを並べる。
- 同じクラスを複数の詳細ファイルへ重複記載しない。
