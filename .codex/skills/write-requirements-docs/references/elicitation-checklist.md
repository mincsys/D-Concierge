# 要件聞き取りチェックリスト

## 1. 背景と目的

- 何が不便で、何を改善したいのか。
- なぜ今このシステムが必要なのか。
- 成果として何ができれば成功なのか。

## 2. 利用者と利用シーン

- 誰が使うのか。
- どの端末、どの環境で使うのか。
- 管理者や運用担当者は存在するのか。

## 3. 対象範囲

- 初期リリースで必須の機能は何か。
- 後回しでよい機能は何か。
- 対象外として明示すべきものは何か。

## 4. 業務と機能

- 利用者はどの順で操作するのか。
- 保存、参照、検索、集計、承認、通知など、必要な機能は何か。
- 失敗時や例外時にどう扱うべきか。

## 5. 画面と操作

- 最低限必要な画面は何か。
- 各画面で何を表示し、何を入力し、何を操作するのか。
- スマートフォン前提か、PC前提か。

## 6. データ

- 何を保存する必要があるのか。
- 誰のデータか、どの単位で持つのか。
- 一覧、詳細、集計で必要なデータは何か。

## 7. 外部連携

- どの外部サービスや外部システムと連携するのか。
- 入出力は何か。
- 同期か非同期か。

## 8. 非機能

- どの程度の速度が必要か。
- どの程度の信頼性が必要か。
- セキュリティ、権限、保存期間、通知要件はあるか。

## 9. 質問順の目安

1. 背景、目的、利用者
2. 初期リリース範囲と対象外
3. 主要業務フロー
4. 主要機能、画面、データ
5. 外部連携と非機能

## 10. 曖昧なときの扱い

- 後続の設計や実装に影響が大きい論点は質問する。
- 軽微な表現や文言は既存文書に合わせて補う。
- 質問しないで進める場合は、文書全体で一貫した前提にそろえる。
