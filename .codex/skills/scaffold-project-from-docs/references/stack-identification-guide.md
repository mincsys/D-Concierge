# Stack Identification Guide

## 何を把握するか

- backend 実行基盤
- frontend 実行基盤
- データベース
- コンテナや仮想化の有無
- テストフレームワーク
- 静的解析ツール
- 外部サービス
- 証跡や成果物の保存先

## どこから判断するか

- 要件定義書:
  - システムの対象範囲
  - 管理画面や外部サービスの存在
- ソフトウェア構成:
  - ソフトウェア名
  - 配置先
  - 実行形態
  - ビルド方針
  - 配備方針
- 静的テスト方針:
  - lint、format check、type check の使用ツール
  - 設定ファイル名
  - 標準実行コマンド
- 単体テスト方針:
  - テストフレームワーク
  - カバレッジ取得方法
  - テスト資材、証跡の保存場所
- 結合テスト方針:
  - 実 DB 利用有無
  - モック方針
  - 実行環境
  - 証跡の保存場所

## 判断時の注意

- 技術名が 1 回出るだけで決めず、関連するコマンド、設定ファイル、責務記述も合わせて確認する。
- docs に根拠がない技術は追加しない。
- docs と既存設定が競合した場合は、競合を明示して止まる。
- `docs/05_開発標準` を前提にしない。
