# Review Checklist

## 進行前

- 主要 6 文書を読んだか。
- `docs/05_開発標準` を前提にしていないか。
- インストール一覧を生成前に提示したか。
- ユーザの明示許可前にファイル生成へ進んでいないか。

## 判断

- 技術スタックは docs の根拠で説明できるか。
- docs にない技術を推測で足していないか。
- 競合する記述を見落としていないか。
- 未確定事項を未確定のまま示しているか。

## 実行

- `sudo` 必須操作を自分で実行していないか。
- ユーザへ依頼するコマンドはそのまま実行できる形か。
- 技術固有ファイルは docs 根拠がある場合だけ生成しているか。
- CI 定義は基盤が明示されている場合だけ生成しているか。

## 完了前

- 作成したものと作成しなかったものを分けて説明できるか。
- 共通テンプレートの利用が技術固定になっていないか。
- 生成結果が docs と矛盾していないか。
