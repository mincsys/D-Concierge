# 本番環境構築

## 1. 文書の目的
【template-guidance】<br>
- 本書が本番環境の構築手順を定義する文書であることを書く。
- 本番運用できる状態にするための手順であり、開発環境構築手順ではないことが分かるようにする。

【/template-guidance】<br>

本書は、対象システムの本番環境を構築し、実運用できる状態にするための環境構成、構築手順、起動確認、監視、ログ、バックアップおよび障害時対応の入口を定義することを目的とする。

## 2. 前提
【template-guidance】<br>
- 対象システム、実行基盤、参照資料、責任範囲、前提契約を書く。
- 特定クラウドや運用方式は、既存資料またはユーザ合意に基づく場合だけ具体化する。
- 秘密値そのもの、個人アカウント情報、実運用の認証情報は書かない。

【/template-guidance】<br>

- 本書は、要件定義書、外部設計書、内部設計書、テスト設計書、README、および設定ファイルを前提とする。
- 本書の対象環境は、ユーザ合意またはリポジトリ事実に基づく本番環境とする。
- 秘密値そのもの、個人アカウント情報、実運用の認証情報は本書に記載しない。
