# 開発環境構築

## 1. 文書の目的
【template-guidance】<br>
- 本書が開発環境の構築手順を定義する文書であることを書く。
- 開発者が作業開始できる状態にするための手順であり、本番運用手順ではないことが分かるようにする。

【/template-guidance】<br>

本書は、対象システムの開発環境を構築し、開発者が実装、確認、テストを開始できる状態にするための手順を定義することを目的とする。

## 2. 前提
【template-guidance】<br>
- 対象 OS、対象環境、参照資料、利用する開発方式を書く。
- 既存資料から一意に確定できる内容はそのまま書く。
- 未確定事項や検討メモは残さない。

【/template-guidance】<br>

- 本書は、要件定義書、外部設計書、内部設計書、テスト設計書、README、および設定ファイルを前提とする。
- 本書の対象環境は、ユーザ合意またはリポジトリ事実に基づく開発環境とする。
- 秘密値そのものは本書に記載しない。

## 3. 必要なツール
【template-guidance】<br>
- OS、ランタイム、パッケージマネージャ、DB クライアント、コンテナ、クラウド CLI、開発支援ツールを書く。
- バージョンは README、lock ファイル、設定ファイル、公式設定から分かる範囲で具体化する。
- 不明なバージョンは推測で固定せず、ユーザ合意に基づいて書く。

| 区分 | ツール | バージョン | 用途 |

| ユーザ合意またはリポジトリ事実に基づく区分 | ユーザ合意またはリポジトリ事実に基づくツール | ユーザ合意またはリポジトリ事実に基づくバージョン | ユーザ合意またはリポジトリ事実に基づく用途 |

## 4. 環境変数・設定ファイル
【template-guidance】<br>
- `.env.example`、環境変数サンプル、設定ファイルから必要な項目を書く。
- 秘密値そのものは書かず、設定先、項目名、用途、設定例の考え方を書く。
- 開発環境用の値と本番環境用の値を混ぜない。
