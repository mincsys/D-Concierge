# 開発環境構築

## 1. 文書の目的

本書は、D-Conciergeの開発環境を構築し、開発者が実装、確認、静的テスト、単体テスト、結合テストを開始できる状態にするための手順を定義することを目的とする。

本書は開発環境を対象とし、本番運用の常駐管理、バックアップ、監視、障害対応は `本番環境構築.md` で扱う。

## 2. 前提

- 本書は、要件定義書、外部設計書、内部設計書、テスト設計書、`pyproject.toml`、`uv.lock`、`src/frontend/package.json`、`src/frontend/package-lock.json`、`infra/compose.yml`、`config.example.yaml` を前提とする。
- 開発環境はLinuxとWindowsを対象とする。
- Python依存関係は `uv` と `.venv` で管理する。
- Node.js依存関係は `npm` と `src/frontend/package-lock.json` で管理する。
- 開発用PostgreSQLはDocker Composeで起動することを標準とする。
- `config.example.yaml` はGit管理対象の設定ひな形とし、開発者はこれをコピーして `config.yaml` を作成する。
- `config.yaml` は環境依存値を含むためGit管理対象外とする。
- Codex CLIの認証情報はGit管理対象外とし、各開発者がローカル環境で事前に認証を完了する。
- sudoまたは管理者権限が必要なツール導入は、開発者が所属環境の標準手順に従って実施する。

## 4. 環境変数・設定ファイル

| 項目 | 設定先 | 用途 | 設定方針 |
| --- | --- | --- | --- |
| `config.example.yaml` | リポジトリルート | `config.yaml` 作成元 | Git管理対象とする |
| `config.yaml` | リポジトリルート | アプリケーション実行設定 | `config.example.yaml` からコピーし、開発者ごとの値へ変更する |
| `app.timezone` | `config.yaml` | 運用者向け日時の基準 | IANA timezone名で指定し、開発標準は `Asia/Tokyo` とする |
| `D_CONCIERGE_DATABASE_URL` | シェル環境変数 | Alembic migration実行時のDB接続先 | migration実行時だけ設定する |
| `D_CONCIERGE_TEST_DATABASE_URL` | シェル環境変数 | バックエンド結合テストのPostgreSQL接続先 | 未指定時は `postgres-test` の既定接続先を使用する |
| `UV_CACHE_DIR` | シェル環境変数 | uvキャッシュ保存先 | Linuxでは `/tmp/uv-cache`、Windowsでは `%TEMP%\uv-cache` を使用する |
| `VITE_BACKEND_TARGET` | シェル環境変数 | Vite開発サーバから実バックエンドへの転送先 | 未指定時は `http://127.0.0.1:8000` を使用する |
| `POSTGRES_DB` | Docker Compose環境変数 | 開発用PostgreSQLのDB名 | 未指定時は `d_concierge` を使用する |
| `POSTGRES_USER` | Docker Compose環境変数 | 開発用PostgreSQLのユーザ名 | 未指定時は `d_concierge` を使用する |
| `POSTGRES_PASSWORD` | Docker Compose環境変数 | 開発用PostgreSQLのパスワード | 開発環境では既定値を使用できる |
| `POSTGRES_PORT` | Docker Compose環境変数 | 開発用PostgreSQLのホスト公開ポート | 未指定時は `5432` を使用する |
| `POSTGRES_TEST_DB` | Docker Compose環境変数 | 結合テスト用PostgreSQLのDB名 | 未指定時は `d_concierge_test` を使用する |
| `POSTGRES_TEST_USER` | Docker Compose環境変数 | 結合テスト用PostgreSQLのユーザ名 | 未指定時は `d_concierge_test` を使用する |
| `POSTGRES_TEST_PASSWORD` | Docker Compose環境変数 | 結合テスト用PostgreSQLのパスワード | 未指定時は `d_concierge_test` を使用する |
| `POSTGRES_TEST_PORT` | Docker Compose環境変数 | 結合テスト用PostgreSQLのホスト公開ポート | 未指定時は `55432` を使用する |

## 6. アプリケーションのセットアップ

### 6.1. Linux

リポジトリルートで以下を実行する。

```bash
cp config.example.yaml config.yaml
UV_CACHE_DIR=/tmp/uv-cache uv sync --all-groups
cd src/frontend
npm ci
cd ../..
```

開発用PostgreSQLを起動する。

```bash
docker compose -f infra/compose.yml up -d postgres
```

初期スキーマを作成する。

```bash
D_CONCIERGE_DATABASE_URL='postgresql+psycopg://d_concierge:d_concierge@127.0.0.1:5432/d_concierge' \
UV_CACHE_DIR=/tmp/uv-cache \
uv run alembic upgrade head
```

### 6.2. Windows

PowerShellを開き、リポジトリルートで以下を実行する。

```powershell
Copy-Item config.example.yaml config.yaml
$env:UV_CACHE_DIR = "$env:TEMP\uv-cache"
uv sync --all-groups
Set-Location src/frontend
npm ci
Set-Location ../..
```

開発用PostgreSQLを起動する。

```powershell
docker compose -f infra/compose.yml up -d postgres
```

初期スキーマを作成する。

```powershell
$env:D_CONCIERGE_DATABASE_URL = "postgresql+psycopg://d_concierge:d_concierge@127.0.0.1:5432/d_concierge"
$env:UV_CACHE_DIR = "$env:TEMP\uv-cache"
uv run alembic upgrade head
```

検証用Codexへ生成成果物を提示する処理ではシンボリックリンクを使用する。Windowsでsymlink作成権限が不足している場合は環境不備として扱うため、開発者モードまたは同等の権限を有効にしてから動作確認する。

## 7. 起動・停止手順

### 7.1. Linux起動手順

バックエンドを起動する。

```bash
UV_CACHE_DIR=/tmp/uv-cache \
uv run uvicorn backend.main:app --app-dir src --host 127.0.0.1 --port 8000 --reload --reload-dir src/backend
```

別ターミナルでフロントエンドを起動する。

```bash
cd src/frontend
npm run dev
```

フロントエンドだけをモックデータで確認する場合は以下を実行する。

```bash
cd src/frontend
npm run dev:mock
```

### 7.2. Windows起動手順

PowerShellでバックエンドを起動する。

```powershell
$env:UV_CACHE_DIR = "$env:TEMP\uv-cache"
uv run uvicorn backend.main:app --app-dir src --host 127.0.0.1 --port 8000 --reload --reload-dir src/backend
```

別PowerShellでフロントエンドを起動する。

```powershell
Set-Location src/frontend
npm run dev
```

フロントエンドだけをモックデータで確認する場合は以下を実行する。

```powershell
Set-Location src/frontend
npm run dev:mock
```

### 7.3. 停止手順

バックエンドとフロントエンドは、起動したターミナルで `Ctrl+C` により停止する。

PostgreSQLを停止する。

```bash
docker compose -f infra/compose.yml stop postgres
```

結合テスト用PostgreSQLを停止する。

```bash
docker compose -f infra/compose.yml stop postgres-test
```

## 8. 動作確認

| 確認項目 | 確認方法 | 期待結果 |
| --- | --- | --- |
| Python依存関係 | `UV_CACHE_DIR=/tmp/uv-cache uv run python --version` | Python 3.12系が表示される |
| Node.js依存関係 | `cd src/frontend && npm run typecheck` | TypeScript type checkが成功する |
| フロントエンド起動 | `http://127.0.0.1:5173` をChromeで開く | 開始画面またはチャット画面が表示される |
| バックエンドAPI | `curl http://127.0.0.1:8000/api/app-config` | 歓迎メッセージと入力候補だけが返る |
| PostgreSQL | `docker compose -f infra/compose.yml exec postgres psql -U d_concierge -d d_concierge -c "select count(*) from alembic_version;"` | migration revisionが登録されている |
| Codex CLI | `codex --version` | Codex CLIのバージョンが表示される |
| 静的テスト | `UV_CACHE_DIR=/tmp/uv-cache uv run ruff check src/backend`、`cd src/frontend && npm run lint` | 対象コマンドが成功する |
| 単体テスト | `UV_CACHE_DIR=/tmp/uv-cache uv run pytest src/backend/tests/unit`、`cd src/frontend && npm run test:unit` | 対象コマンドが成功する |
| 結合テスト | `docker compose -f infra/compose.yml up -d postgres-test`、`UV_CACHE_DIR=/tmp/uv-cache uv run pytest src/backend/tests/integration`、`cd src/frontend && npm run test:integration` | PostgreSQLテストDBを使用して対象コマンドが成功する |
