# 画像添付

## 目的

本メモは、将来構想要件定義から分離したユーザ指示送信時の画像ファイル添付、添付画像保存、codex exec連携、履歴再表示の詳細を整理する。

## 基本方針

画像添付は、利用者がユーザ指示送信時に画像ファイルを追加し、そのチャット実行の入力として扱う機能である。

添付画像はデータソースとして登録しない。後続チャットの検索対象にも含めない。

## 画面での扱い

チャット画面では、入力欄の近くにファイル添付ボタンを表示する。

利用者が画像ファイルを選択した場合、送信前に添付画像のプレビューを表示する。利用者は送信前に添付画像を解除できる。

送信後のユーザ指示表示では、ユーザ指示本文と添付画像を同じメッセージ内で確認できるようにする。

過去チャットを再表示する場合も、保存済みの添付画像をユーザ指示表示内に表示する。

## 保存方針

添付画像はチャット実行に紐づけて保存する。

保存する主な情報:

- 添付画像ID。
- チャットID。
- 実行IDまたはジョブID。
- 利用者ID。
- 表示用ファイル名。
- MIME type。
- 保存先の論理パス。
- 作成日時。

生成用codex execのワーキングディレクトリ配下に入力添付用ディレクトリを作成し、そこへ添付画像を配置してもよい。ただし、履歴表示ではワーキングディレクトリ内のファイルを直接参照せず、アプリケーションが保存済み添付画像として管理するデータを参照する。

## codex exec連携

回答生成時は、保存済み添付画像を生成用codex execへ画像入力として渡す。

起動例:

```bash
codex exec --json --output-schema <回答スキーマ> \
  -i <保存済み添付画像のパス> \
  -C <対象セッションディレクトリ> \
  "<利用者のユーザ指示>"
```

複数画像を許可する場合は、画像ごとに `-i` を指定する。MVPでは画像添付を扱わないため、具体的な上限数は将来構想の外部設計または内部設計で定義する。

## 検証と制約

許可するファイル種別は画像に限定する。

後続設計で決めること:

- 許可する画像MIME type。
- 1ファイルあたりのサイズ上限。
- 1回のユーザ指示で添付できる画像数。
- 保存形式。
- サムネイル生成の有無。
- ウイルススキャンや画像メタデータ削除の扱い。

添付画像の保存先や表示URLでは、OS依存の絶対パスを利用者へ表示しない。
