# 構成拡張と設定例

## 目的

本メモは、将来構想要件定義から分離した技術構成の追加方針、Docker Compose構成の拡張、設定追加例を整理する。

## 技術構成の追加方針

将来構想では、本システムで採用した `FastAPI + React/Vite + PostgreSQL` 構成を維持する。Next.jsなど別のフルスタック構成へ置き換えるのではなく、FastAPI側に認証、ジョブ管理、監視機能を追加する。

## Docker Compose構成の拡張

将来構想では、本システムのDocker Compose構成を次のように拡張できる。

- アプリケーションコンテナ。
- PostgreSQLコンテナ。
- 必要に応じたバックグラウンドワーカー。
- 必要に応じたバックアップ用ジョブ。

開発時は引き続きDockerを必須にしない。開発者はFastAPIとReact/Viteをローカル起動し、PostgreSQLだけDocker ComposeまたはローカルDBを選べる。

## 設定追加例

将来構想では、本システムの設定ファイルに次の項目を追加する。

設定例:

```yaml
auth:
  required: true
  session_timeout_minutes: 60

database:
  url: "postgresql+psycopg://d_concierge:password@db:5432/d_concierge"

jobs:
  max_concurrent_jobs: 2
  queue_timeout_seconds: 600
  execution_timeout_seconds: 900
  allow_user_cancel: true

monitoring:
  enabled: true
  notify_on_consecutive_failures: 3

history:
  enable_pin: true
  enable_title_edit: true

deployment:
  development:
    require_docker: false
  production:
    recommended: "docker_compose"
    optional_worker: true
```

設定値は、アプリケーションごとに変更できる。
