# ロール・管理者機能

## 目的

本メモは、将来構想として検討するロールと管理者機能を整理する。

ユーザログイン、アカウント登録、ログインセッション、設定ダイアログ、アカウント削除の詳細は、`../ユーザログイン・アカウント管理.md` に定義する。

## ロール構想

ロールは、複数利用者による組織利用で、通常利用者と運用確認を行う利用者を区別するための概念である。

本システムからの追加ロール候補は次の通りである。

| ロール | 説明 |
| --- | --- |
| 一般ユーザ | チャットの作成、回答閲覧、自分の履歴検索を行う。 |
| 管理者 | 運用状態、ジョブ状態、エラー情報を確認する。 |

## 管理者機能構想

管理者は、運用調査に必要な範囲でジョブ状態、エラー状態、監視情報を確認する。

管理者画面では、秘密情報、絶対パス、環境変数、認証情報、不要な全文データを表示しない。エラー詳細はトレースログ全量ではなく、必要範囲の要約と分類を表示する。

管理者は、データソースそのものを画面から追加・更新しない。データソースの配置や `AGENTS.md`、Skillsの差し替えは、引き続き開発者または運用者がアプリケーション設定として行う。

## 対象外

ユーザログイン・アカウント管理の初期範囲では、ロールの概念を設けない。管理者ロール、管理者画面、管理者によるユーザ管理は初期範囲に含めない。
