# 認証とユーザ管理

## 目的

本メモは、将来構想要件定義から分離したログイン必須化、ユーザ情報、ロール管理、ログインセッション管理の詳細を整理する。

## 本システムからの主な変更点

| 項目 | 本システム | 将来構想 |
| --- | --- | --- |
| 利用者識別 | ローカル利用者ID単位 | ログインユーザ単位 |
| 履歴保存 | ローカル利用者ごとに分離 | ログインユーザごとに分離 |
| 画面表示 | 利用者名表示は任意 | ログインユーザ名を表示 |
| アクセス制御 | 必須ではない | 未ログイン利用を禁止 |

## ユーザ情報

将来構想では、少なくとも次のユーザ情報を保持する。

- ユーザID (ログイン識別子)
- ユーザ名
- ロール
- 最終ログイン日時

ロールは、本システムには存在しない新しい概念である。

本システムからの追加ロール:

| ロール | 説明 |
| --- | --- |
| 一般ユーザ | チャットの作成、回答閲覧、自分の履歴検索を行う。 |
| 管理者 | 運用状態、ジョブ状態、エラー情報を確認する。 |

ユーザ新規作成時のロールは「一般ユーザ」とし、ロールを変更する場合はスクリプトで直接DBを更新する。
管理者は、データソースそのものを画面から追加・更新しない。データソースの配置や `AGENTS.md`・Skillsの差し替えは、引き続き開発者または運用者がアプリケーション設定として行う。

## 認証・セッション管理

ログイン必須化に伴い、FastAPIが認証APIとセッション管理を担当する。

PostgreSQLには、少なくとも次の情報を保存する。

- ユーザ。
- ロール。
- ログインセッション。
- ユーザごとのチャット履歴。
- ユーザごとのピン留め、タイトル編集情報。

React/Vite側は、ログイン状態を確認し、未ログイン時はログイン画面を表示する。認証状態の最終判定はFastAPI側で行い、APIアクセス時に毎回確認する。
