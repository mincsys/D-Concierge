# revealAnswerモジュール設計

## 1. 文書の目的

本書は、`revealAnswer` モジュールの責務、不変条件、公開プロパティ、公開関数、および公開イベントを定義することを目的とする。

## 2. 前提

- 本書の対象は `モジュール一覧.md` で詳細設計対象とした `revealAnswer` モジュールのみとする。
- 本モジュールは表示演出の順序を制御し、回答ブロック本文の検証やMarkdown描画は行わない。

## 3. 責務

- 回答表示前に作業プロセスを閉じる遅延を入れる。
- 回答ブロックごとのMarkdown本文を表示用チャンクへ分割する。
- Mermaidコードブロックは分割せず、1チャンクとして扱う。
- 現行ストリームであることを都度確認し、古いストリームでは演出を中断する。
- 表示開始、途中回答、表示完了をコールバックで通知する。

## 4. 不変条件

- `isCurrent()` が `false` を返した後は、以降のコールバックを呼ばない。
- Mermaidコードブロックの内部は文字数チャンク分割しない。
- 最終的な `onAnswerComplete` には、入力された回答ブロックと参照元を欠落なく渡す。

## 5. 公開メソッド

| メソッド | 役割 | 入力 | 出力 | 事前条件 | 事後条件 |
| --- | --- | --- | --- | --- | --- |
| `revealSubmittedAnswer` | 回答表示演出を実行する | run ID、回答、現行判定、表示コールバック群 | 演出完了Promise | 回答ブロック配列が渡されていること<br>必要なコールバックが渡されていること | 現行ストリームであれば、作業プロセス完了、回答開始、回答ブロック本文の更新、回答完了が順に通知されること |
| `splitRevealChunks` | Markdownを表示チャンクへ分割する | Markdown文字列 | チャンク配列 | 入力が文字列であること | Mermaidコードブロックが1チャンクとして保持され、他の本文は短いチャンクへ分割されること |

## 6. 公開イベント

| イベント名 | 発火条件 | 通知内容 |
| --- | --- | --- |
| `onThoughtComplete` | 回答表示開始前の遅延が終わったとき | 作業プロセスを閉じる要求 |
| `onAnswerStart` | 回答ブロック本文のチャンク表示を開始するとき | run IDと初期回答 |
| `onAnswerChange` | チャンクが1つ表示対象に追加されたとき | run IDと表示中回答 |
| `onAnswerComplete` | 全チャンクの表示が完了したとき | run IDと完全な回答 |
