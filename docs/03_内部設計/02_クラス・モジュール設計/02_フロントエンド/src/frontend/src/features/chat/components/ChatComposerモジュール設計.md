# ChatComposerモジュール設計

## 1. 文書の目的

本書は、`ChatComposer` モジュールの責務、不変条件、公開プロパティ、公開関数、および公開イベントを定義することを目的とする。

## 2. 前提

- 本書の対象は `モジュール一覧.md` で詳細設計対象とした `ChatComposer` モジュールのみとする。
- ファイル添付ボタンと表示設定ボタンはMVPでは表示部品であり、添付処理や設定処理は持たない。

## 3. 責務

- ユーザ指示の入力欄を表示し、入力内容に応じてtextarea高さを調整する。
- `send`、`cancel`、`canceling` の動作モードに応じて主要アクションボタンの表示、無効化、イベントを切り替える。
- 非制御入力と制御入力の両方に対応する。
- Ctrl+EnterまたはCommand+Enterで送信できるようにする。
- 送信後、非制御入力の場合は入力値をクリアする。

## 4. 不変条件

- 空白だけのメッセージは送信しない。
- `actionMode` が `send` 以外の場合、キーボード送信は実行しない。
- `actionMode` が `canceling` の場合、主要アクションボタンは無効化する。
- textarea高さは最小値と最大値の範囲内に収める。
- `value` が渡された場合、内部状態ではなく `onValueChange` を通じて値を更新する。

## 5. 公開プロパティ

| 名前 | 型 | 役割 | 読み取り専用 |
| --- | --- | --- | --- |
| `actionMode` | `send`、`cancel`、`canceling` | 主要アクションの動作モードを決める | はい |
| `autoFocus` | 真偽値 | 初回描画時に入力欄へフォーカスするかを決める | はい |
| `className` | 文字列 | 外側formへ追加するclassName | はい |
| `focusSignal` | 数値または未指定 | 値が変わったとき入力欄へフォーカスするための信号 | はい |
| `value` | 文字列または未指定 | 制御入力として扱う現在値 | はい |
| `onSubmit` | 関数 | 送信成功時の通知先 | はい |
| `onCancel` | 関数または未指定 | キャンセル操作時の通知先 | はい |
| `onValueChange` | 関数または未指定 | 制御入力の値変更通知先 | はい |
| `onFocus` | 関数または未指定 | 入力欄フォーカス時の通知先 | はい |

## 6. 公開メソッド

| メソッド | 役割 | 入力 | 出力 | 事前条件 | 事後条件 |
| --- | --- | --- | --- | --- | --- |
| `ChatComposer` | 指示入力UIを描画する | 動作モード、入力値、イベントハンドラ | React要素 | `onSubmit` が渡されていること | 入力欄、補助ボタン、主要アクションボタンが表示されること |

## 7. 公開イベント

| イベント名 | 発火条件 | 通知内容 |
| --- | --- | --- |
| `onSubmit` | `send` モードで空でない指示を送信したとき | trim済みメッセージ |
| `onCancel` | `cancel` モードで主要アクションを選択したとき | キャンセル要求 |
| `onValueChange` | 制御入力として文字列が変更されたとき | 変更後メッセージ |
| `onFocus` | textareaがフォーカスされたとき | フォーカス通知 |
