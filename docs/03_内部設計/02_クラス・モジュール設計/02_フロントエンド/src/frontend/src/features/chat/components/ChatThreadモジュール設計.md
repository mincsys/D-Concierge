# ChatThreadモジュール設計

## 1. 文書の目的

本書は、`ChatThread` モジュールの責務、不変条件、公開プロパティ、公開関数、および公開イベントを定義することを目的とする。

## 2. 前提

- 本書の対象は `モジュール一覧.md` で詳細設計対象とした `ChatThread` モジュールのみとする。
- 本モジュールは受け取った `ChatSession` を表示し、run状態の更新やAPI通信は親コンテナへ委譲する。

## 3. 責務

- チャット内の全runについて、ユーザ指示、作業プロセス、回答、状態メッセージを表示する。
- 最新runの状態から、継続入力欄の送信/キャンセル/キャンセル中モードを決定する。
- 継続指示後に対象runへスクロールする。
- 回答内PDF参照元の選択、作業プロセスの開閉、キャンセル、継続指示を親へ通知する。

## 4. 不変条件

- runの表示順序は `session.runs` の順序を維持する。
- 各runの作業プロセスは、中間メッセージが空でも見出しを表示する。
- `statusMessage` はキャンセル要求中、キャンセル済み、エラー、タイムアウトのいずれでも同じ小さな枠付きメッセージとして表示する。
- キャンセル操作の対象は最新runに限定する。
- `受付`、`実行中`、`検証中` の最新runでは入力欄の主要操作をキャンセルにする。
- `キャンセル要求中` または `cancelingRunId` と一致する最新runでは入力欄をキャンセル中にする。
- スクロール対象処理が完了したら `onScrollTargetHandled` を呼び、親側の対象IDを解除できるようにする。

## 5. 公開プロパティ

| 名前 | 型 | 役割 | 読み取り専用 |
| --- | --- | --- | --- |
| `session` | チャットセッション | 表示対象チャットとrun一覧 | はい |
| `sidebarCollapsed` | 真偽値 | レイアウト幅の切り替えに使う | はい |
| `openThoughtRunIds` | run ID集合 | 開いている作業プロセスを判定する | はい |
| `cancelingRunId` | run IDまたは未指定 | キャンセル処理中runを判定する | はい |
| `scrollReserveRunId` | run IDまたは未指定 | 継続run用の最低高さ確保対象 | はい |
| `scrollTargetRunId` | run IDまたは未指定 | 自動スクロール対象run | はい |
| `onToggleThought` | 関数 | 作業プロセス開閉の通知先 | はい |
| `onOpenPdf` | 関数 | PDF参照元選択の通知先 | はい |
| `onCancelRun` | 関数 | キャンセル操作の通知先 | はい |
| `onScrollTargetHandled` | 関数 | スクロール完了の通知先 | はい |
| `onSubmitInstruction` | 関数 | 継続指示送信の通知先 | はい |

## 6. 公開メソッド

| メソッド | 役割 | 入力 | 出力 | 事前条件 | 事後条件 |
| --- | --- | --- | --- | --- | --- |
| `ChatThread` | チャットスレッドを描画する | チャットセッション、表示状態、イベントハンドラ | React要素 | `session.runs` が配列であること<br>必要なイベントハンドラが渡されていること | 全runの指示と結果が表示されること<br>下部に継続入力欄が固定表示されること |

## 7. 公開イベント

| イベント名 | 発火条件 | 通知内容 |
| --- | --- | --- |
| `onToggleThought` | 作業プロセス見出しを選択したとき | 対象run ID |
| `onOpenPdf` | 回答内参照元リンクを選択したとき | PDF参照元 |
| `onCancelRun` | 最新runのキャンセルボタンを選択したとき | 対象run ID |
| `onScrollTargetHandled` | 自動スクロール処理が完了したとき | 完了通知 |
| `onSubmitInstruction` | 継続入力欄から指示を送信したとき | trim済みメッセージ |
