# ディレクトリ構成

## 1. 文書の目的

本書は、D-Concierge MVPの内部設計におけるsource-controlledな管理対象ディレクトリを、ルートから末端まで定義することを目的とする。

本書では、実装コード、設計文書、Codex設定資産、テストコード、フロントエンド単体確認用モックを対象にする。生成物、依存ライブラリ、セッション作業領域、データソース実体、トレースログ実体は対象外とする。

## 2. 前提

- 本書に記載するのはsource-controlledな管理対象ディレクトリのみとする。
- `.venv/`、`.uv-cache/`、`src/frontend/node_modules/`、`src/frontend/dist/`、`src/frontend/tsconfig.tsbuildinfo`、`output/playwright/` は管理対象外とする。
- `codex/readonly/`、`codex/sessions/`、`codex/sessions_validator/`、`codex/saved_artifacts/` は実行時または用途別データ配置であり、source-controlledな管理対象外とする。
- `src/frontend/backend_mock/` はフロントエンド単体確認用の開発支援資産として管理するが、アプリ本体のクラス・モジュール設計対象から除外する。
- `docs/03_内部設計/02_クラス・モジュール設計/` は、バックエンドのクラス設計とフロントエンドのモジュール設計を同じカテゴリで管理する。

## 5. 命名と所有ルール

- バックエンドの実装ディレクトリ名は英小文字とアンダースコアを使用する。
- バックエンドの副作用実装は `infrastructure/` 配下へ置き、`domain/` から直接参照しない。
- バックエンドの抽象IFは `application/ports/` へ置く。
- バックエンドのテストコードは `src/backend/tests/` 配下へ置く。
- フロントエンドの機能固有コードは `src/frontend/src/features/` 配下へ置く。
- フロントエンドのテストコードは `src/frontend/tests/` 配下へ置く。
- `src/frontend/src/styles/globals.css` には、Tailwind/shadcnテーマ、body初期設定、外部ライブラリ生成DOM向けスタイルだけを置く。
- `src/frontend/backend_mock/` は `/api/...` 境界を通じたフロントエンド単体確認用資産として維持し、UIコンポーネントから直接参照しない。
