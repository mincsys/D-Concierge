# 回答検証・再生成処理設計

## 1. 文書の目的

本書は、回答検証・再生成処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: 生成用codex execが最終回答候補を返したとき。
- 関連文書: `docs/03_内部設計/03_内部IF設計/Codex実行IF.md`、`docs/03_内部設計/03_内部IF設計/参照元ファイルIF.md`

## 3. 処理概要

- 回答候補の固定検証を行い、参照元形式と成果物参照の基本制約を確認する。
- Codex出力スキーマの `locator.start_page` / `locator.end_page` を、内部・API・DBで使用する `locator.page_start` / `locator.page_end` へ正規化する。
- 検証用codex execで参照元整合性を確認する。
- 検証失敗時は再生成上限まで生成へ戻し、上限到達時はエラー終端へ返す。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `ValidateAnswerUseCase` | 検証順序と再生成判断 |
| `AnswerCandidate` | 回答候補の構造制約 |
| `RetryPolicy` | 再生成可否判断 |
| `CodexRunner` | 検証用codex exec実行 |
| `FileReferenceStore` | PDF参照元の安全確認 |
| `TraceLogWriter` | 検証失敗理由と再生成回数の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | 回答候補JSONを構文解析し、Codex出力スキーマに適合することを確認する。 |
| 2 | Codex出力の `locator.start_page` / `locator.end_page` を `locator.page_start` / `locator.page_end` へ正規化し、`AnswerCandidate` として構築する。 |
| 3 | 必須項目、空でないMarkdown本文、`source_type = pdf`、PDF参照元、ページ範囲、成果物参照形式、危険HTMLを固定検証する。 |
| 4 | 参照元ファイルIFでPDF参照元のパス安全性と存在を確認する。 |
| 5 | 検証用codex execへ現在の回答候補と参照元情報を渡す。 |
| 6 | 検証合格なら採用済み回答として返す。 |
| 7 | 検証不合格で再生成可能なら `comment` または固定検証理由を生成プロンプトへ渡して再生成する。 |
| 8 | 再生成上限到達時は検証失敗として呼出元へ返す。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- 生成用codex execから最終回答候補が取得できている。
- 検証用Codexホームと検証プロンプトが利用可能である。
- 再生成上限が設定されている。

### 6.2. 事後条件

- 合格時は採用可能な回答本文、参照元、成果物候補が返る。
- 不合格時は再生成理由または終端エラー理由が返る。

### 6.3. 不変条件

- 固定検証を通過しない回答候補は検証用codex execへ渡さない。
- 検証不合格の回答を採用済み回答として保存しない。
- 再生成回数は設定上限を超えない。
- 検証用Codexの `valid` / `comment` は処理中の一時判定結果であり、DBへ永続化しない。
- 検証用Codexへresumeを使う場合でも、現在の回答候補と参照元情報を検証対象の正として渡し、過去の検証文脈だけに依存しない。
- 固定検証通過後の参照元locatorは `page_start` / `page_end` で扱い、`start_page` / `end_page` を内部・API・DBへ持ち込まない。

## 7. 副作用

| 副作用 | 発生箇所 | 補足 |
| --- | --- | --- |
| codex exec起動 | CodexRunner | 検証用Codexホームを使用 |
| ファイル読込 | FileReferenceStore | PDF参照元確認 |
| ログ出力 | TraceLogWriter | 固定検証結果、参照元検証結果、再生成回数 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 回答候補JSON不正 | 検証失敗とし、再生成可能なら再生成する |
| 出力スキーマ不一致 | 検証失敗とし、再生成可能なら再生成する |
| 空回答 | 検証失敗とし、再生成可能なら再生成する |
| `source_type` が `pdf` 以外 | 検証失敗とし、再生成可能なら再生成する |
| ページ範囲不正 | 検証失敗とし、再生成可能なら再生成する |
| 危険HTML検知 | 検証失敗とし、再生成可能なら再生成する |
| 参照元が存在しない | 検証失敗とし、再生成可能なら再生成する |
| 検証用codex exec起動失敗 | 検証処理エラーとして呼出元へ返す |
| 再生成上限到達 | runをエラー終端できる失敗結果を返す |

## 9. ログ出力

| タイミング | ログ内容 |
| --- | --- |
| 固定検証開始 | trace_id、run_id、回答候補ID |
| 固定検証失敗 | trace_id、run_id、失敗項目 |
| 参照元検証失敗 | trace_id、run_id、参照元ID、理由 |
| 再生成判断 | trace_id、run_id、再生成回数、上限、検証失敗理由、検証用Codexのcomment要約 |
| 検証合格 | trace_id、run_id |

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 本処理単独ではDBトランザクションを開始しない |
| 終了点 | 呼出元の回答保存処理へ検証結果を返す時点 |
| ロールバック条件 | 本処理内の副作用はcodex execとファイル読込であり、DB rollback対象を持たない |
