# 処理一覧

## 1. 文書の目的

本書は、D-Conciergeで設計対象とする主要処理を一覧化し、個別処理設計への入口を提供することを目的とする。

## 2. 前提

- 対象は、内部実装として手順、副作用、例外、トランザクションを設計する必要がある処理とする。
- 画面表示仕様やHTTP IF項目の全文は外部設計書で扱い、本書では処理順序、責務境界、状態更新、ログ出力を扱う。
- バックエンド処理はAPI/SSEエンドポイント単位を入口にしつつ、非同期実行や検証の内部処理も独立した処理として扱う。

## 3. 収集対象

| 観点 | 方針 |
| --- | --- |
| 1 処理の単位 | 利用者操作、API/SSE受付、または内部ジョブを起点として、状態更新や副作用のまとまりが完結する単位 |
| 詳細ファイルへ展開する条件 | DB更新、Codex実行、SSE配信、ファイル操作、トレースログ、トランザクション制御を伴う処理 |
