# 継続指示受付処理設計

## 1. 文書の目的

本書は、継続指示受付処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: 利用者が既存チャット画面で継続指示を送信する。
- 関連文書: `docs/03_内部設計/03_内部IF設計/チャットRepositoryIF.md`、`docs/03_内部設計/03_内部IF設計/RunExecutionDispatcherIF.md`

## 3. 処理概要

- 既存チャットの存在、通常操作対象であること、未完了runの有無を確認し、継続指示と新しいrunを保存する。
- 受付応答にはチャットID、run ID、SSE URL、受付状態を含める。
- REST受付成功後、`RunExecutionDispatcher` へ `ExecuteChatRunUseCase.execute(chat_id, run_id, trace_id)` をバックグラウンド登録する。
- dispatcher登録失敗時は対象runを `エラー` へ更新し、受付APIはHTTP 500へ変換する。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `AppendChatRunUseCase` | 継続指示受付と未完了run確認 |
| `ChatRepository` | チャット存在確認、未完了run検索、指示/run保存 |
| `RunExecutionDispatcher` | 受付済みrunのバックグラウンド登録 |
| `RunStatePolicy` | 未完了run判定 |
| `Runtime Provider` | run ID、時刻の生成 |
| `TraceLogWriter` | 例外の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | REST境界でチャットIDと指示本文を検証する。 |
| 2 | Repositoryで対象チャットを取得し、通常操作対象であることを確認する。 |
| 3 | 対象チャットに未完了runが存在する場合は受付を拒否する。 |
| 4 | run IDと受付日時を生成する。 |
| 5 | Repositoryトランザクション内で継続指示と受付済みrunを保存する。 |
| 6 | `RunExecutionDispatcher` へチャットID、run ID、trace_idを渡し、チャット実行処理をバックグラウンド登録する。 |
| 7 | dispatcher登録成功後にSSE URLを生成し、チャットIDを含む受付応答を返す。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- 対象チャットが存在する。
- 対象チャットが通常操作対象である。
- 対象チャットに未完了runが存在しない。
- 継続指示本文が空でない。

### 6.2. 事後条件

- 既存チャットに継続指示と受付済みrunが追加される。
- 応答には対象チャットID、run ID、SSE URL、`受付` 状態が含まれる。
- dispatcher登録失敗時は、保存済みrunが `エラー` へ更新され、受付応答は返さない。

### 6.3. 不変条件

- 同一チャットで複数の未完了runを同時に作らない。
- 継続指示は既存チャットのsession IDを引き継ぐ。
- 未完了runの排他は事前確認だけでなくDBの部分UNIQUE制約で保証する。
- dispatcher登録の成否は受付処理の一部として扱い、登録できないrunを `受付` のまま残さない。

## 7. 副作用

| 副作用 | 発生箇所 | 補足 |
| --- | --- | --- |
| DB読込 | Repository | チャット存在確認、未完了run確認 |
| DB書込 | Repository | 継続指示、run保存 |
| バックグラウンド登録 | RunExecutionDispatcher | `ExecuteChatRunUseCase.execute` をrun ID単位で登録する |
| ID発番 | Runtime Provider | run ID |
| 時刻取得 | Runtime Provider | 受付日時、最終更新日時 |
| ログ出力 | TraceLogWriter | 失敗 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| チャットが存在しない | HTTP 404へ変換する |
| 対象チャットが`削除中` | 競合としてHTTP 409へ変換する |
| 未完了runが存在する | 競合としてHTTP 409へ変換する |
| 指示本文が空 | 入力不正としてHTTP 400へ変換する |
| 未完了run一意制約違反 | 競合としてHTTP 409へ変換する |
| dispatcher登録失敗 | 保存済みrunを `エラー` へ更新し、システムエラーとしてHTTP 500へ変換する |
| DB保存失敗 | rollbackし、HTTP 500へ変換する |

## 9. ログ出力

| タイミング | ログ内容 |
| --- | --- |
| dispatcher登録失敗 | trace_id、chat_id、run_id、失敗理由 |
| 例外発生 | trace_id、エラー分類、処理段階、例外種別、スタックトレース |

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 未完了run確認後、継続指示とrun保存を開始する時点 |
| 終了点 | 継続指示とrun保存完了 |
| ロールバック条件 | DB制約違反、保存失敗、更新対象消失 |
