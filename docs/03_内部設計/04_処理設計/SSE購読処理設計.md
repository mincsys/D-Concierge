# SSE購読処理設計

## 1. 文書の目的

本書は、SSE購読処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: フロントエンドが受付応答のSSE URLへ接続する。
- 本処理は認証済みユーザだけが呼び出せる。
- 関連文書: `docs/03_内部設計/03_内部IF設計/SSEイベント配信IF.md`

## 3. 処理概要

- 認証済みユーザが所有する通常操作対象のチャットに属する対象runの現在状態を取得し、SSE接続へ初期状態を送信する。
- 以降、run ID単位のイベントを購読し、状態、中間メッセージ、回答、エラー、キャンセルを配信する。
- SSE送信ループは非同期ストリームとして構成し、イベント未到着時はブロッキング待機せず、短周期で接続切断を確認する。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `presentation/sse` | SSE接続、イベント形式変換、切断検知 |
| `presentation/rest` | 認証依存関係、trace_id生成 |
| `SubscribeRunEventsUseCase` | 対象run確認と現在状態取得 |
| `RunEventBroker` | publish/subscribe |
| `ChatRepository` | 現在状態と保存済み結果の取得 |
| `TraceLogWriter` | 異常の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | SSE境界でtrace_idを生成し、認証依存関係で現在ユーザを取得する。 |
| 2 | チャットIDとrun IDを検証する。 |
| 3 | Repositoryで認証済みユーザが所有する対象チャットが通常操作対象であることと、対象runの現在状態を取得する。 |
| 4 | `RunEventBroker` へ購読登録する。 |
| 5 | 現在状態をSSE `state` として送信する。 |
| 6 | イベントキューを非ブロッキングで確認し、未到着時は短周期の非同期待機へ戻る。 |
| 7 | publishされたイベントを受け取り、SSE形式でブラウザへ送信する。 |
| 8 | 終端イベントまたは接続切断で購読を解除する。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- 対象チャットとrunが存在する。
- 認証済みユーザIDが確定している。
- 対象チャットが認証済みユーザに紐づく通常操作対象である。
- SSE接続が確立できる。

### 6.2. 事後条件

- 接続開始時に現在状態が1回送信される。
- 終端イベント送信後に接続を閉じられる状態になる。

### 6.3. 不変条件

- 購読解除後の接続へイベントを送信しない。
- SSE payloadは画面バックエンドAPI IFに定義された形式に変換する。
- 接続切断はrun実行処理のキャンセル理由にしない。
- イベント未到着のSSE接続はREST API処理用のワーカーを占有し続けない。

## 7. 副作用

| 副作用 | 発生箇所 | 補足 |
| --- | --- | --- |
| DB読込 | Repository | 対象runと現在状態 |
| SSE送信 | presentation/sse | state、message、answer、error、canceled |
| メモリ上購読登録 | RunEventBroker | run ID単位 |
| ログ出力 | TraceLogWriter | 異常 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 未ログイン | HTTP 401相当として接続前に失敗させる |
| 対象runなし | HTTP 404相当としてSSE接続前に失敗させる |
| 対象チャットが`削除中` | HTTP 409相当として接続前に失敗させる |
| 接続切断 | 購読解除し、正常な切断として扱う |
| イベント変換失敗 | 接続を閉じ、traceログへ記録する |
| Broker内部エラー | 接続を閉じ、HTTP/SSEエラーとして扱う |

## 9. ログ出力

| タイミング | ログ内容 |
| --- | --- |
| 例外発生 | trace_id、chat_id、run_id、エラー分類、例外種別、スタックトレース |

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 本処理ではDB更新トランザクションを開始しない |
| 終了点 | 現在状態取得の読取完了 |
| ロールバック条件 | DB更新を行わないため該当なし |
