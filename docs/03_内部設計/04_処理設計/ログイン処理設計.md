# ログイン処理設計

## 1. 文書の目的

本書は、ログイン処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: 利用者がログイン画面でログインボタンを押し、フロントエンドが `POST /api/auth/login` を呼び出す。
- ログイン成功時はログインセッションCookieを設定する。
- 既存ログインセッションCookieが含まれる場合は、既存セッション行を削除して新しいセッションを発行する。

## 3. 処理概要

- REST境界でリクエストJSONをPydanticスキーマへ変換する。
- `LoginUseCase` がユーザIDとパスワードを検証する。
- 認証成功時、新しいログインセッションを作成する。
- Cookieへ設定するログインセッショントークンを応答へ付与し、現在ユーザを返す。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `account_router` | `POST /api/auth/login` のREST境界 |
| `LoginUseCase` | ログイン処理の調停 |
| `PasswordHasherPort` | パスワード検証 |
| `SessionTokenProviderPort` | セッショントークン発行と照合用ハッシュ生成 |
| `AccountRepositoryPort` | ユーザ取得、既存セッション削除、新規セッション作成 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | REST境界でtrace_idを発行し、リクエストJSONを検証する。 |
| 2 | Cookieに既存ログインセッショントークンがあれば取得する。 |
| 3 | `LoginUseCase` を呼び出す。 |
| 4 | Repositoryからログイン用ユーザ情報を取得する。 |
| 5 | ユーザなしの場合はユーザID項目の入力エラー、`deleting`の場合はユーザID項目の利用不可エラー、パスワード不一致の場合はパスワード項目の入力エラーを返す。 |
| 6 | 新しいログインセッショントークンを発行し、照合用ハッシュを生成する。 |
| 7 | トランザクション内で既存Cookieに対応するセッションがあれば削除し、新規ログインセッションを作成する。 |
| 8 | commit後、Cookie設定とユーザ情報をAPI応答へ変換する。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- ユーザIDとパスワードがリクエストに含まれている。
- Repository、Hasher、TokenProvider、Clockが注入済みである。

### 6.2. 事後条件

- 認証成功時、`login_sessions` に新しいセッション行が作成される。
- Cookieへログインセッショントークンが設定される。
- 応答としてユーザIDとユーザ名を返す。

### 6.3. 不変条件

- ユーザなし、`deleting`、パスワード不一致はAPI項目名に対応する項目別エラーとして返す。
- パスワード生値とCookieトークン生値をDBへ保存しない。
- 認証失敗時は新しいセッションを作成しない。

## 7. 副作用

| 種別 | 対象 | 内容 |
| --- | --- | --- |
| DB登録 | `login_sessions` | 新規ログインセッション作成 |
| DB削除 | `login_sessions` | 既存Cookieに対応するセッション削除 |
| Cookie設定 | ブラウザ | ログインセッショントークンを設定 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 入力形式不正 | 項目別入力エラーとして返す |
| ユーザなし | `user_id` の項目別入力エラーとして返す |
| ユーザが`deleting` | `user_id` の項目別入力エラーとして返す |
| パスワード不一致 | `password` の項目別入力エラーとして返す |
| ハッシュ検証失敗 | システムエラーとして返し、トレースログへ記録する |
| DB保存失敗 | システムエラーとして返し、トレースログへ記録する |

## 9. ログ出力

入力不正と認証失敗はトレースログへ記録しない。ハッシュ検証失敗、DB保存失敗、予期しない例外をtrace_id付きで記録する。

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 認証成功後のセッション更新開始時 |
| 終了点 | 既存セッション削除と新規セッション作成commit時 |
| rollback条件 | セッション削除またはセッション作成でDB例外が発生した場合 |
| トランザクション外処理 | Cookie設定、HTTP応答変換 |
