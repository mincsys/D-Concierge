# チャット物理削除処理設計

## 1. 文書の目的

本書は、チャット物理削除処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: `ChatDeletionDispatcher` が削除対象チャットをバックグラウンド実行へ登録したとき。
- 削除受付処理により、対象チャットは`deleting`へ更新済みである。
- 本処理は、実行中runの終了待ち、生成用・検証用作業領域削除、保存済み成果物実体削除、DBカスケード削除を担当する。
- 未完了runへの終了要求は、チャット実行処理と同じ `CodexRunner` の実行中プロセス登録を参照できる `CancelRequesterPort` をcomposition rootから受け取る。
- トレースログと共有データソース実体は削除対象外とする。

## 3. 処理概要

- Repositoryから `ChatDeletionTarget` を取得する。
- 未完了runが存在する場合、キャンセル要求または状態整合を行い、runが終端状態になるまで物理ファイル削除へ進まない。
- 終端状態確認後、生成用・検証用セッション作業領域を削除する。
- 保存済み成果物メタ情報の `storage_path` に対応する成果物実体と空の親セッションディレクトリを削除する。
- ファイル削除が完了した後、DB上のチャット一式をカスケード削除する。
- 途中失敗した場合は対象チャットを`deleting`のまま維持し、起動時再登録で再試行できるようにする。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `ChatDeletionDispatcher` | 削除ジョブの非同期登録 |
| `ExecuteChatDeletionUseCase` | 物理削除処理の調停 |
| `ChatRepository` | 削除対象取得、run状態取得、DBカスケード削除 |
| `CancelRequesterPort` | 実行中Codex実行コンテナへの終了要求 |
| `SessionWorkdirCleanupPort` | 生成用・検証用セッション作業領域削除 |
| `SavedArtifactDeletionPort` | 保存済み成果物実体削除 |
| `TraceLoggerPort` | 物理削除失敗の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | `ExecuteChatDeletionUseCase` がチャットIDとtrace_idを受け取る。 |
| 2 | Repositoryから `ChatDeletionTarget` を取得する。対象なしの場合は削除済みとして正常終了する。 |
| 3 | `ChatDeletionTarget.unfinished_runs` を確認する。 |
| 4 | 未完了runが存在し、終了要求が必要な状態の場合は `CancelRequesterPort` へ終了要求を送る。 |
| 5 | 未完了runが残っている間は、作業領域削除と成果物削除へ進まず、削除ジョブを終了する。対象チャットは`deleting`のままとし、起動時再登録または次回登録で再開する。 |
| 6 | 未完了runが存在しない場合、`SessionWorkdirCleanupPort` で生成用・検証用作業領域を削除する。 |
| 7 | `SavedArtifactDeletionPort` で保存済み成果物実体と空の親セッションディレクトリを削除する。 |
| 8 | Repositoryトランザクション内でチャット、利用者メッセージ、run、中間メッセージ、回答、参照元、成果物メタ情報をカスケード削除する。 |
| 9 | 正常終了時は削除ジョブの登録状態を解除する。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- 対象チャットは`deleting`である。
- DB削除前に、作業領域削除と成果物削除に必要な情報をRepositoryから取得できる。
- `CancelRequesterPort`、作業領域削除、成果物削除のPortがcomposition rootで注入済みである。

### 6.2. 事後条件

- 正常終了時、対象チャットに紐づくDBデータは削除される。
- 正常終了時、対象チャットの生成用・検証用セッション作業領域は削除される。
- 正常終了時、対象チャットの保存済み成果物メタ情報に対応する成果物実体は削除される。
- 途中失敗時、対象チャットは`deleting`のまま維持される。

### 6.3. 不変条件

- 未完了runが存在する間は、作業領域と保存済み成果物実体を削除しない。
- DBカスケード削除は、作業領域削除と保存済み成果物削除が完了した後に実行する。
- 作業領域削除では共有データソース実体を削除しない。
- 保存済み成果物削除では `generator.saved_artifacts_dir` 配下の対象ファイルと空の親セッションディレクトリだけを削除する。
- ファイルまたはディレクトリが存在しない場合は、再試行時の冪等性として削除済み扱いにする。
- トレースログは削除対象に含めない。

## 7. 副作用

| 種別 | 対象 | 内容 |
| --- | --- | --- |
| コンテナ終了要求 | 生成用・検証用Codex実行コンテナ | 未完了runの終了要求 |
| ファイル削除 | 生成用・検証用作業領域 | `<workdir>/<user-id>/<session-id>` の削除 |
| ファイル削除 | 保存済み成果物領域 | 対象 `storage_path` と空の親セッションディレクトリの削除 |
| DB削除 | チャット関連テーブル | 対象チャット一式のカスケード削除 |
| ログ出力 | トレースログ | 物理削除失敗、パス安全検証失敗、DB削除失敗 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 削除対象なし | すでに削除済みとして正常終了する |
| 未完了runあり | 必要に応じて終了要求を送り、ファイル削除とDB削除へ進まず処理を終了する |
| 終了要求結果が `not_registered` | 準正常系として扱い、未完了runのDB状態確認を優先する |
| 作業領域削除失敗 | トレースログへ記録し、DB削除へ進まない |
| 保存済み成果物削除失敗 | トレースログへ記録し、DB削除へ進まない |
| DBカスケード削除失敗 | トレースログへ記録し、対象チャットを`deleting`のまま維持する |
| ファイルまたはディレクトリなし | 削除済みとして扱い、次工程へ進む |

## 9. ログ出力

| 条件 | 内容 |
| --- | --- |
| 作業領域削除失敗 | trace_id、chat_id、削除対象種別、正規化済みパス、例外要約 |
| 保存済み成果物削除失敗 | trace_id、chat_id、storage_path、例外要約 |
| DBカスケード削除失敗 | trace_id、chat_id、例外種別、例外要約 |

通常の対象なし、ファイルなし、ディレクトリなしはトレースログへ記録しない。

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 削除対象取得 | 読取専用トランザクションまたは短いセッションで取得する |
| DBカスケード削除開始点 | 作業領域削除と保存済み成果物削除が完了した後 |
| DBカスケード削除終了点 | 対象チャット一式の削除commit時 |
| rollback条件 | DB削除中に例外が発生した場合 |
| トランザクション外処理 | Codex実行コンテナ終了要求、作業領域削除、保存済み成果物削除、トレースログ出力 |
