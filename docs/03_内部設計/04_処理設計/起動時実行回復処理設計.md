# 起動時実行回復処理設計

## 1. 文書の目的

本書は、アプリ起動時に未完了runを回復または終端させ、`削除中`のチャットを物理削除へ再登録する内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: FastAPIアプリ起動時。
- 関連文書: `docs/03_内部設計/03_内部IF設計/RunExecutionDispatcherIF.md`、`docs/03_内部設計/03_内部IF設計/ChatDeletionDispatcherIF.md`、`docs/03_内部設計/03_内部IF設計/チャットRepositoryIF.md`

## 3. 処理概要

- DBに残った未完了runを取得する。
- `受付` は再登録し、`実行中` / `検証中` はプロセス喪失として `エラー` に整合する。
- `キャンセル要求中` はプロセス終了要求なしで `キャンセル済み` に整合する。
- `削除中`のチャットを取得し、`ChatDeletionDispatcher` へ再登録する。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `RunExecutionDispatcher` | `受付` runの再登録 |
| `ChatDeletionDispatcher` | `削除中`チャットの再登録 |
| `ExecuteChatRunUseCase` | 再登録後の実行本体 |
| `ChatRepository` | 未完了run取得、状態条件付き更新 |
| `TraceLogWriter` | 回復失敗理由の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | Repositoryで `受付`、`実行中`、`検証中`、`キャンセル要求中` のrunを取得する。 |
| 2 | `受付` のrunを `RunExecutionDispatcher` へ再登録する。 |
| 3 | `受付` の再登録に失敗した場合は、状態条件付き更新で `エラー` へ更新する。 |
| 4 | `実行中` / `検証中` のrunは、対応するcodexプロセスを失ったものとして `エラー` へ更新する。 |
| 5 | `キャンセル要求中` のrunは、プロセス終了要求を行わず `キャンセル済み` へ更新する。 |
| 6 | Repositoryで`削除中`のチャットIDを取得する。 |
| 7 | 取得したチャットIDを `ChatDeletionDispatcher` へ再登録する。 |
| 8 | 回復件数、終端件数、削除再登録件数、失敗件数をトレースログへ記録する。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- DB接続、dispatcher、トレースログが利用可能である。

### 6.2. 事後条件

- 起動時点で取り残されていた `受付` runは、再登録済みまたは `エラー` になっている。
- プロセスを失った `実行中` / `検証中` runは `エラー` になっている。
- `キャンセル要求中` runは `キャンセル済み` になっている。
- `削除中`のチャットは削除ジョブへ再登録済み、または次回起動で再登録可能な状態になっている。

### 6.3. 不変条件

- 回復処理は終端済みrunを変更しない。
- 同一runを複数回dispatcherへ登録しない。
- 回復処理中の個別run失敗は、他runの回復処理を妨げない。
- 削除ジョブ再登録失敗は対象チャットを通常履歴へ戻さない。

## 7. 副作用

| 副作用 | 発生箇所 | 補足 |
| --- | --- | --- |
| DB読込/更新 | Repository | 未完了run取得、状態更新 |
| バックグラウンド登録 | RunExecutionDispatcher | `受付` runの再登録 |
| バックグラウンド登録 | ChatDeletionDispatcher | `削除中`チャットの再登録 |
| ログ出力 | TraceLogWriter | 失敗 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 未完了run取得失敗 | 起動時エラーとして扱い、アプリ起動失敗または管理者対応対象にする |
| `受付` run再登録失敗 | 対象runを `エラー` へ更新し、次のrunを処理する |
| 削除ジョブ再登録失敗 | トレースログへ記録し、次のチャットを処理する |
| 状態条件付き更新不成立 | 他処理で状態が進んだものとして、現在状態を正にする |

## 9. ログ出力

| タイミング | ログ内容 |
| --- | --- |
| 回復失敗 | trace_id、run_id、失敗理由 |
| 削除再登録失敗 | trace_id、chat_id、失敗理由 |

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 各runの状態更新直前 |
| 終了点 | 各runの状態更新完了ごと |
| ロールバック条件 | 個別runのDB更新失敗時。dispatcher登録済みのrunは状態条件付き更新により二重終端を防ぐ |
