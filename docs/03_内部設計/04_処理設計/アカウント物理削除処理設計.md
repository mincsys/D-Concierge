# アカウント物理削除処理設計

## 1. 文書の目的

本書は、アカウント物理削除処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: `AccountDeletionDispatcher` が削除対象ユーザIDをバックグラウンド実行へ登録する。
- 対象ユーザはアカウント削除受付処理で`deleting`へ更新済みである。
- 未完了runが残る場合は物理削除を延期し、次回起動時または次回登録時に再試行する。

## 3. 処理概要

- Repositoryから削除対象ユーザのチャット、未完了run、Codex作業領域、保存済み成果物参照を取得する。
- 未完了runに終了要求を送る。
- 未完了runが終端していない場合、ファイル削除とDB削除へ進まない。
- 生成用・検証用のユーザ単位作業領域を削除する。
- 保存済み成果物のユーザ単位ディレクトリを削除する。
- ファイル削除完了後、対象ユーザに紐づくDBデータを削除する。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `AccountDeletionDispatcher` | 削除ジョブ登録 |
| `ExecuteAccountDeletionUseCase` | 物理削除の調停 |
| `AccountRepositoryPort` | 削除対象取得、DBデータ削除 |
| `CancelRequesterPort` | 未完了runのCodex実行コンテナ終了要求 |
| `SessionWorkdirCleanupPort` | ユーザ単位の生成用・検証用作業領域削除 |
| `SavedArtifactDeletionPort` | ユーザ単位の保存済み成果物削除 |
| `TraceLoggerPort` | 物理削除失敗の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | `ExecuteAccountDeletionUseCase` をユーザIDとtrace_idで呼び出す。 |
| 2 | Repositoryからアカウント削除対象DTOを取得する。 |
| 3 | 対象ユーザが存在しない場合は物理削除済みとして終了する。 |
| 4 | 未完了runが存在する場合、各runへ終了要求を送る。 |
| 5 | 未完了runが終端していない場合、削除を延期して終了する。 |
| 6 | `codex/sessions/<user-id>/` を削除する。 |
| 7 | `codex/sessions_validator/<user-id>/` を削除する。 |
| 8 | `codex/saved_artifacts/<user-id>/` を削除する。 |
| 9 | ファイル削除完了後、Repositoryトランザクション内で対象ユーザに紐づくDBデータを削除する。 |
| 10 | 物理削除失敗時はトレースログへ記録し、対象ユーザを`deleting`のまま維持する。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- 対象ユーザIDが削除ジョブとして渡されている。
- Repository、CancelRequester、作業領域削除IF、成果物削除IF、TraceLoggerが注入済みである。

### 6.2. 事後条件

- 未完了runがない場合、対象ユーザの作業領域、保存済み成果物、DBデータが削除される。
- 未完了runが残る場合、対象ユーザは`deleting`のまま残る。
- 失敗時は次回起動時に再登録できる状態が維持される。

### 6.3. 不変条件

- 未完了runが残る場合、ファイル削除とDB削除へ進まない。
- ファイル削除完了前にDBデータを削除しない。
- 共有データソース実体とトレースログ実体を削除しない。
- 物理削除失敗時に対象ユーザを通常状態へ戻さない。

## 7. 副作用

| 種別 | 対象 | 内容 |
| --- | --- | --- |
| コンテナ終了要求 | Codex実行コンテナ | 未完了runへの終了要求 |
| ファイル削除 | `codex/sessions/<user-id>/` | 生成用作業領域削除 |
| ファイル削除 | `codex/sessions_validator/<user-id>/` | 検証用作業領域削除 |
| ファイル削除 | `codex/saved_artifacts/<user-id>/` | 保存済み成果物削除 |
| DB削除 | 対象ユーザ関連データ | ユーザ、チャット、run、回答、参照元、成果物メタ情報の削除 |
| ログ出力 | トレースログ | 物理削除失敗の調査情報 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 削除対象なし | 物理削除済みとして正常終了する |
| 未完了runあり | 終了要求後、物理削除を延期する |
| 終了要求結果が `not_registered` | 準正常系として扱い、未完了runのDB状態確認を優先する |
| 作業領域削除失敗 | トレースログへ記録し、DB削除へ進まない |
| 保存済み成果物削除失敗 | トレースログへ記録し、DB削除へ進まない |
| DB削除失敗 | トレースログへ記録し、対象ユーザを`deleting`のまま維持する |

## 9. ログ出力

作業領域削除失敗、保存済み成果物削除失敗、DB削除失敗、予期しない例外をtrace_id付きで記録する。未完了runが残ること自体は通常の延期として扱い、原則としてトレースログへ記録しない。

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | ファイル削除完了後のDBデータ削除開始時 |
| 終了点 | 対象ユーザ関連DBデータ削除commit時 |
| rollback条件 | DB削除で例外が発生した場合 |
| トランザクション外処理 | Codex実行コンテナ終了要求、作業領域削除、保存済み成果物削除、トレースログ出力 |
