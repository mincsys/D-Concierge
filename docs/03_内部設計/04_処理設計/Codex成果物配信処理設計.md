# Codex成果物配信処理設計

## 1. 文書の目的

本書は、Codex成果物配信処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: 回答Markdown内の保存済み成果物URLをブラウザが取得する。
- 関連文書: `docs/03_内部設計/03_内部IF設計/成果物ファイルIF.md`

## 3. 処理概要

- 保存済み成果物IDからメタ情報を取得し、採用済み成果物領域内のファイルだけを配信する。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `GetArtifactUseCase` | 成果物配信の調停 |
| `ChatRepository` | 成果物メタ情報取得 |
| `FileArtifactStore` | 保存済み成果物読込 |
| `PathSecurityService` | 保存領域内パス確認 |
| `TraceLogWriter` | 取得開始、拒否、成功、例外の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | REST境界で成果物IDを検証する。 |
| 2 | Repositoryで成果物メタ情報を取得する。 |
| 3 | FileArtifactStoreで保存済み成果物領域内のファイルを解決する。 |
| 4 | MIMEタイプと保存済みファイル実体を確認する。 |
| 5 | 保存済み成果物をストリーミング応答として返す。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- 成果物IDが採用済み回答に紐付いている。
- 保存済み成果物ファイルが存在する。

### 6.2. 事後条件

- 許可済みMIMEタイプの成果物内容だけが返る。
- 対象が存在しない場合や許可外の場合はファイル内容を返さない。

### 6.3. 不変条件

- Codex作業領域内の未採用ファイルを直接配信しない。
- 保存済み成果物領域外のファイルを配信しない。
- MIMEタイプは保存時メタ情報と配信時実体で矛盾しない。

## 7. 副作用

| 副作用 | 発生箇所 | 補足 |
| --- | --- | --- |
| DB読込 | Repository | 成果物メタ情報 |
| ファイル読込 | FileArtifactStore | 保存済み成果物 |
| ログ出力 | TraceLogWriter | 成功、拒否、失敗 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 成果物IDなし | HTTP 404へ変換する |
| 保存済みファイルなし | HTTP 404へ変換し、データ不整合として記録する |
| 許可外MIMEタイプ | HTTP 403へ変換する |
| ファイル読込失敗 | HTTP 500へ変換する |

## 9. ログ出力

| タイミング | ログ内容 |
| --- | --- |
| 取得開始 | trace_id、artifact_id |
| 安全検証失敗 | trace_id、artifact_id、理由 |
| 取得成功 | trace_id、artifact_id、MIMEタイプ、ファイル実体の読込結果 |
| 例外発生 | trace_id、エラー分類 |

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 成果物メタ情報の読取専用セッション開始時 |
| 終了点 | メタ情報読取完了 |
| ロールバック条件 | DB更新を行わないため該当なし |
