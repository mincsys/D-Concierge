# Codex成果物配信処理設計

## 1. 文書の目的

本書は、Codex成果物配信処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: 回答ブロック本文内の保存済み成果物URLをブラウザが取得する。
- 本処理は認証済みユーザだけが呼び出せる。
- 関連文書: `docs/03_内部設計/03_内部IF設計/成果物ファイルIF.md`

## 3. 処理概要

- 認証済みユーザが所有する通常操作対象のチャットに紐づく保存済み成果物IDからメタ情報を取得し、採用済み成果物領域内のファイルだけを配信する。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `presentation/rest` | 認証依存関係、HTTP入力検証、trace_id生成、HTTP応答変換 |
| `GetArtifactUseCase` | 成果物配信の調停 |
| `ChatRepository` | 成果物メタ情報取得 |
| `FileArtifactStore` | 保存済み成果物読込 |
| `PathSecurityService` | 保存領域内パス確認 |
| `TraceLogWriter` | 例外の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | REST境界でtrace_idを生成し、認証依存関係で現在ユーザを取得する。 |
| 2 | 成果物IDを検証する。 |
| 3 | Repositoryで認証済みユーザが所有する関連チャットが通常操作対象であることと、成果物メタ情報を取得する。 |
| 4 | FileArtifactStoreで保存済み成果物領域内のファイルを解決する。 |
| 5 | MIMEタイプと保存済みファイル実体を確認する。 |
| 6 | 保存済み成果物をストリーミング応答として返す。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- 成果物IDが採用済み回答に紐付いている。
- 認証済みユーザIDが確定している。
- 関連チャットが認証済みユーザに紐づく通常操作対象である。
- 保存済み成果物ファイルが存在する。

### 6.2. 事後条件

- 許可済みMIMEタイプの成果物内容だけが返る。
- 対象が存在しない場合や許可外の場合はファイル内容を返さない。

### 6.3. 不変条件

- Codex作業領域内の未採用ファイルを直接配信しない。
- 保存済み成果物領域外のファイルを配信しない。
- MIMEタイプは保存時メタ情報と配信時実体で矛盾しない。

## 7. 副作用

| 副作用 | 発生箇所 | 補足 |
| --- | --- | --- |
| DB読込 | Repository | 成果物メタ情報 |
| ファイル読込 | FileArtifactStore | 保存済み成果物 |
| ログ出力 | TraceLogWriter | 失敗 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 未ログイン | 未ログイン応答を返す |
| 成果物IDなし | HTTP 404へ変換する |
| 関連チャットが`deleting` | HTTP 409へ変換する |
| 保存済みファイルなし | HTTP 404へ変換し、データ不整合として記録する |
| 許可外MIMEタイプ | HTTP 403へ変換する |
| ファイル読込失敗 | HTTP 500へ変換する |

## 9. ログ出力

| タイミング | ログ内容 |
| --- | --- |
| 例外発生 | trace_id、artifact_id、エラー分類、例外種別、スタックトレース |

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 成果物メタ情報の読取専用セッション開始時 |
| 終了点 | メタ情報読取完了 |
| ロールバック条件 | DB更新を行わないため該当なし |
