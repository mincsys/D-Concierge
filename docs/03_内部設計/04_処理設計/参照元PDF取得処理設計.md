# 参照元PDF取得処理設計

## 1. 文書の目的

本書は、参照元PDF取得処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: 利用者が回答内のPDF参照元リンクを開く。
- 本処理は認証済みユーザだけが呼び出せる。
- 関連文書: `docs/03_内部設計/03_内部IF設計/参照元ファイルIF.md`

## 3. 処理概要

- 認証済みユーザが所有する通常操作対象のチャットに紐づく保存済み参照元IDからPDF locatorを取得し、共有データソース内の許可されたPDFだけを配信する。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `presentation/rest` | 認証依存関係、HTTP入力検証、trace_id生成、HTTP応答変換 |
| `GetReferenceDataUseCase` | 参照元取得の調停 |
| `ChatRepository` | 参照元メタ情報取得 |
| `FileReferenceStore` | PDFファイル取得 |
| `PathSecurityService` | 許可範囲判定 |
| `TraceLogWriter` | 例外の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | REST境界でtrace_idを生成し、認証依存関係で現在ユーザを取得する。 |
| 2 | 参照元IDを検証する。 |
| 3 | Repositoryで認証済みユーザが所有する関連チャットが通常操作対象であること、参照元メタ情報、locatorを取得する。 |
| 4 | PathSecurityServiceでPDF相対パスを正規化し、共有データソース配下であることを確認する。 |
| 5 | FileReferenceStoreでPDFファイルを開く。 |
| 6 | `application/pdf` としてストリーミング応答を返す。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- 参照元IDが保存済み回答に紐付いている。
- 認証済みユーザIDが確定している。
- 関連チャットが認証済みユーザに紐づく通常操作対象である。
- 共有データソースルートが設定済みである。

### 6.2. 事後条件

- 許可範囲内のPDFファイル内容だけが返る。
- 対象が存在しない場合や許可外の場合はファイル内容を返さない。

### 6.3. 不変条件

- 絶対パスや共有データソース外のパスを配信しない。
- PDF以外の参照元種別を本システムの配信対象にしない。
- 実ファイルパスをHTTP応答本文へ含めない。

## 7. 副作用

| 副作用 | 発生箇所 | 補足 |
| --- | --- | --- |
| DB読込 | Repository | 参照元メタ情報 |
| ファイル読込 | FileReferenceStore | PDFファイル |
| ログ出力 | TraceLogWriter | 失敗 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| 未ログイン | 未ログイン応答を返す |
| 参照元IDなし | HTTP 404へ変換する |
| 関連チャットが`削除中` | HTTP 409へ変換する |
| PDFファイルなし | HTTP 404へ変換する |
| パストラバーサル検知 | HTTP 403へ変換し、traceログへ記録する |
| ファイル読込失敗 | HTTP 500へ変換する |

## 9. ログ出力

| タイミング | ログ内容 |
| --- | --- |
| 例外発生 | trace_id、reference_id、エラー分類、例外種別、スタックトレース |

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 参照元メタ情報の読取専用セッション開始時 |
| 終了点 | メタ情報読取完了 |
| ロールバック条件 | DB更新を行わないため該当なし |
