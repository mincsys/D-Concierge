# 履歴一覧取得処理設計

## 1. 文書の目的

本書は、履歴一覧取得処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: フロントエンドが起動時、またはrun終端後に履歴一覧を取得する。
- 関連文書: `docs/03_内部設計/03_内部IF設計/チャットRepositoryIF.md`

## 3. 処理概要

- ローカル利用者のチャット履歴を更新日時降順で取得し、画面表示用の最新run状態を返す。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `ListChatHistoriesUseCase` | 履歴一覧取得の調停 |
| `ChatRepository` | チャットと最新runの検索 |
| `presentation/schemas/api.py` | API応答への変換 |
| `TraceLogWriter` | 例外の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | REST境界でtrace_idとローカル利用者IDを確定する。 |
| 2 | Repositoryで対象利用者のチャット一覧と最新runを取得する。 |
| 3 | 更新日時降順に整列された結果を画面応答スキーマへ変換する。 |
| 4 | 履歴一覧を返す。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- DB接続が利用可能である。
- ローカル利用者IDが確定している。

### 6.2. 事後条件

- 履歴0件の場合は空配列を返す。
- 各履歴にはチャットID、タイトル、最新状態、更新日時が含まれる。

### 6.3. 不変条件

- 履歴一覧取得はDBを更新しない。
- 回答ブロック本文や中間メッセージ全文は一覧へ含めない。

## 7. 副作用

| 副作用 | 発生箇所 | 補足 |
| --- | --- | --- |
| DB読込 | Repository | チャット一覧、最新run |
| ログ出力 | TraceLogWriter | 失敗 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| DB接続失敗 | HTTP 500へ変換する |
| 応答変換失敗 | システムエラーとしてtraceログへ記録する |

## 9. ログ出力

| タイミング | ログ内容 |
| --- | --- |
| 例外発生 | trace_id、利用者ID、エラー分類、例外種別、スタックトレース |

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | 読取専用セッション開始時 |
| 終了点 | 履歴一覧読取完了 |
| ロールバック条件 | 読取専用のためDB更新rollbackは発生しない |
