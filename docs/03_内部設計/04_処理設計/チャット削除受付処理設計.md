# チャット削除受付処理設計

## 1. 文書の目的

本書は、チャット削除受付処理の内部手順、副作用、例外、トランザクション境界を定義することを目的とする。

## 2. 前提

- 呼出契機: 利用者が削除確認ダイアログでOKを選択し、フロントエンドが `DELETE /api/chats/{chat_id}` を呼び出す。
- 本処理は削除要求の受付を担当し、作業領域、保存済み成果物実体、DBデータの物理削除完了を待たない。
- 物理削除は [チャット物理削除処理設計.md](チャット物理削除処理設計.md) に従って非同期に実行する。

## 3. 処理概要

- REST境界でチャットIDを検証する。
- Repositoryトランザクション内で対象チャットを`削除中`へ更新する。
- 対象チャットがすでに`削除中`の場合も、削除受付済みとして扱う。
- チャット状態更新後、`ChatDeletionDispatcher` へ削除ジョブを登録する。
- 削除ジョブ登録結果にかかわらず、チャット状態更新が成立していれば削除受付応答を返す。登録失敗はトレースログへ記録し、起動時再登録で再試行する。

## 4. 関連要素

| 要素 | 役割 |
| --- | --- |
| `create_api_router` | `DELETE /api/chats/{chat_id}` のREST境界 |
| `DeleteChatUseCase` | 削除受付の調停 |
| `ChatRepository` | チャット状態更新 |
| `ChatDeletionDispatcher` | 物理削除ジョブ登録 |
| `TraceLoggerPort` | 削除ジョブ登録失敗の記録 |

## 5. 処理シーケンス

| 順序 | 処理 |
| --- | --- |
| 1 | REST境界でチャットIDを検証し、trace_idを発行する。 |
| 2 | `DeleteChatUseCase` を呼び出す。 |
| 3 | Repositoryトランザクション内で対象チャットを`削除中`へ更新する。 |
| 4 | 対象チャットが存在しない場合は対象なしエラーとして返す。 |
| 5 | 対象チャットがすでに`削除中`の場合は更新済みと同じ削除受付結果へ変換する。 |
| 6 | トランザクションcommit後、`ChatDeletionDispatcher` へチャットIDとtrace_idを渡して削除ジョブを登録する。 |
| 7 | 登録結果が `registered` または `already_registered` の場合、削除受付応答を返す。 |
| 8 | 登録結果が `failed` の場合、トレースログへ登録失敗を記録し、削除受付応答を返す。 |

## 6. 事前条件 / 事後条件 / 不変条件

### 6.1. 事前条件

- チャットIDが空でない。
- DB接続とTransactionManagerが利用可能である。
- `ChatDeletionDispatcher` がcomposition rootで注入済みである。

### 6.2. 事後条件

- 対象チャットは`削除中`として保存される。
- 対象チャットは履歴一覧、履歴詳細、継続指示、参照元取得、成果物取得の通常操作対象外になる。
- 物理削除ジョブが登録済み、登録済み扱い、または起動時再登録で再試行可能な状態になる。
- API応答として `chat_id` と `chat_state` を返す。

### 6.3. 不変条件

- 削除受付処理は、作業領域、保存済み成果物実体、DB関連データの物理削除を実行しない。
- 削除受付成功後は、物理削除失敗があっても対象チャットを通常履歴へ復帰させない。
- 削除受付応答の `chat_state` は`削除中`を表す値だけを返す。

## 7. 副作用

| 種別 | 対象 | 内容 |
| --- | --- | --- |
| DB更新 | `chats` | チャット状態を`削除中`へ更新 |
| バックグラウンド登録 | `ChatDeletionDispatcher` | 物理削除ジョブ登録 |
| ログ出力 | トレースログ | 削除ジョブ登録失敗時の調査情報 |

## 8. 例外処理

| 条件 | 扱い |
| --- | --- |
| チャットID形式不正 | 入力エラーとして返す |
| 対象チャットなし | 対象なしエラーとして返す |
| すでに`削除中` | 削除受付済みとして `chat_id` と `chat_state` を返す |
| チャット状態更新失敗 | 削除受付失敗としてシステムエラーへ変換し、トレースログへ記録する |
| 削除ジョブ登録失敗 | トレースログへ記録し、削除受付応答は返す |

## 9. ログ出力

| 条件 | 内容 |
| --- | --- |
| チャット状態更新失敗 | trace_id、chat_id、例外種別、例外要約 |
| 削除ジョブ登録失敗 | trace_id、chat_id、登録結果、診断メッセージ |

通常の対象なし、すでに`削除中`の競合はトレースログへ記録しない。

## 10. トランザクション

| 項目 | 内容 |
| --- | --- |
| 開始点 | チャット状態更新開始時 |
| 終了点 | チャット状態更新commit時 |
| rollback条件 | チャット状態更新またはDB制約違反が発生した場合 |
| トランザクション外処理 | `ChatDeletionDispatcher` への登録と登録失敗ログ出力 |
