# 履歴検索と履歴整理

## 目的

本メモは、将来構想要件定義から分離したチャット履歴検索、ピン留め、タイトル編集の詳細を整理する。

## チャット履歴検索

検索対象:

- チャットタイトル。
- 利用者の質問文。
- 回答本文。
- 参照元ラベル。

検索対象外:

- 秘密情報を含む可能性がある内部ログ。
- Codex execの生JSONL全体。
- 管理者向けエラー詳細。
- トレースログ。

検索結果には、該当チャットのタイトル、更新日時、ヒットした本文の短い抜粋を表示する。

## ピン留め

ピン留めされたチャットは、履歴一覧の上部に表示する。

ピン留めの用途:

- 繰り返し参照する回答をすぐ開く。
- 調査中のチャットを見失わないようにする。
- 作業の起点となるチャットを固定する。

## タイトル編集

MVPでは、タイトルは最初の質問文から自動生成する。将来構想では、利用者が業務上わかりやすい名前へ変更できるようにする。

タイトル編集の例:

- 自動生成タイトル: `要件定義を成功させるポイント`
- 編集後タイトル: `IPA資料から整理した要件定義成功ポイント`

タイトル編集は、チャット作成者だけが行える。
