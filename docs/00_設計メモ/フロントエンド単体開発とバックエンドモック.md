# フロントエンド単体開発とバックエンドモック

## 目的

本メモは、バックエンド未実装時やバックエンドから切り離して画面を確認したい場合の、フロントエンド単体開発とテストの方針を整理する。

## 起動方針

フロントエンドを実バックエンドへ接続して確認する場合は、通常の開発サーバを起動する。

```bash
npm run dev
```

このモードでは、画面からの `/api/...` 通信を実バックエンドへ転送する。転送先は `VITE_BACKEND_TARGET` で指定し、未指定時はローカルの既定バックエンドへ接続する。

フロントエンドだけを単体で確認する場合は、バックエンドモック付きの開発サーバを起動する。

```bash
npm run dev:mock
```

このモードでは、Vite middleware が `/api/...` を受け、`mock/backend_mock/` 配下の固定データ、実行中状態、疑似SSE、参照元、Codex成果物を返す。バックエンド実装後も、フロントエンドだけを確認したい場合はこのモードを使う。

## 配置方針

モック用のコード、固定データ、参照元、Codex成果物は `mock/backend_mock/` に集約する。

- `data/`: app-config、履歴一覧、履歴詳細、SSEイベントテンプレート。
- `assets/references/`: 参照元データ取得APIで返す参照元PDF。
- `assets/artifacts/`: Codex成果物取得APIで返す画像やHTMLなど。
- `server/`: Vite middleware、APIハンドラ、疑似SSE、状態管理、MIMEタイプ解決。

画面コンポーネントは `mock/backend_mock/` を直接参照しない。画面は本番と同じように `mock/src/features/*/api/` のAPI境界を通じて `/api/...` へ通信する。

## Playwright確認

Playwright CLIでフロントエンド単体の画面確認を行う場合は、`npm run dev:mock` を起動し、Chromeで開く。

実バックエンドとの結合状態を確認する場合は、実バックエンドを起動した上で `npm run dev` を起動し、同じ画面操作を確認する。

個別の異常系や境界値だけを差し替えたい場合は、バックエンドモック本体を増やしすぎず、Playwright routeで対象テスト内だけレスポンスを上書きする。
