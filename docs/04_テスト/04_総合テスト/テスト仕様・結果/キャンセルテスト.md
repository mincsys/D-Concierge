# キャンセルテスト

## 1. 文書の目的

本書は、総合テスト方針で定義したBIZ-02 キャンセルに対応するテストケース、実施結果および証跡を記録することを目的とする。

## 2. 前提

- 本書は、`総合テスト方針.md` に基づいて作成する。
- 本書の対象は、利用者が画面で実行中として扱っているチャット実行処理のキャンセルとする。
- 受付、実行中、検証中のキャンセルを確認する。
- キャンセルされたチャット実行処理の部分回答、未検証回答、途中Codex成果物は最終回答として表示しない。
- 同じテスト対象を `チャット実行テスト.md`、`履歴再表示テスト.md` へ重複記載しない。

## 3. テスト対象

| 対象区分 | 対象 | 確認目的 |
| --- | --- | --- |
| 業務シナリオ | BIZ-02 キャンセル | 利用者が回答生成中のチャット実行処理をキャンセルし、キャンセル済みとして終端できることを確認する |
| 画面 | チャット画面 | キャンセルボタン、キャンセル要求中表示、キャンセル済み表示、キャンセル失敗表示を確認する |
| API/SSE | IF-SB-07、IF-SB-06 | キャンセル要求RESTとSSEのキャンセル結果が画面へ反映されることを確認する |
| 外部実行 | 実Codex CLI | 実行中のcodex execに対するキャンセルが画面上の終端状態へ反映されることを確認する |
| データ | PostgreSQL | キャンセル済み状態が履歴に残り、未検証回答が保存済み回答として扱われないことを確認する |
