# チャット画面

## 1. 文書の目的

本書は、利用者がユーザ指示、回答、実行状態、中間メッセージ、チャット履歴を確認し、継続指示やキャンセルを行うチャット画面の外部仕様を定義することを目的とする。

## 2. 前提

- チャット画面は、回答中、回答済み、キャンセル済み、エラー、タイムアウト、削除中または削除済みの状態を扱う。
- 履歴一覧はサイドバー内に表示する。
- 回答生成中は送信操作をキャンセル操作へ切り替える。
- 回答生成完了後も、同じチャットへ継続指示できる。
- 同じチャットに未完了のチャット実行処理がある場合、継続指示は送信できない。
- 回答本文内の保存済みCodex成果物画像は、回答ブロック内表示に加えて拡大表示できる。
- 履歴検索とアプリケーション設定変更は対象外である。
- チャット画面はログイン後に表示する保護対象画面である。
- 履歴タイトル、履歴一覧更新、SSE購読解除・再接続の共通ルールは「チャット履歴・実行中表示設計」に従う。

## 5. イベント一覧

### 5.1. 初期表示時

1. チャット画面を表示する。
2. `GET /api/chat-histories` で履歴一覧を取得する。
3. 履歴がある場合は、サイドバーにチャットタイトルだけを表示する。
4. 表示対象チャットが指定されている場合は、`GET /api/chats/{chat_id}` で履歴詳細を取得する。
5. 継続中のチャット実行処理がある場合は、保存済み内容を復元し、対象SSEへ再接続する。
6. ログイン中ユーザ情報をサイドバー左下へ表示する。

### 5.2. 新しいチャット選択時

1. 利用者が新しいチャット操作を選択する。
2. 表示中チャットでSSE購読中の場合は、利用者操作による購読解除として扱う。
3. 購読解除をエラー表示せず、開始画面を表示する。

### 5.3. その他メニュー表示時

1. チャット表示中に、利用者がその他メニューボタンを選択する。
2. 画面右上にその他操作メニューをポップアップ表示する。
3. メニュー内に `削除する` を赤系で表示する。
4. 利用者が `削除する` を選択した場合、削除確認ダイアログを表示する。
5. 利用者がメニュー外を選択するかEscキーを押した場合、その他操作メニューを閉じる。

### 5.4. 履歴項目メニュー表示時

1. 利用者が履歴項目にマウスホバーまたはキーボードフォーカスする。
2. 画面は該当履歴項目の右端にその他メニューボタンを表示する。
3. 利用者が履歴項目のその他メニューボタンを選択する。
4. 画面は履歴項目メニューをポップアップ表示する。このとき履歴選択は行わない。
5. メニュー内に `削除する` を赤系で表示する。
6. 利用者が `削除する` を選択した場合、削除確認ダイアログを表示する。
7. 利用者がメニュー外を選択するかEscキーを押した場合、履歴項目メニューを閉じる。

### 5.5. 継続指示送信時

1. 利用者が入力欄に継続指示を入力する。
2. 画面は入力欄が空でないことを確認する。
3. 同じチャットに未完了のチャット実行処理がないことを確認する。
4. `POST /api/chats/{chat_id}/runs` を呼び出す。
5. 受付成功時は、チャット実行処理IDとSSE URLを保持し、SSE購読を開始する。
6. 受付成功後、`GET /api/chat-histories` で履歴一覧を再取得する。
7. 入力不正または受付失敗時は、利用者向けエラーメッセージを表示し、画面遷移しない。

### 5.6. SSE購読時

1. `GET /api/chats/{chat_id}/runs/{run_id}/sse` へ接続する。
2. 接続成立直後の状態通知を受け取り、該当チャット実行処理の表示状態を更新する。
3. 接続成立直後の状態通知が完了、キャンセル済み、エラー、タイムアウトのいずれかである場合は、`GET /api/chats/{chat_id}` と `GET /api/chat-histories` を再取得する。
4. 接続成立直後の状態通知が継続中である場合は、SSE購読を継続する。
5. 中間メッセージを受け取った場合は、中間メッセージ領域へ追加表示する。システム固定の中間メッセージは、`作業を開始します。`、`作業が完了しました。`、`回答の検証を開始します。`、`回答の検証を完了しました。`、`回答を修正します。` の本文で表示する。
6. 最終回答を受け取った場合は、検証済み回答ブロック、各ブロックの参照元、Codex成果物を表示する。
7. キャンセル済み、エラー、タイムアウトを受け取った場合は、利用者向けメッセージを表示する。
8. 完了、キャンセル済み、エラー、タイムアウトのいずれかが確定した後、`GET /api/chat-histories` で履歴一覧を再取得する。
9. 利用者操作によるSSE購読解除はエラー扱いにしない。
10. 表示中チャットで意図せずSSEが切断された場合は、利用者向けエラーを表示する。

### 5.7. キャンセル時

1. 利用者がキャンセルボタンを選択する。
2. `POST /api/chats/{chat_id}/runs/{run_id}/cancel` を呼び出す。
3. キャンセル要求中は、キャンセルボタンをビジーアイコン表示へ切り替え、押下不可にする。
4. キャンセル要求を受け付けた場合は、対象チャット実行処理のスレッド領域に `処理をキャンセルしています。` を表示する。
5. SSEでキャンセル済みを受け取った場合は、対象チャット実行処理のスレッド領域に `処理をキャンセルしました。` を表示する。
6. キャンセル済みのチャット実行処理では、部分回答や未検証回答を最終回答として表示しない。
7. キャンセル要求に失敗した場合は、利用者向けエラーを表示する。

### 5.8. 履歴選択時

1. 利用者がサイドバーのチャットタイトルを選択する。
2. 表示中チャットでSSE購読中の場合は、利用者操作による購読解除として扱う。
3. `GET /api/chats/{chat_id}` で選択したチャットの履歴詳細を取得する。
4. チャット実行処理を履歴詳細取得結果の順序で表示する。
5. 継続中のチャット実行処理がある場合は、保存済み内容を復元し、対象SSEへ再接続する。
6. 履歴詳細取得に失敗した場合は、利用者向けエラーを表示する。

### 5.9. サイドバー操作時

1. 利用者が折りたたみ操作を選択した場合、サイドバーを縮小表示する。
2. 利用者が展開操作を選択した場合、サイドバーを通常表示へ戻す。
3. サイドバーの表示状態変更では、チャット実行処理の状態を変更しない。

### 5.10. 中間メッセージ開閉時

1. 利用者が中間メッセージ領域を選択する。
2. 折りたたみ中の場合は、中間メッセージ本文を展開表示する。
3. 展開中の場合は、中間メッセージ本文を折りたたみ表示する。
4. 開閉状態は画面表示だけの状態とし、チャット履歴の内容は変更しない。

### 5.11. 参照元を開く時

1. 利用者が回答ブロック本文内または回答ブロック直下の参照元リンクを選択する。
2. 選択した表示用参照元メタ情報を参照元ビューアへ渡す。
3. 参照元ビューア側で参照元本体取得URLから参照元データを取得し、表示する。

### 5.12. 回答内画像拡大表示時

1. 利用者が回答本文内の画像に表示される拡大表示操作を選択する。
2. 画面は対象画像のURLと代替テキストを画像ビューアへ渡す。
3. 画像ビューアはダイアログ内に対象画像を表示する。
4. 利用者は拡大、縮小、ドラッグによるパン、マウスホイールによるズーム、全体表示を行える。
5. 利用者が閉じる操作を選択した場合、画像ビューアを閉じ、チャット画面へ戻る。

### 5.13. チャット削除時

1. 利用者がその他メニューまたは履歴項目メニューで `削除する` を選択する。
2. 画面は削除確認ダイアログを表示する。
3. 利用者がOKを選択した場合、`DELETE /api/chats/{chat_id}` を呼び出す。
4. 利用者がキャンセルまたは閉じる操作を選択した場合、削除要求を送信せず元の画面表示を維持する。
5. 削除要求受付成功時は、対象チャットを履歴一覧から除外し、`GET /api/chat-histories` で履歴一覧を再取得する。
6. 表示中チャットが削除対象の場合、表示中チャットのSSE購読を利用者操作による解除として扱い、開始画面へ遷移する。
7. 表示中ではない履歴項目を削除した場合、現在表示中チャットを維持する。
8. 削除要求受付に失敗した場合、`チャットを削除できませんでした。時間を置いて再度お試しください。` を表示し、画面遷移しない。
9. 表示中チャットが別ブラウザなどで削除中になっていることを検知した場合、`このチャットは削除中のため操作できません。` を表示し、開始画面へ戻して履歴一覧を再取得する。
10. 表示中チャットが別ブラウザなどで削除済みになっていることを検知した場合、`このチャットは削除されました。` を表示し、開始画面へ戻して履歴一覧を再取得する。
