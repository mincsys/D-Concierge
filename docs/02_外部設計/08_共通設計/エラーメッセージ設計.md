# エラーメッセージ設計

## 1. 文書の目的

本書は、D-Concierge MVPで利用者へ表示するエラーメッセージの発生条件、表示文言、画面、トレースログとの関係を定義することを目的とする。

## 2. 前提

- 利用者向けエラーには内部パス、秘密情報、スタックトレースを含めない。
- エラー発生時は必要に応じてトレースログをファイル保存する。
- キャンセルは利用者操作の結果であり、失敗時を除き障害扱いにしない。
- 表示中チャットで意図せずSSEが切断された場合は利用者向けエラー表示とする。
- 利用者操作による画面遷移や別履歴表示でSSE購読を解除した場合は、エラー扱いにしない。

## 4. 表示方針

- 利用者が次に取れる行動を短く示す。
- 内部構成、絶対パス、コマンドライン、環境変数、秘密情報を表示しない。
- 回答表示の一部要素だけが失敗した場合、回答ブロック本文の閲覧を可能な範囲で継続する。
- キャンセル済み、エラー、タイムアウトは履歴詳細でも状態として表示する。

## 5. メッセージと状態の関係

| 実行状態 | 主な表示メッセージ |
| --- | --- |
| 受付 | 処理開始を示す画面表示。 |
| 実行中 | 中間メッセージ。 |
| 検証中 | 中間メッセージ。 |
| キャンセル要求中 | MSG-006 |
| キャンセル済み | MSG-007 |
| 完了 | 最終回答。 |
| エラー | MSG-002、MSG-004、MSG-005、MSG-014、MSG-015、MSG-016 |
| タイムアウト | MSG-009 |
