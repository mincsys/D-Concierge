# エラーメッセージ設計

## 1. 文書の目的

本書は、D-Conciergeで利用者へ表示するエラーメッセージの発生条件、表示文言、画面、トレースログとの関係を定義することを目的とする。

## 2. 前提

- 利用者向けエラーには内部パス、秘密情報、スタックトレースを含めない。
- 利用者向け表示文言はバックエンドの `shared/user_messages.py` に集約する。
- エラー発生時は必要に応じてトレースログをファイル保存する。
- キャンセルは利用者操作の結果であり、失敗時を除き障害扱いにしない。
- 表示中チャットで意図せずSSEが切断された場合は利用者向けエラー表示とする。
- 利用者操作による画面遷移や別履歴表示でSSE購読を解除した場合は、エラー扱いにしない。

## 4. 表示方針

- 利用者が次に取れる行動を短く示す。
- 内部構成、絶対パス、コマンドライン、環境変数、秘密情報を表示しない。
- 内部例外の診断文、スタックトレース、トレースログ向け `diagnostic_message` は利用者向けメッセージへ流用しない。
- 回答表示の一部要素だけが失敗した場合、回答ブロック本文の閲覧を可能な範囲で継続する。
- キャンセル済み、エラー、タイムアウトは履歴詳細でも状態として表示する。
- 削除中または削除済みの通常競合は、利用者向けメッセージを表示して開始画面へ戻す。物理削除失敗や予期しない内部失敗だけをトレースログ対象にする。
- ログイン画面、アカウント登録画面、設定ダイアログの入力エラーは、対象入力欄の近くに表示する。
- アカウント登録、ログイン、ログアウト、ユーザ名変更、パスワード変更、アカウント削除では、失敗時だけメッセージを表示する。

## 5. メッセージと状態の関係

| 実行状態 | 主な表示メッセージ |
| --- | --- |
| 受付 | 処理開始を示す画面表示。 |
| 実行中 | 中間メッセージ。 |
| 検証中 | 中間メッセージ。 |
| キャンセル要求中 | MSG-006 |
| キャンセル済み | MSG-007 |
| 完了 | 最終回答。 |
| エラー | MSG-002、MSG-004、MSG-005、MSG-014、MSG-015、MSG-016、MSG-040 |
| タイムアウト | MSG-009 |

チャット状態とメッセージの関係は次の通りである。

| チャット状態 | 主な表示メッセージ |
| --- | --- |
| 有効 | 通常の画面表示。 |
| 削除中 | MSG-019 |
| 対象なし | MSG-020 |

認証・アカウント操作とメッセージの関係は次の通りである。

| 操作 | 主な表示メッセージ |
| --- | --- |
| 認証状態確認 | MSG-021、MSG-033 |
| アカウント登録 | MSG-022、MSG-023、MSG-024、MSG-025、MSG-026、MSG-027、MSG-028、MSG-029、MSG-034 |
| ログイン | MSG-022、MSG-027、MSG-030、MSG-031、MSG-033、MSG-035 |
| ログアウト | MSG-021、MSG-036 |
| ユーザ名変更 | MSG-021、MSG-025、MSG-026、MSG-037 |
| パスワード変更 | MSG-021、MSG-027、MSG-028、MSG-029、MSG-032、MSG-038 |
| アカウント削除 | MSG-021、MSG-039 |
