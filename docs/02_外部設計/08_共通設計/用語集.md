# 用語集

## 1. 文書の目的

本書は、D-Concierge MVP外部設計書で使用する主要用語の意味を統一することを目的とする。

## 2. 用語一覧

| 用語 | 意味 |
| --- | --- |
| D-Concierge | 汎用的なユーザ指示応答・分析チャットアプリ。 |
| MVP | 本外部設計で対象とする最小実装範囲。 |
| 利用者 | チャット画面からユーザ指示を送信し、回答、参照元、中間メッセージ、過去履歴を確認する人。 |
| 共有1ユーザ | DBに事前登録された1ユーザを全利用者が共有するMVPの利用者識別方式。 |
| チャット | 一連のユーザ指示応答をまとめる単位。 |
| 継続指示 | 既存チャットに対して追加でユーザ指示を送信すること。 |
| チャット実行処理 | ユーザ指示1回ごとの受付、回答生成、検証、キャンセル、終了状態をまとめる処理単位。 |
| run_id | API上でチャット実行処理IDを表す契約名。 |
| chat_id | API上でチャットIDを表す契約名。 |
| Codex成果物 | 生成用codex execが作成し、保存済み領域へコピーされた回答表示用の成果物。 |
| artifact | APIパスや項目名でCodex成果物を表す契約名。 |
| 参照元 | 回答の根拠となる共有データソース上のデータと位置情報。 |
| 表示用参照元メタ情報 | 画面が参照元リンクと参照元ビューアを表示するために受け取る情報。参照元種別、表示ラベル、参照元本体取得URL、画面表示用の参照位置を含む。 |
| reference_id | 参照元本体取得APIのURLパスで参照元IDを表す契約名。 |
| 参照元ビューア | 参照元種別に応じて根拠資料を表示する画面部品。 |
| PDF参照元ビューア | PDF参照元を表示する設定例のビューア。 |
| codex exec | 回答生成または参照元検証を行う外部実行処理。 |
| 生成用codex exec | 利用者のユーザ指示に対する回答候補を生成するcodex exec。 |
| 検証用codex exec | 参照元が回答内容を支えているか確認するcodex exec。 |
| AGENTS.md | codex execの振る舞いを制御する指示ファイル。 |
| Skills | codex execが用途別処理に利用する指示・手順群。 |
| 出力契約 | 生成用codex execの最終回答、または検証用codex execの検証結果が満たすべき構造化出力の契約。 |
| 生成用Codex側の会話継続ID | 生成用codex execの継続指示で使う会話継続ID。D-ConciergeのチャットID、セッションID、チャット実行処理IDとは別に管理する。 |
| 検証用Codex側の会話継続ID | 検証用codex execの2回目以降の参照元検証で使う会話継続ID。生成用Codex側の会話継続IDとは別に管理する。 |
| SSE | サーバからWeb画面へ実行状態や中間メッセージを逐次配信する通信方式。 |
| REST受付 | ユーザ指示送信やキャンセルなどをHTTPリクエストとして受け付ける方式。 |
| 中間メッセージ | 回答生成や検証の途中状況を利用者へ示すメッセージ。 |
| 形式検証 | 回答候補が出力契約に適合するか確認する検証。 |
| 参照元検証 | 回答内容が参照元に支えられているか確認する検証。 |
| キャンセル | 利用者が実行中の回答生成または検証を取りやめる操作。 |
| タイムアウト | 設定された時間内に回答生成から検証完了まで到達しない状態。 |
| トレースログ | 障害調査用にファイル保存する内部記録。 |
| 共有データソース | 用途別アプリが検索・分析対象とする読み取り用データ。 |
| セッション作業領域 | codex execがチャット単位で利用する作業ディレクトリ。 |

## 3. 表記ルール

- 機能名、画面表示、状態名では「キャンセル」を使用する。
- 文書本文の論理名では「Codex成果物」を使用し、APIパスや項目名では `artifact` を契約名として使用する。
- 画面向けの参照元表示は、表示用参照元メタ情報を使用する。
- `SSE` は `GET /api/chats/{chat_id}/runs/{run_id}/sse` で購読するストリームを指す。
- チャット履歴一覧は `GET /api/chat-histories` で取得する。
