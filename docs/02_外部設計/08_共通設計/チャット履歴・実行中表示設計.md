# チャット履歴・実行中表示設計

## 1. 文書の目的

本書は、D-Concierge MVPにおけるチャット履歴、チャットタイトル、回答中チャットの再表示、SSE購読解除・再接続の共通ルールを定義することを目的とする。

## 2. 前提

- ユーザ指示1回ごとの処理単位はチャット実行処理とする。
- チャットは複数のチャット実行処理を持ち、各チャット実行処理は1つのユーザ指示を持つ。
- チャット履歴一覧は `GET /api/chat-histories` で取得する。
- チャット履歴詳細は `GET /api/chats/{chat_id}` で取得する。
- チャット履歴詳細では、バックエンドがチャット実行処理を開始日時の昇順で返し、画面は返却順に表示する。
- ユーザ指示受付APIは履歴一覧1行分の情報を返さない。
- 履歴一覧上にキャンセル操作は表示しない。

## 3. チャットタイトル

- 新規チャット受付時に、最初のユーザ指示本文からチャットタイトルを作成する。
- DBに保存するタイトルは最大50文字とし、ユーザ指示本文の先頭から50文字までを使用する。
- 改行や連続空白は、タイトル表示に適した形へ正規化してから保存する。
- 正規化後に空になった場合は、既定タイトルを保存する。
- 画面幅に収まらないタイトルは、画面側で末尾を `...` として省略表示する。
- MVPでは、AIによるタイトル生成、利用者によるタイトル編集、回答完了後の自動改題は行わない。

## 4. 履歴一覧の表示と制御情報

- サイドバーの履歴一覧で利用者に表示する文字情報は、チャットタイトルのみとする。
- 履歴一覧APIは、画面制御用に `chat_id`（チャットID）、`title`（チャットタイトル）、`updated_at`（最終更新日時）に加え、最新のチャット実行処理から導出した `latest_run_id`（最新チャット実行処理ID）、`latest_state`（最新チャット実行処理状態）を返す。
- `latest_state` と `updated_at` は契約名であり、サイドバーに文字表示しない。
- 履歴一覧は最終更新日時の降順で並べる。
- 回答中、検証中、キャンセル要求中などの状態をタイトル横にバッジ表示しない。

## 5. 履歴一覧の更新

- 初回ユーザ指示も継続指示も、REST受付時点で履歴表示対象にする。
- 画面はユーザ指示受付後に履歴一覧を再取得する。
- 画面はSSEで完了、キャンセル済み、エラー、タイムアウトを受け取った後に履歴一覧を再取得する。
- 履歴一覧再取得に失敗した場合でも、表示中チャットの実行表示とSSE購読は継続する。
- 最終更新日時はチャット実行処理開始時と、完了、キャンセル済み、エラー、タイムアウト確定時に更新する。
- 中間メッセージ受信だけでは最終更新日時を更新せず、履歴一覧の並び順を動かさない。

## 6. 回答中チャットの扱い

- 初回ユーザ指示の回答中でも、対象チャットは履歴一覧に表示する。
- 2ターン目以降の回答中でも、対象チャットは履歴一覧に表示する。
- 同一チャットに未完了のチャット実行処理がある場合、継続指示は送信不可とする。
- 回答中チャットを開いている場合のみ、対象チャット実行処理IDに対するキャンセル操作を表示する。
- キャンセル済み、エラー、タイムアウトのチャット実行処理は状態付きで履歴に残す。
- キャンセル済み、エラー、タイムアウトでは、未検証回答や途中Codex成果物を最終回答として表示しない。

## 7. 画面遷移とSSE

- 利用者操作で別履歴を開く、または画面を離れることでSSE購読を解除する場合は、エラー扱いにしない。
- SSE購読を解除しても、バックエンド側の実行は継続する。
- 回答中チャットへ戻った場合、画面は履歴詳細を取得し、チャット実行処理に紐づくユーザ指示、状態、中間メッセージを復元する。
- 履歴詳細で復元する中間メッセージは、画面表示用に整形・マスク済みの保存済み本文だけとする。
- 状態通知、最終回答、エラー、キャンセル結果は配信用の一時通知として扱い、汎用イベントとして永続化しない。
- 履歴詳細取得時点で対象チャット実行処理が継続中の場合、画面はSSEへ再接続する。
- SSE再接続時に過去の配信内容は再送しない。
- SSE接続成立直後に、画面は対象チャット実行処理の現在状態を状態通知として必ず受け取る。
- 履歴詳細取得時点では継続中でもSSE再接続直前に終了状態へ変わっていた場合、画面は接続成立直後の状態通知で終了状態を検知する。
- 接続成立直後の状態通知が完了、キャンセル済み、エラー、タイムアウトのいずれかである場合、画面は履歴詳細と履歴一覧を再取得し、回答、利用者向けメッセージ、履歴順を復元する。
- 再接続後は、接続成立直後の現在状態通知と、その時点以降の状態変化だけをSSEで受け取る。
- 表示中チャットで意図せずSSEが切断された場合は、利用者向けエラーを表示する。
