# 固定検証でMermaid記法を確認できていない

## 内容

現在の固定検証は、生成用Codexの最終JSON形式、回答本文、PDF参照元path、ページ範囲などを確認している。

一方、回答本文内の `mermaid` fenced code block が、フロントエンドで実際に描画可能なMermaid記法になっているかは固定検証していない。そのため、検証済み回答として採用された後に、画面で `Mermaid図を表示できませんでした。` となる場合がある。

## 影響

回答本文としては保存・表示できるが、図解を期待した回答でMermaid図が描画されず、利用者が画面上で初めて不正な図表記に気付く。

## 対応方針

バックエンドの回答検証処理に、Markdown本文から `mermaid` fenced code block を抽出するローカル表示検証を追加する。

Mermaid文法はPythonで再実装せず、フロントエンドと同じ `mermaid` パッケージを使うNodeスクリプトへ検証を委譲する。初期実装では `mermaid.parse()` による軽量な構文検証を行い、失敗時は固定検証NGとして生成用CodexへMermaidブロックの修正を促す再生成指示を返す。

画面描画そのものまで厳密に保証する必要が出た場合は、次段階で `mermaid.render()` とDOM環境、またはPlaywrightによる総合テストへ拡張する。

## 設計と実装の評価

設計の改善案として扱うのがよい。現行設計ではMarkdown/Mermaid表示はフロントエンド表示責務として定義されているが、回答採用前に画面で描画できないMermaidを検出する要求が後から出ている。

現行実装は設計の最低限には合っているが、利用者体験としては検証済み回答に描画不能な図が残るため不十分。第3案として、バックエンド固定検証に軽量なMermaid構文検証を追加し、画面描画そのものは総合テストで担保する。
