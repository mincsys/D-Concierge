## 内容

正式総合テスト結果は round-10 の Playwright/Chrome および実 Codex UI 追加確認後、現時点では `合格 94件 / 部分確認 0件 / 保留 7件` である。総合テスト方針の完了条件である全ケース実施済み・全ケース合格には達していない。

round-5 では、認証、アカウント管理、チャット受付失敗、履歴一覧失敗の 15 件を Chrome Playwright で追加確認した。round-5b では、認証警告、アカウント管理、履歴一覧・再表示、チャット削除の 11 件を Chrome Playwright で追加確認した。round-8 では、履歴再表示、キャンセル、チャット削除の 12 件を Chrome Playwright で追加確認し、ユーザ明示の認証済み Codex ホームを使って `ST-CHAT-025` を Codex Docker smoke として追加確認した。round-9 では、チャット回答表示、危険HTML抑止、参照元表示、成果物表示、未完了run競合、参照元/成果物異常表示、キャンセル操作状態、キャンセル連打、キャンセル失敗、削除受付失敗、履歴再表示の 16 件を Chrome Playwright で追加確認した。round-10 では、継続指示受付失敗、SSE接続失敗/途中切断、生成/検証/タイムアウト/AIサービスプロバイダ側エラー、実行中/検証中キャンセル、キャンセル後遅延イベント、実行中チャット削除、別ブラウザ競合、トレースログ保持、削除中再送、ユーザ別分離の 18 件を Chrome Playwright で追加確認した。これにより、合格は 94 件になった。

round-10 では認証済み `$HOME/.codex` を使うbackend/frontend経由で実 Codex UI 新規チャットも再試行したが、画面終端が `エラー発生` だったため、`ST-CHAT-005` と `ST-CHAT-006` は保留を維持する。新規チャット正常完了に到達しなかったため、`ST-CHAT-011` の継続指示と `ST-HISTORY-008` の履歴からの継続指示も保留を維持する。

正式総合テストの全ケースについて、Playwright/Chromeで確認すべき画面操作・表示・遷移は直接証跡を保存し、API/DB/ファイル等の補助確認は補助根拠として整理したうえで、全ケースが合格になること。

## 再実施条件

残る保留 7 件は、backend/frontend を実利用に近い状態で起動し、Chrome Playwright でケースごとの操作、表示、遷移、SSE 状態、削除後状態、DB/ファイル状態を実行ログまたはスクリーンショットとして公式 evidence に保存する。実 Codex 正常実行が必要なケースは、認証済み Codex ホームまたは `CODEX_API_KEY` を使用し、画面起点の送信、SSE、回答表示、継続指示、履歴再表示までをケース単位で記録する。Playwright/Chrome で確認できないケースは、ケース ID ごとに確認不能理由と代替確認条件を記録する。
