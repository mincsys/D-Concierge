# Viteのchunkサイズ警告解消検討

## 背景

`npm run build` は成功しているが、Vite のビルド結果で一部 chunk が 500KB を超える警告が出ている。

主な対象は Mermaid 関連の非同期 chunk と、初期 bundle の `index-*.js` である。

## 現状

- `mermaid` は `MermaidRenderer.tsx` で通常 import している。
- Markdown 表示経由で Mermaid 図を描画するため、Mermaid 図を表示しない利用時にも初期 bundle へ影響する可能性がある。
- `pdfjs-dist` など、常時使わない viewer 系ライブラリも初期読み込みに含まれている可能性がある。
- 警告はビルド失敗ではないため、現時点で機能停止は発生していない。

## 検討事項

### Mermaid の遅延読み込み

- `import mermaid from "mermaid"` をやめ、Mermaid 図が必要になったタイミングで `import("mermaid")` する。
- `mermaid.initialize(...)` は遅延読み込み後に一度だけ実行する形へ整理する。
- 初回 Mermaid 表示時は読み込み待ちが発生するため、読み込み中表示と失敗時表示を用意する。
- 画面切替時に古い非同期描画結果が反映されないよう、既存のキャンセル制御を維持する。

### PDF viewer 等の遅延読み込み

- PDF 参照表示、Mermaid 拡大表示、画像拡大表示など、常時使わない viewer を `React.lazy` などで分割する。
- 初回表示時の loading 表示と、PDF worker 設定タイミングに注意する。

### manualChunks の検討

- 必要に応じて `vite.config.ts` の `build.rollupOptions.output.manualChunks` で Mermaid、Markdown、PDF 系を分割する。
- ただし、manualChunks は警告を整理する効果はあるが、初期読み込み量の根本削減には遅延読み込みが必要である。

## 対応方針案

1. Mermaid の遅延読み込みを先に対応する。
2. `npm run build` の chunk 警告と初期 bundle サイズの変化を確認する。
3. まだ初期 bundle が大きい場合、PDF viewer などの遅延読み込みを追加検討する。
4. 警告だけが残る場合は、manualChunks または `chunkSizeWarningLimit` の調整要否を判断する。

## 完了条件

- `npm run build` が成功する。
- Mermaid 図、Markdown 表示、PDF 参照表示が従来どおり利用できる。
- 初期 bundle または対象 chunk のサイズが低減している。
- 残る警告がある場合は、警告を許容する理由または追加対応方針が整理されている。
